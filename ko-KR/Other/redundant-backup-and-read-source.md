# 중복 백업과 읽기 소스 전환

중복 백업은 이미 업로드된 파일을 다른 저장 채널에도 한 벌 더 저장하는 기능입니다.

사용자가 파일을 볼 때는 원본과 백업의 차이가 거의 없습니다. 실제로 어느 저장 채널에서 읽어 오느냐만 달라집니다.

## 할 수 있는 일

| 기능 | 설명 |
| --- | --- |
| 파일을 한 번 더 저장 | 다른 업로드 채널에 사본을 만들어 단일 저장소 장애 위험을 줄입니다 |
| 읽기 소스 전환 | 백업 후 원본 채널과 백업 채널 사이에서 읽기 위치를 바꿀 수 있습니다 |
| 단일 파일 백업 | 파일 상세 화면에서 중요한 파일 하나만 백업합니다 |
| 일괄 백업 | 관리자 화면에서 여러 파일을 선택해 한 번에 백업합니다 |
| 전체 보강 백업 | 기타 설정에서 폴더 단위로 빠진 백업을 채웁니다 |

## 설정 위치

```text
시스템 설정 -> 기타 설정 -> 중복 백업
```

![중복 백업](../../image/other/冗余备份截图.png)

이곳에서는 특정 폴더 또는 전체 파일에 대해 백업을 추가할 수 있습니다. 백업 채널은 직접 선택할 수도 있고, 자동 전환을 선택해 시스템이 적절한 채널을 고르게 할 수도 있습니다.

## 파일 상세에서 백업하기

관리 화면에서 파일 상세를 열면 해당 파일만 바로 백업할 수 있습니다.

![파일 상세 백업](../../image/other/文件详情里文件备份.png)

중요한 파일을 나중에 따로 보호하고 싶을 때 적합합니다. 백업이 성공하면 파일 상세에 읽기 소스 전환 항목이 표시됩니다.

## 여러 파일 일괄 백업

관리 화면에서 여러 파일을 선택한 뒤 일괄 백업을 실행할 수 있습니다.

![일괄 백업](../../image/other/批量备份截图.png)

파일 상세 백업, 선택 백업, 기타 설정의 중복 백업은 모두 같은 백업 기능을 다른 입구에서 실행하는 방식입니다.

## 백업 후 읽기 소스 전환

| 읽기 소스 | 설명 |
| --- | --- |
| 원본 채널 | 처음 업로드한 저장 채널에서 읽습니다 |
| 백업 채널 | 중복 백업으로 만든 사본 채널에서 읽습니다 |

![읽기 소스 전환](../../image/other/备份成功切换读取源.png)

어느 쪽으로 전환하든 이미지 표시, 동영상 재생, 다운로드 링크는 평소처럼 사용할 수 있습니다.

## 건너뛰는 경우

| 경우 | 이유 |
| --- | --- |
| 이미 백업됨 | 중복 저장으로 공간을 낭비하지 않기 위해 |
| 원본과 백업 채널이 같음 | 같은 저장소에 다시 저장하면 백업 의미가 없기 때문에 |
| 사용 가능한 백업 채널 없음 | 다른 적절한 저장 채널을 찾지 못했기 때문에 |

## 삭제할 때

파일을 삭제하면 원본 파일과 백업 파일이 함께 삭제됩니다.
