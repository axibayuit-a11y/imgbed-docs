# 연합 분산 인덱스

연합 분산 인덱스는 여러 ImgBed 사이트가 서로 파일 목록을 공유할 수 있게 합니다.

간단히 말하면:

- 내 사이트의 선택한 폴더를 다른 사람과 공유할 수 있습니다.
- 다른 노드에 참여하고, 그 노드가 공유한 파일 목록을 내 관리자 패널로 동기화할 수 있습니다.
- 연합 파일은 주로 탐색, 검색, 링크 열기에 사용됩니다. 내 저장소로 다시 업로드되는 것은 아닙니다.

## 설정 위치

엽니다:

```text
System Settings -> Other Settings -> Federated Distributed Index
```

![로컬 연합 노드](../../image/other/联盟图/联盟分布式索引本地节点.png)

페이지에는 세 개의 탭이 있습니다.

| 탭 | 목적 |
| --- | --- |
| 로컬 노드 | 내 노드를 활성화하고, 공개 도메인을 확인하고, 공유 폴더를 선택하며, 송신 인덱스를 업데이트합니다 |
| 내가 참여한 노드 | 내가 참여한 다른 ImgBed 노드를 관리합니다 |
| 내 노드에 참여하려는 노드 | 내 노드에 참여하려는 다른 사람의 요청을 관리합니다 |

## 처음 설정하기

1. `Local Node`를 엽니다.
2. `Enable`을 켭니다.
3. `Sync folders`에서 공유할 폴더를 선택합니다.
4. `Update Outbound Index`를 클릭합니다.
5. ImgBed가 도메인 변경을 감지하면 계속하기 전에 현재 도메인이 올바른지 확인합니다.

동기화 폴더는 여러 개 선택할 수 있습니다.

동기화 폴더 목록이 비어 있으면 모든 폴더가 공유됩니다.

## 로컬 노드

### 공개 도메인

공개 도메인은 다른 노드가 내 노드에 접근할 때 사용하는 사이트 URL입니다.

ImgBed가 이를 자동으로 감지하므로 직접 입력할 필요가 없습니다. 처음 인덱스를 업데이트할 때 ImgBed는 현재 접근 URL이 운영 도메인인지 확인합니다.

나중에 도메인을 변경하면 인덱스를 업데이트할 때 다시 확인을 요청합니다.

### 동기화 폴더

동기화 폴더는 연합 노드와 공유할 파일을 결정합니다.

예를 들어 다음만 선택하면:

```text
/1/
/2/
```

다른 노드는 이 두 디렉터리 안의 파일만 볼 수 있습니다.

### 송신 인덱스 업데이트

다른 노드가 내 사이트에서 동기화할 수 있는 파일 목록을 업데이트합니다.

다음 경우에 사용합니다.

- 연합 기능을 처음 활성화할 때.
- 공유하려는 파일을 업로드했을 때.
- 동기화 폴더를 변경했을 때.
- 공개 도메인을 변경했고 확인이 필요할 때.

## 내가 참여한 노드

`Nodes I Joined`는 다른 노드를 구독하는 곳입니다.

![내가 참여한 노드](../../image/other/联盟图/我加入的节点.png)

### 다른 노드에 참여 요청하기

1. 상대 소유자에게 초대 링크를 요청합니다.
2. 입력 상자에 붙여 넣습니다.
3. `Request to Join`을 클릭합니다.
4. 상대 소유자가 자신의 관리자 패널에서 승인할 때까지 기다립니다.

승인되면 노드 상태가 승인됨으로 바뀝니다.

### 수신 인덱스 업데이트

`Update Inbound Index`는 내가 참여한 노드에서 파일 목록을 동기화합니다.

다음 경우에 사용합니다.

- 상대 소유자가 방금 내 요청을 승인했을 때.
- 상대 소유자가 공유 콘텐츠가 업데이트되었다고 알려 주었을 때.
- 참여한 모든 연합 파일 목록을 새로고침하고 싶을 때.

특정 노드 하나만 업데이트하려면 해당 노드 카드에서 `Update Index`를 클릭합니다.

![인덱스 업데이트](../../image/other/联盟图/更新索引.png)

### 구독 취소

더 이상 특정 노드를 동기화하지 않으려면 `Unsubscribe`를 클릭합니다.

구독을 취소하면 해당 노드의 연합 인덱스가 로컬 사이트에서 제거됩니다.

## 내 노드에 참여하려는 노드

`Nodes Joining Me`는 다른 사람의 요청을 처리하는 곳입니다.

![내 노드에 참여하려는 노드](../../image/other/联盟图/加入我的节点.png)

### 초대 링크 생성하기

1. 로컬 노드가 활성화되어 있는지 확인합니다.
2. ImgBed가 공개 도메인을 확인할 수 있도록 `Update Outbound Index`를 최소 한 번 클릭합니다.
3. `Nodes Joining Me`를 엽니다.
4. `Reset Invitation Link`를 클릭합니다.
5. 초대 링크를 복사해 상대 소유자에게 보냅니다.

초대 링크가 비어 있다면 보통 공개 도메인이 아직 확인되지 않은 것입니다. `Local Node`로 돌아가 `Update Outbound Index`를 클릭하세요.

### 참여 요청 처리하기

누군가 요청을 제출하면 `Nodes Joining Me` 목록에 표시됩니다.

| 작업 | 의미 |
| --- | --- |
| Approve | 상대 노드가 내 공유 파일 목록을 동기화할 수 있게 합니다 |
| Reject | 참여 요청을 거절합니다 |
| Delete | 완료된 기록을 삭제합니다 |
| Check Status | 상대 측이 이 관계를 계속 유지하는지 확인합니다 |

승인 후에도 상대 측에서 내 공유 파일을 보려면 `Update Inbound Index`를 클릭해야 합니다.

![초대된 노드 승인](../../image/other/联盟图/邀请节点同意.png)

## 메시지

관계가 승인된 뒤에는 노드 카드에서 `Message`를 클릭합니다.

메시지는 연합 관계에 대한 연락 용도로만 사용됩니다. 파일, 태그, 디렉터리, 권한은 변경하지 않습니다.

![메시지](../../image/other/联盟图/留言功能.png)

## 연합 파일 보기

동기화가 완료되면 관리자 파일 목록으로 돌아갑니다.

페이지 상단에서 로컬 파일과 연합 파일을 전환합니다. 연합 파일에서는 동기화된 콘텐츠를 탐색할 수 있습니다.

연합 파일은 주로 보기, 검색, 미리보기, 링크 복사에 사용됩니다. 로컬 파일이 아니므로 내 사이트에서 이동, 삭제, 태그 변경, 백업을 할 수 없습니다.

![관리자의 연합 파일](../../image/other/联盟图/联盟管理显示效果图.png)

## 자주 묻는 질문

### 관계 기록이 없어 다시 신청하라고 표시되는 이유는 무엇인가요?

대개 상대 측에서 나를 삭제하고 기록도 제거하여 더 이상 관계를 찾을 수 없다는 의미입니다. 새 참여 요청을 제출하세요.

![관계 기록이 없을 때 재신청](../../image/other/联盟图/无关系记录重新申请.png)

### 참여 후 파일이 보이지 않는 이유는 무엇인가요?

다음을 확인하세요.

1. 상대 소유자가 내 요청을 승인했습니다.
2. 상대 소유자가 `Update Outbound Index`를 클릭했습니다.
3. 내가 `Update Inbound Index`를 클릭했습니다.
4. 상대 소유자의 동기화 폴더에 공유하려는 디렉터리가 포함되어 있습니다.

### 도메인 변경이 감지되면 어떻게 해야 하나요?

현재 운영 도메인으로 관리자 패널을 열고 있다면 확인하고 계속합니다.

임시 주소를 사용 중이라면 취소하고, 운영 도메인으로 관리자 패널을 다시 연 뒤 다시 시도합니다.

### 동기화 폴더 목록이 비어 있다는 것은 무슨 뜻인가요?

동기화 폴더 목록이 비어 있으면 모든 폴더가 공유된다는 뜻입니다.

일부 디렉터리만 공유하려면 해당 폴더를 수동으로 선택합니다.

### 송신 인덱스 업데이트와 수신 인덱스 업데이트의 차이

| 버튼 | 간단한 의미 |
| --- | --- |
| Update Outbound Index | 다른 사람이 나에게서 동기화할 수 있는 내용을 업데이트합니다 |
| Update Inbound Index | 내가 다른 사람에게서 동기화한 내용을 업데이트합니다 |
