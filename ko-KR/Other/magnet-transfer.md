# 마그넷 링크 전송 안내

마그넷 링크 기능은 마그넷 링크 안의 파일을 내려받은 뒤, 지정한 클라우드 업로드 채널로 자동 업로드합니다.

애니메이션, 동영상, 압축 파일 등을 옮길 때 유용합니다. 링크를 붙여 넣으면 백그라운드에서 다운로드 작업이 만들어지고, 완료 후 업로드 목록에 최종 링크가 표시됩니다.

![마그넷 링크](../../image/other/磁力链接/磁力链接.png)

## 사용 위치

홈 화면의 업로드 영역에서 사용할 수 있습니다.

마그넷 링크를 입력창에 붙여 넣고, 모드를 `전송`으로 선택한 뒤 업로드를 누릅니다.

![마그넷 전송 업로드](../../image/other/磁力链接/上传番剧.png)

## 사용 전 준비

관리 화면에서 마그넷 전송 설정을 먼저 완료해야 합니다.

보통 다음 항목이 필요합니다.

| 준비 항목 | 용도 |
| --- | --- |
| GitHub 계정 | 다운로드 작업 실행에 사용 |
| 업로드 채널 | Google Drive, OneDrive 같은 최종 저장 위치 |
| 저장 디렉터리 | 전송 완료 파일이 저장될 경로 |
| 작업 제한 시간 | 오래 걸리는 작업의 타임아웃 |

## 전송 방법

1. 홈 화면 업로드 입력창에 마그넷 링크를 붙여 넣습니다.
2. 모드가 `전송`인지 확인합니다.
3. 업로드를 누릅니다.
4. 시스템이 마그넷 작업을 만들 때까지 기다립니다.
5. 작업이 시작되면 오른쪽 아래의 `마그넷 작업` 창에서 진행률을 확인합니다.

다운로드와 업로드에는 시간이 걸릴 수 있습니다. 속도는 마그넷 소스 상태, GitHub 실행 환경, 선택한 저장 채널에 따라 달라집니다.

![다운로드 중](../../image/other/磁力链接/磁力链接下载中.png)

## 완료 후

작업이 끝나면 업로드 목록에 파일명과 링크가 표시됩니다.

동영상은 동영상 미리보기로, 이미지는 이미지 미리보기로, 그 외 파일은 일반 파일 아이콘으로 표시됩니다.

![완료된 동영상](../../image/other/磁力链接/下载好后的视频.png)

다음 형식의 링크를 바로 복사할 수 있습니다.

| 링크 형식 | 사용처 |
| --- | --- |
| 원본 링크 | 파일을 직접 열 때 |
| Markdown | Markdown 글이나 설명에 붙여 넣을 때 |
| HTML | 웹 페이지 코드에 넣을 때 |
| BBCode | BBCode를 지원하는 게시판에서 사용할 때 |

## 작업 상태

| 상태 | 의미 |
| --- | --- |
| 대기 중 | 작업이 생성되어 실행을 기다리는 중 |
| 다운로드 중 | 마그넷 리소스를 내려받는 중 |
| 업로드 중 | 내려받은 파일을 클라우드 채널로 업로드하는 중 |
| 완료 | 업로드가 끝나 링크를 복사할 수 있음 |
| 실패 | 안내 메시지를 확인한 뒤 다시 시도해야 함 |

## 문제 해결

- 링크를 붙여 넣었는데 시작하지 않으면 마그넷 전송이 켜져 있고, GitHub 계정과 업로드 채널이 선택되어 있는지 확인합니다.
- 다운로드가 계속 느리면 마그넷 자체에 유효한 피어가 부족할 수 있습니다.
- 완료 후 미리보기가 보이지 않으면 먼저 파일 링크가 직접 열리는지 확인합니다.
- 작업이 실패하면 마그넷 링크, 저장 채널, 업로드 디렉터리 설정을 다시 확인합니다.
