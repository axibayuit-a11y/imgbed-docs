# 사용자 빈도 제한

사용자 빈도 제한은 짧은 시간에 업로드나 요청이 과도하게 발생할 때 이용을 제한하는 기능입니다.

공개 업로드를 허용하는 사이트라면 스팸 업로드나 과도한 사용을 줄이기 위해 설정해 두는 것이 좋습니다.

## 설정 위치

```text
시스템 설정 -> 보안 설정 -> 사용자 빈도 제한
```

![빈도 제한](../../image/other/用户频控截图.png)

## 주요 설정

| 항목 | 설명 |
| --- | --- |
| 사용 | 빈도 제한을 켜거나 끕니다 |
| 시간 범위 | 제한을 계산할 시간 구간 |
| 최대 횟수 | 해당 시간 안에 허용할 작업 횟수 |
| 대상 작업 | 업로드 등 제한할 작업 |
| 차단 시간 | 제한을 넘었을 때 얼마 동안 막을지 |

## 설정 예시

공개 업로드를 허용한다면 처음에는 너무 빡빡하지 않게 시작하고, 실제 사용량을 보며 조정하는 편이 좋습니다.

```text
10분 동안 30회까지 허용
초과 시 30분 차단
```

이 정도면 일반 사용자의 연속 업로드는 크게 방해하지 않으면서, 비정상적인 반복 업로드는 막기 쉽습니다.

## 제한 안내

제한에 걸린 사용자는 업로드나 작업이 거부되었다는 안내를 보게 됩니다.

![빈번한 오류 안내](../../image/other/频繁报错提示.png)

## 조정 팁

- 제한이 너무 강하면 정상 사용자의 일괄 업로드도 막힐 수 있습니다.
- 공개 사이트에서는 처음부터 완전 무제한으로 두지 않는 것이 안전합니다.
- IP 위치 조회와 사용자 관리를 함께 쓰면 의심스러운 사용을 더 쉽게 추적할 수 있습니다.
- 이벤트나 일시적인 사용량 증가가 예상되면 한시적으로 제한을 완화해도 됩니다.
