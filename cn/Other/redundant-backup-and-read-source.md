# 冗余备份与读取源切换

冗余备份用来给已经上传的文件再存一份副本。  
主文件和备份文件都可以作为读取源使用，平时看起来没有区别；区别只是文件实际从哪个存储渠道读取。

## 冗余备份能做什么

| 功能 | 说明 |
| --- | --- |
| 多存一份文件 | 把文件备份到另一个上传渠道，降低单个渠道出问题时的风险。 |
| 切换读取源 | 文件备份成功后，可以在主渠道和备份渠道之间切换读取。 |
| 单文件备份 | 在文件详情里给某一个文件备份。 |
| 批量备份 | 在管理页框选多个文件后批量备份。 |
| 全局冗余备份 | 在其他设置里按文件夹批量补备份。 |

## 冗余备份入口

冗余备份在：

系统配置 -> 其他设置 -> 冗余备份

![冗余备份截图](../../image/other/冗余备份截图.png)

这里适合给某个文件夹或全部文件统一补备份。  
备份渠道可以手动选择，也可以选择自动切换，让系统自己找合适的备份渠道。

## 文件详情里备份

在管理后台点开文件详情，可以直接点击备份文件。

![文件详情里文件备份](../../image/other/文件详情里文件备份.png)

这个入口适合临时给单个重要文件补备份。  
备份成功后，文件详情里会出现可切换的读取源。

## 框选批量备份

在管理后台可以框选多个文件，然后执行批量备份。

![批量备份截图](../../image/other/批量备份截图.png)

这个入口适合一次处理一批文件。  
框选备份、文件详情备份、其他设置里的冗余备份，本质上都是同一个备份功能，只是入口不同。

## 备份成功后切换读取源

备份完成后，文件详情里可以切换读取源：

| 读取源 | 说明 |
| --- | --- |
| 主渠道 | 从原始上传渠道读取文件。 |
| 备份渠道 | 从备份渠道读取文件。 |

![备份成功切换读取源](../../image/other/备份成功切换读取源.png)

主渠道和备份渠道对用户访问来说没有区别。  
切到哪个读取源，后续访问文件时就优先从哪个渠道读取。

## 哪些情况会跳过

备份时遇到下面情况会跳过，不是出错：

| 情况 | 为什么跳过 |
| --- | --- |
| 已经有备份 | 一个文件已经有备份时，不会重复备份。 |
| 主备渠道相同 | 备份渠道不能和主渠道一样，否则只是把文件放回原来的地方，没有备份意义。 |
| 没有可用备份渠道 | 当前没有合适的其他渠道可用。 |

简单理解就是：备份必须放到另一个渠道里；已经备份过的文件不会重复占空间。

## 主渠道和备份渠道怎么理解

| 名称 | 说明 |
| --- | --- |
| 主渠道 | 文件第一次上传时使用的渠道。 |
| 备份渠道 | 后来通过冗余备份生成的副本渠道。 |
| 主读取源 | 当前从主渠道读取。 |
| 备份读取源 | 当前从备份渠道读取。 |

主读取源和备份读取源没有功能高低之分。  
只要备份文件可用，切到备份读取源后，图片、视频、下载链接这些正常访问方式都照常使用。

## 删除文件时会发生什么

删除文件时，系统会同时删除主文件和备份文件。
