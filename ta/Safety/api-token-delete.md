# API Token மூலம் கோப்பு நீக்கம்

API Token மூலம் கோப்புகளை நீக்குவது ஸ்கிரிப்ட்கள், தானியக்கப் பணிகள் மற்றும் மூன்றாம் தரப்பு நிரல்களுக்காக வடிவமைக்கப்பட்டுள்ளது. நிர்வாகப் பக்கத்தைத் திறக்க தேவையில்லை. தள URL, token மற்றும் தெளிவான கோப்பு ID-களை வழங்கினால், ImgBed-இலிருந்து ஒரு கோப்பையோ பல கோப்புகளையோ நீக்கலாம்.

நீக்கம் என்பது எழுதும் செயல்; கட்டளை இயக்கப்பட்ட பிறகு தரவு உண்மையாகவே நீக்கப்படும். முதலில் `imgbed-token-list.mjs` பயன்படுத்தி நீக்க வேண்டிய `fileId` மதிப்புகளை உறுதிப்படுத்தி, பின்னர் அந்த ID-களை நீக்க ஸ்கிரிப்டுக்கு வழங்கவும்.

![API Token திருத்துதல்](../../image/Safety/apitoken/编辑api token.png)

## தொடங்குவதற்கு முன்

நிர்வாகப் பலகையைத் திறந்து இங்கே செல்லவும்:

```text
System Settings -> Security Settings -> API Token
```

API Token உருவாக்கும் போது அல்லது திருத்தும் போது, அந்த token-க்கு நீக்க அனுமதி இருப்பதை உறுதிசெய்யவும். இந்த ஸ்கிரிப்டுக்கு `delete` அனுமதி மட்டும் போதும்.

Token-ஐ சூழல் மாறியிலும் வைக்கலாம்:

```powershell
$env:IMGBED_API_TOKEN="your API Token"
```

## நீக்க API நடத்தை

நீக்க ஸ்கிரிப்ட் பின்வரும் பின்தள நீக்க API-யை அழைக்கும்:

```text
POST /api/manage/delete/batch
```

கோரிக்கையில் API Token இருக்க வேண்டும்:

```text
Authorization: Bearer <token>
```

கோரிக்கை உடல் எடுத்துக்காட்டு:

```json
{
  "fileIds": ["photos/2026/a.txt"],
  "deleteStrictness": "strict"
}
```

`fileIds`-இல் ஒரு கோப்பு இருந்தால், அது ஒற்றை கோப்பு நீக்கம். பல கோப்புகள் இருந்தால், அது தொகுதி நீக்கம். பின்தளம் ஒரு கோரிக்கையில் அதிகபட்சம் 15 கோப்புகளை செயல்படுத்தும்; ஸ்கிரிப்ட் `--batch-size` அடிப்படையில் பணியை தானாக பல கோரிக்கைகளாகப் பிரிக்கும்.

API NDJSON முன்னேற்ற ஓட்டத்தைத் திருப்பி வழங்கும். பொதுவான நிகழ்வுகள்: `batch_start`, `file_step`, `file_done`, `batch_complete`, மற்றும் `batch_error`. ஸ்கிரிப்ட் இந்நிகழ்வுகளைப் பகுத்து வாசிக்க எளிய வெளியீடாகவோ JSON வெளியீடாகவோ சுருக்கும்.

வெற்றிகரமான நீக்கத்திற்குப் பிறகு, பின்தளம் கோப்பு குறியீடுகள், அடைவு புள்ளிவிவரங்கள், கொள்ளளவு புள்ளிவிவரங்கள் மற்றும் கேச் சுத்தம் ஆகியவற்றை தானாக கையாளும்.

## நீக்க ஸ்கிரிப்ட் அளவுருக்கள்

| அளவுரு | கட்டாயம் | விளக்கம் |
| --- | --- | --- |
| `--base-url <url>` | ஆம் | ImgBed தள URL, உதாரணமாக `https://image.ai6.me`. |
| `--token <token>` | ஆம் | API Token. `IMGBED_API_TOKEN` சூழல் மாறியையும் பயன்படுத்தலாம். |
| `--file-id <id>` | ஆம் | நீக்க வேண்டிய கோப்பு ID. இதை பல முறை வழங்கலாம். |
| `--strictness <strict\|soft>` | இல்லை | நீக்கத்தின் கடுமை. இயல்புநிலை `strict`. |
| `--batch-size <n>` | இல்லை | ஒரு கோரிக்கைக்கு கோப்புகளின் எண்ணிக்கை. இயல்புநிலை `15`, அதிகபட்சம் `15`. |
| `--retries <n>` | இல்லை | தற்காலிக தோல்விகளுக்கான மீண்டும் முயற்சி எண்ணிக்கை. இயல்புநிலை `3`. |
| `--timeout-ms <n>` | இல்லை | ஒவ்வொரு கோரிக்கைக்கும் நேரவரம்பு. இயல்புநிலை `180000`. |
| `--output <pretty\|json>` | இல்லை | வெளியீட்டு வடிவம். இயல்புநிலை `pretty`. |
| `--save-response <path>` | இல்லை | இறுதி முடிவை JSON கோப்பாகச் சேமிக்கும். |
| `-h` / `--help` | இல்லை | ஸ்கிரிப்ட் உதவியை காட்டும். |

இந்த ஸ்கிரிப்ட் நீங்கள் வழங்கும் தெளிவான `--file-id` மதிப்புகளை மட்டுமே நீக்கும். இது மங்கலான பொருத்தம் செய்யாது, ஒரு அடைவை முழுவதுமாக காலி செய்யாது, அல்லது கமா பிரிக்கப்பட்ட பட்டியல்களிலிருந்தோ உள்ளூர் கோப்புகளிலிருந்தோ நீக்க ID-களைப் படிக்காது.

## கடுமையான நீக்கமும் மென்மையான நீக்கமும்

| முறை | விளக்கம் |
| --- | --- |
| `strict` | இயல்புநிலை முறை. தொலை சேமிப்பகத்தில் நீக்கம் தோல்வியடைந்தால், மீண்டும் முயற்சிக்க அல்லது காரணத்தை விசாரிக்க ImgBed பதிவு வைத்திருக்கும். |
| `soft` | தொலை சேமிப்பகத்தில் நீக்கம் தோல்வியடைந்தாலும், ImgBed பதிவு சுத்தம் செய்யப்படும்; முடிவில் எச்சரிக்கை திரும்பும். |

கட்டளை வெற்றியாக கருதப்பட வேண்டும் என்றால் தொலை கோப்பு உண்மையில் நீக்கப்பட வேண்டும் எனில், இயல்புநிலை `strict` முறையைப் பயன்படுத்தவும். தொலை தளம் அந்த பொருளை இனி நீக்க முடியாதபோது, ImgBed பதிவை மட்டும் சுத்தம் செய்ய விரும்பினால் `soft` பயன்படுத்தவும்.

## எடுத்துக்காட்டுகள்

ஒரு கோப்பை நீக்குதல்:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --file-id "photos/2026/a.txt"
```

சூழல் மாறியிலிருந்து Token பயன்படுத்துதல்:

```powershell
$env:IMGBED_API_TOKEN="your API Token"

node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt"
```

பல கோப்புகளை நீக்குதல்:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --file-id "photos/2026/b.txt" `
  --file-id "photos/2026/c.txt"
```

தொலை நீக்கம் தோல்வியடைந்தாலும் ImgBed பதிவை சுத்தம் செய்தல்:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --strictness soft
```

JSON வெளியீட்டையும் முடிவைச் சேமித்தலையும் பயன்படுத்துதல்:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --output json `
  --save-response ".\delete-result.json"
```

ஒவ்வொரு கோரிக்கையையும் 5 கோப்புகளுக்கு மட்டுப்படுத்துதல்:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --file-id "photos/2026/b.txt" `
  --batch-size 5
```

## நீக்கும் முன் `fileId` சரிபார்க்கவும்

நீக்க ஸ்கிரிப்டுக்கு ImgBed கோப்பு ID-கள் தேவை. முதலில் பட்டியல் ஸ்கிரிப்டைப் பயன்படுத்தி ஒரு அடைவில் உள்ள கோப்புகளைப் பார்க்கலாம்:

```powershell
node imgbed-token-list.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --files `
  --dir "photos/2026" `
  --count 10 `
  --output json
```

திரும்பும் முடிவில் உள்ள `name` புலம் பொதுவாக நீக்க ஸ்கிரிப்டுக்கு வழங்கக்கூடிய `fileId` ஆகும்.

## FAQ

### நீக்கம் தோல்வியடைந்தது, ஆனால் கோப்பு இன்னும் பட்டியலில் இருப்பது ஏன்?

இயல்புநிலை `strict` முறையில், தொலை சேமிப்பக நீக்கம் தோல்வியடைந்தால் ImgBed பதிவு வைத்திருக்கப்படும். தொலை கோப்பு இன்னும் இருக்கும் போது உள்ளூர் குறியீடு மட்டும் நீக்கப்படுவதை இது தவிர்க்கும். ImgBed பதிவை மட்டும் சுத்தம் செய்ய வேண்டும் என்று உறுதிசெய்த பிறகு, அதே `fileId`-ஐ `soft` உடன் மீண்டும் முயற்சிக்கவும்.

### முடிவில் எச்சரிக்கைகள் ஏன் உள்ளன?

எச்சரிக்கைகள் பொதுவாக தொலை நீக்கம், கேச் சுத்தம் அல்லது புள்ளிவிவர இறுதிப்படுத்தல் ஆகியவற்றின் போது ஏற்பட்ட உயிர்க்கட்டற்ற சிக்கலைக் குறிக்கும். மீண்டும் முயற்சி செய்ய வேண்டுமா என்பதை தீர்மானிக்க எளிதாக ஸ்கிரிப்ட் எச்சரிக்கைகளைச் சுருக்கும்.

### முழு அடைவையும் ஒரே முறையில் நீக்க முடியுமா?

இந்த ஸ்கிரிப்ட் அடைவை காலி செய்யும் செயல்பாட்டை வழங்காது. முதலில் பட்டியல் ஸ்கிரிப்டைப் பயன்படுத்தி தெளிவான `fileId` மதிப்புகளைத் தேர்ந்தெடுத்து, பின்னர் நீக்க வேண்டிய கோப்புகளை ஒன்றன்பின் ஒன்றாக வழங்கவும்.
