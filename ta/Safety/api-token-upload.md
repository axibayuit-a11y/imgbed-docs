# API Token மூலம் கோப்பு பதிவேற்றம்

API Token பதிவேற்றங்கள் ஸ்கிரிப்ட்கள், தானியக்கப் பணிகள் மற்றும் மூன்றாம் தரப்பு நிரல்களுக்காக வடிவமைக்கப்பட்டவை. வலை UI திறக்க தேவையில்லை. தள URL, token, உள்ளூர் கோப்பு பாதை மற்றும் உண்மையான பதிவேற்ற சேனலை வழங்கினால், கோப்பை ImgBed-க்கு பதிவேற்றலாம்; பதிலில் கோப்பு URL இருக்கும்.

![API Token திருத்துதல்](../../image/Safety/apitoken/编辑上传权限api.png)

## தொடங்குவதற்கு முன்

நிர்வாகப் பலகையைத் திறந்து இங்கே செல்லவும்:

```text
System Settings -> Security Settings -> API Token
```

API Token உருவாக்கும் போது அல்லது திருத்தும் போது, அதற்கு பதிவேற்ற அனுமதி இருப்பதையும் உண்மையான இயல்புநிலை பதிவேற்ற சேனல் பயன்படுத்தப்படுவதையும் உறுதிசெய்யவும். API Token பதிவேற்றங்கள் நுண்ணறிவு அனுப்புதல் நுழைவைப் பயன்படுத்தாது; ஸ்கிரிப்ட்களும் உண்மையான சேனலை வழங்க வேண்டும்.

## பதிவேற்ற ஸ்கிரிப்ட்களைப் பதிவிறக்குதல்

ஆவணத் தொகுப்பு இரண்டு Node.js ஸ்கிரிப்ட்களை வழங்குகிறது:

| ஸ்கிரிப்ட் | நோக்கம் |
| --- | --- |
| <a href="/tools/imgbed-token-single-upload.mjs" download>ஒற்றை கோரிக்கை பதிவேற்ற ஸ்கிரிப்டைப் பதிவிறக்கவும்</a> | `/upload`-ஐ ஒருமுறை அழைக்கும். சிறிய கோப்புகள் மற்றும் இணைப்பு சோதனைகளுக்கு பயனுள்ளது. |
| <a href="/tools/imgbed-token-chunk-upload.mjs" download>துண்டாக்கிய பதிவேற்ற ஸ்கிரிப்டைப் பதிவிறக்கவும்</a> | API Token துண்டாக்கம், நேரடி பதிவேற்றம் அல்லது தள பதிவேற்ற அமர்வுகளைப் பயன்படுத்தும். பெரிய கோப்புகளுக்கு பரிந்துரைக்கப்படுகிறது. |

Node.js 18 அல்லது அதற்கு மேல் தேவை.

## கிடைக்கும் சேனல்களைப் பட்டியலிடுதல்

இரு ஸ்கிரிப்ட்களும் தற்போதைய API Token-க்கு கிடைக்கும் பதிவேற்ற சேனல்களைப் பட்டியலிட முடியும்:

```powershell
node imgbed-token-single-upload.mjs --base-url "https://your-domain" --token "your API Token" --list-channels
node imgbed-token-chunk-upload.mjs --base-url "https://your-domain" --token "your API Token" --list-channels
```

சேனல்களைப் பட்டியலிடும்போது `--file` மற்றும் `--channel` தேவையில்லை. பதிலில் இயல்புநிலை பதிவேற்ற சேனல், பதிவேற்ற சேனல் key-கள், துணை சேனல் பெயர்கள் மற்றும் சுமை சமநிலை நிலை அடங்கும். ரகசியங்கள், புதுப்பிப்பு token-கள் மற்றும் பிற உணர்வுப்பூர்வ அமைப்பு மதிப்புகள் திருப்பி வழங்கப்படாது.

## பதிவேற்ற முறையைத் தேர்வு செய்தல்

| முறை | சிறந்த பயன்பாடு | விளக்கம் |
| --- | --- | --- |
| ஒற்றை கோரிக்கை பதிவேற்றம் | சிறிய கோப்புகள், எளிய ஸ்கிரிப்ட்கள், இணைப்பு சோதனைகள் | முழு கோப்பையும் `/upload`-க்கு ஒரே கோரிக்கையில் அனுப்பும். |
| துண்டாக்கிய பதிவேற்றம் | பெரிய கோப்புகள் அல்லது நேரவரம்பை மீறக்கூடிய கோப்புகள் | ஸ்கிரிப்ட் சேனலுக்கேற்ப துண்டாக்கிய, நேரடி அல்லது பதிவேற்ற அமர்வு ஓட்டத்தைத் தேர்வு செய்யும். |

பெரிய கோப்புகளுக்கு முதலில் துண்டாக்கிய பதிவேற்ற ஸ்கிரிப்டைப் பயன்படுத்தவும். ஒற்றை கோரிக்கை பதிவேற்றங்கள் Cloudflare கோரிக்கை அளவு, Worker நினைவகம் மற்றும் ஒவ்வொரு தளத்தின் சொந்த வரம்புகளால் கட்டுப்படுத்தப்படும்.

## ஒற்றை கோரிக்கை பதிவேற்றம்

ஒற்றை கோரிக்கை ஸ்கிரிப்ட் `/upload`-க்கு ஒரு கோரிக்கையை அனுப்பும்.

```powershell
node imgbed-token-single-upload.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --file "D:\test\image.png" `
  --channel s3 `
  --folder "photos/2026"
```

Token-ஐ சூழல் மாறியிலும் வைக்கலாம்:

```powershell
$env:IMGBED_API_TOKEN="your API Token"
node imgbed-token-single-upload.mjs --base-url "https://your-domain" --file "D:\test\image.png" --channel s3
```

### ஒற்றை கோரிக்கை அளவுருக்கள்

| அளவுரு | கட்டாயம் | விளக்கம் |
| --- | --- | --- |
| `--base-url <url>` | ஆம் | ImgBed தள URL, உதாரணமாக `https://image.ai6.me`. |
| `--token <token>` | ஆம் | API Token. `IMGBED_API_TOKEN` சூழல் மாறியையும் பயன்படுத்தலாம். |
| `--file <path>` | ஆம் | உள்ளூர் கோப்பு பாதை. |
| `--channel <key>` | ஆம் | பதிவேற்ற சேனல். |
| `--folder <path>` | இல்லை | பதிவேற்ற கோப்புறை, உதாரணமாக `photos/2026` அல்லது `/user/`. |
| `--name-type <type>` | இல்லை | பெயரிடும் முறை; பின்தள `uploadNameType`-க்கு பொருத்தப்படும். இயல்புநிலை `default`. |
| `--channel-name <name>` | இல்லை | துணை சேனல் அல்லது கணக்கைத் தேர்வு செய்யும். விடப்பட்டால், பின்தள சேனல் அமைப்பு முடிவு செய்யும். |
| `--retries <n>` | இல்லை | தற்காலிக தோல்விகளுக்கான மீண்டும் முயற்சி எண்ணிக்கை. இயல்புநிலை `3`. |
| `--timeout-ms <n>` | இல்லை | கோரிக்கை நேரவரம்பு. இயல்புநிலை `180000`. |
| `--output <pretty\|json>` | இல்லை | வெளியீட்டு வடிவம். இயல்புநிலை `pretty`. |
| `--save-response <path>` | இல்லை | இறுதி JSON பதிலை கோப்பாகச் சேமிக்கும். |
| `--list-channels` | இல்லை | தற்போதைய token-க்கு கிடைக்கும் சேனல்களைப் பட்டியலிட்டு வெளியேறும். |

### ஒற்றை கோரிக்கை அளவு வரம்புகள்

சாத்தியமானவரை ஒற்றை கோரிக்கை கோப்புகளை 100 MB-க்கு குறைவாக வைத்திருக்கவும்.

இந்த சேனல்களுக்கு ஒற்றை `/upload`-ஐ தடுக்கும் தெளிவான வரம்புகள் ஸ்கிரிப்டில் உள்ளன:

| சேனல் | ஒற்றை கோரிக்கை வரம்பு |
| --- | ---: |
| Telegram | 20 MiB |
| Discord | 10 MiB |
| S3 | 64 MiB |
| WebDAV | 64 MiB |
| GitHub Releases | 64 MiB |
| GitLab Packages | 64 MiB |

கோப்பு இவ்வரம்புகளில் ஒன்றை மீறினால், ஸ்கிரிப்ட் பொருந்தும் பிழையை உள்ளூரிலேயே தெரிவிக்கும். பிற சேனல்களுக்கு ஸ்கிரிப்டில் கடினமாக நிர்ணயிக்கப்பட்ட 100 MB உள்ளூர் சோதனை இல்லை. கோரிக்கை உடல் Cloudflare அல்லது தளத்தின் திறனை மீறினால், பிழையை Cloudflare அல்லது தொலை தளம் திருப்பி வழங்கும்.

## துண்டாக்கிய பதிவேற்றம்

துண்டாக்கிய பதிவேற்ற ஸ்கிரிப்ட் முதலில் இலக்கு கோப்பை தீர்மானிக்க பின்தளத்தைக் கேட்டு, பிறகு தேர்ந்தெடுக்கப்பட்ட சேனலுக்கான பெரிய கோப்பு ஓட்டத்தைப் பின்பற்றும். துண்டு அமர்வு, இணைப்பு அல்லது நிறைவு கோரிக்கைகளை நீங்கள் தனியாக எழுத தேவையில்லை.

```powershell
node imgbed-token-chunk-upload.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --file "D:\test\video.zip" `
  --channel github `
  --folder "photos/2026" `
  --concurrency 3
```

வெற்றிகரமான பதிவேற்றத்திற்குப் பிறகு, ஸ்கிரிப்ட் இதை அச்சிடும்:

```text
success
src: /file/photos/2026/example.png
url: https://your-domain/file/photos/2026/example.png
fileId: photos/2026/example.png
```

| புலம் | விளக்கம் |
| --- | --- |
| `src` | தளத்தின் உள் கோப்பு பாதை. |
| `url` | உங்கள் சொந்த ஸ்கிரிப்ட்கள் அல்லது தரவுத்தளப் பதிவுகளுக்கு ஏற்ற முழு பொதுப் URL. |
| `fileId` | பின்னர் கேள்விகள், மேலாண்மை அல்லது பதிவுகளுக்கு பயனுள்ள கோப்பு ID. |
| `channelName` | துண்டாக்கிய ஸ்கிரிப்ட் உண்மையில் பயன்படுத்திய துணை சேனல் அல்லது கணக்கைத் திருப்பி வழங்கலாம். |

`--output json` உடன், நிரல்முறை பயன்பாட்டுக்காக ஸ்கிரிப்ட் முழு JSON பதிலை அச்சிடும்.

## நேரடி ஒற்றை கோரிக்கை API அழைப்பு

ஸ்கிரிப்டைப் பயன்படுத்தவில்லை என்றால், ஒற்றை கோரிக்கை பதிவேற்ற முனையத்தை நேரடியாக அழைக்கலாம்:

```text
POST https://your-domain/upload?uploadChannel=s3&uploadFolder=photos/2026&uploadNameType=default
Authorization: Bearer your API Token
Content-Type: multipart/form-data
```

படிவப் புலம்:

| புலம் | கட்டாயம் | விளக்கம் |
| --- | --- | --- |
| `file` | ஆம் | பதிவேற்ற வேண்டிய கோப்பு. |

கோரிக்கை அளவுருக்கள்:

| அளவுரு | கட்டாயம் | விளக்கம் |
| --- | --- | --- |
| `uploadChannel` | ஆம் | உண்மையான பதிவேற்ற சேனல். |
| `uploadFolder` | இல்லை | பதிவேற்ற கோப்புறை. |
| `uploadNameType` | இல்லை | பெயரிடும் முறை. |
| `channelName` | இல்லை | துணை சேனல் அல்லது கணக்கைத் தேர்வு செய்யும். |

வெற்றிகரமான பதில்கள் இதுபோல் இருக்கும்:

```json
{
  "success": true,
  "src": "/file/photos/2026/example.png",
  "url": "https://your-domain/file/photos/2026/example.png",
  "fileId": "photos/2026/example.png"
}
```

## FAQ

### பெரிய ஒற்றை கோரிக்கை பதிவேற்றங்கள் தோல்வியடைகின்றன

ஒற்றை கோரிக்கை `/upload` முழு கோப்பையும் ஒரே கோரிக்கையில் அனுப்பும். பெரிய கோப்புகள் Cloudflare அல்லது தொலை தளத்தால் தடுக்கப்படலாம். பெரிய கோப்புகளுக்கு துண்டாக்கிய பதிவேற்ற ஸ்கிரிப்டைப் பயன்படுத்தவும்.

### `--channel-name` அமைக்கப்பட்டுள்ளது, ஆனால் பதிவேற்றம் இன்னும் தோல்வியடைகிறது

தேர்ந்தெடுக்கப்பட்ட சேனலில் அந்த பெயருடைய துணை சேனல் உண்மையில் உள்ளதா, அது இயக்கப்பட்டுள்ளதா என்பதை சரிபார்க்கவும். `--channel-name` விடப்பட்டால், அந்த சேனலின் அமைப்பைப் பொறுத்து பின்தளம் கிடைக்கக்கூடிய கணக்கைத் தேர்வு செய்யும்.

### முடிவை வேறு நிரலில் பயன்படுத்த விரும்புகிறேன்

`--output json` பயன்படுத்தவும், அல்லது `--save-response result.json` சேர்க்கவும். முழு கோப்பு URL பெற `url` புலத்தைப் படிக்கவும்.

### Yandex காப்பகங்களை பதிவேற்ற முடியவில்லை

Yandex காப்பக வடிவங்களை ஆதரிக்காது. இது அவர்களின் தளக் கொள்கையால் இருக்கலாம். Yandex பயன்படுத்தும் போது சாத்தியமானவரை காப்பகமற்ற கோப்புகளைப் பதிவேற்றவும்.
