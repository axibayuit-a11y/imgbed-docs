# S3-Kanal hinzufügen

## Geeignet für

Verwenden Sie den S3-Kanal, wenn:

- Sie einen S3-kompatiblen Objektspeicher anbinden möchten, etwa Cloudflare R2, Backblaze B2, MinIO, AWS S3 oder einen anderen kompatiblen Anbieter.
- Sie das klassische S3-Konfigurationsmodell mit `Endpoint`, Access Key und Secret Key bevorzugst.
- Sie nicht den nativen Cloudflare-R2-Binding-Kanal verwenden möchten oder Ihr Anbieter nicht Cloudflare R2 ist.

## Was Sie vorher benötigen

| Voraussetzung | Zweck |
| --- | --- |
| S3 Endpoint | API-Endpunkt des Objektspeichers. |
| Bucket-Name | Bucket, in dem Dateien gespeichert werden. |
| Region | Speicherregion. Bei Cloudflare R2 meistens `auto`. |
| Access Key ID | S3-Zugriffsschlüssel-ID. |
| Secret Access Key | Geheimer S3-Zugriffsschlüssel. |
| CDN-Domain | Optionaler eigener Zugriffsdomainname. |

Beispiel für Cloudflare R2 über die S3-API:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## Wo hinzufügen

1. Öffnen Sie die Systemeinstellungen.
2. Gehen Sie zu Upload-Einstellungen.
3. Klicken Sie oben rechts auf Kanal hinzufügen.
4. Wählen Sie `S3`.

## Feldübersicht

| Feld | Funktion | Erforderlich |
| --- | --- | --- |
| Kanalname | Ein gut erkennbarer Name für diesen S3-Kanal, z. B. `s3test` oder `R2-S3`. | Ja |
| Kanal aktivieren | Legt fest, ob dieser Kanal bei der Upload-Auswahl verwendet wird. | Empfohlen |
| Endpoint | Vollständiger S3-Service-Endpunkt inklusive `https://`. | Ja |
| Eigene CDN-Domain | Optional. Wenn gesetzt, bevorzugen generierte Dateilinks diese Domain. | Nein |
| Bucket-Name | Bucket-Name, z. B. `s3test` oder `img-r2`. | Ja |
| Bucket-Region | Region. Bei Cloudflare R2 meistens `auto`. | Ja |
| Access Key ID | S3-Zugriffsschlüssel-ID. | Ja |
| Secret Access Key | Geheimer S3-Zugriffsschlüssel. | Ja |
| Path-style access | Kompatibilitätsschalter. Standardmäßig aus. Manche MinIO-, OpenList- oder selbst gehostete S3-Dienste benötigen ihn. | Nein |
| Quotenlimit | Steuert, ob dieser S3-Kanal abhängig von der Kapazität bei Uploads berücksichtigt wird. | Nein |
| Kapazitätslimit | Nach Aktivierung des Quotenlimits erforderlich, z. B. `10 GB`. | Erforderlich bei aktivem Quotenlimit |
| Schwellenwert | Stoppt Schreibvorgänge nach Erreichen dieses Prozentsatzes, z. B. `90%`. | Erforderlich bei aktivem Quotenlimit |
| Bemerkung | Notizen für Ihre eigene Wartung. | Nein |

## Cloudflare-R2-S3-API-Schlüssel erstellen

1. Öffnen Sie das Cloudflare Dashboard.
2. Gehen Sie zu `R2 Object Storage`.
3. Suchen Sie `Manage R2 API Tokens` oder den Eintrag zur API-Token-Verwaltung.

![R2-API-Token-Einstieg](../../image/upload/s3/r2%20api%20token入口.png)

4. Erstellen Sie ein R2-API-Token mit Zugriff auf den Ziel-Bucket.

![Account-API-Token erstellen](../../image/upload/s3/创建账户api令牌.png)

5. Kopieren Sie die erzeugten Werte `Access Key ID` und `Secret Access Key`.

![Token-Werte kopieren](../../image/upload/s3/复制令牌值.png)

6. Kehren Sie zur R2-Bucket-Seite zurück und prüfen Sie den Bucket-Namen.
7. Notiere den S3-API-Endpunkt des Kontos. Er sieht normalerweise so aus:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## Einrichtungsschritte

1. Öffnen Sie die Upload-Einstellungen.
2. Klicken Sie auf Kanal hinzufügen.
3. Wählen Sie `S3`.
4. Tragen Sie einen gut erkennbaren Kanalnamen ein, z. B. `s3test`.
5. Tragen Sie den S3-API-Endpunkt bei `Endpoint` ein.
6. Wenn Sie ein eigenes CDN nutzen, tragen Sie es bei `Custom CDN domain` ein; sonst leer lassen.
7. Tragen Sie den Bucket-Namen ein.
8. Tragen Sie die Region ein. Für Cloudflare R2 im Beispiel ist das `auto`.
9. Tragen Sie Access Key ID und Secret Access Key ein.
10. Lassen Sie Path-style access ausgeschaltet, sofern Ihr Anbieter ihn nicht ausdrücklich verlangt.
11. Wenn Sie Kapazitätssteuerung möchten, aktivieren Sie das Quotenlimit und tragen Sie Kapazitätslimit sowie Schwellenwert ein.
12. Klicken Sie auf Speichern.

![S3-Konfiguration hinzufügen](../../image/upload/s3/添加s3配置.png)

## Prüfung

| Prüfung | So prüfen Sie es |
| --- | --- |
| Kanalkarte erscheint | Nach dem Speichern sollte in den Upload-Einstellungen eine S3-Kanalkarte sichtbar sein. |
| Kanal ist aktiv | Der Schalter oben rechts auf der Karte bleibt eingeschaltet. |
| Wichtige Felder sind gespeichert | Die Detailansicht zeigt Endpoint, Bucket, Region, Path-style access und verwandte Felder. |
| Upload funktioniert | Laden Sie ein Testbild hoch und prüfen Sie, ob das Objekt im Ziel-Bucket erscheint. |
| Link öffnet sich | Der nach dem Upload zurückgegebene Bildlink lässt sich normal öffnen. |
| Kapazitätsanzeige funktioniert | Bei aktivem Quotenlimit zeigt die Karte belegten Speicher und konfiguriertes Limit. |

S3-Kapazitätsstatistiken werden aus den lokalen Dateidatensätzen von ImgBed berechnet, nicht in Echtzeit aus dem Anbieter-Bucket abgefragt. Nach dem Speichern einer S3-Konfiguration berechnet das System die Quotenübersicht anhand der aktuellen D1-Einträge neu.

## FAQ

### Soll ich Path-style access aktivieren?

In der Regel nein.

Aktivieren Sie ihn nur, wenn Ihr S3-kompatibler Anbieter keine Virtual-hosted-style-URLs unterstützt oder die Dokumentation des Anbieters Path-style access ausdrücklich verlangt. Manche MinIO-, selbst gehostete S3- und OpenList-kompatible Endpunkte benötigen das.

### Wofür ist die CDN-Domain?

Wenn Sie ein CDN oder eine eigene Zugriffsdomain vor den Bucket schaltest, tragen Sie sie hier ein.

Nach der Konfiguration bevorzugt ImgBed diese Domain für generierte Dateilinks. Wenn Sie kein CDN nutzen, lassen Sie das Feld leer.

### Warum schlägt der Upload fehl?

Prüfen Sie zuerst diese Punkte:

1. Der Endpoint enthält die vollständige `https://`.
2. Der Bucket-Name stimmt.
3. Die Region passt zu den Vorgaben des Anbieters.
4. Access Key ID und Secret Access Key wurden vollständig kopiert.
5. Der Schlüssel hat Schreibrechte für den Ziel-Bucket.
6. Der Anbieter benötigt keinen Path-style access, oder er wurde aktiviert, falls nötig.

## Kurzcheck

```text
Prepare S3 Endpoint, Bucket, Region, Access Key, and Secret Key
-> Open Upload Settings
-> Add Channel
-> Select S3
-> Enter Endpoint / Bucket / Region / Access Key / Secret Key
-> Enter a CDN domain if needed
-> Leave path-style access off by default
-> Enable quota limit if needed
-> Save
-> Upload a test image and check the result
```
