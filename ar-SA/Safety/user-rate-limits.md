# حدود معدل الرفع للمستخدمين

تتحكم user rate limits في عدد مرات رفع الملفات من الصفحة الرئيسية بواسطة المستخدمين العاديين أو الزوار. يساعد ذلك على منع إساءة استخدام صفحات الرفع العامة.

تؤثر هذه الميزة على رفع الصفحة الرئيسية فقط. رفع المسؤول والرفع عبر API Tokens لا يخضعان لهذه الحدود.

## أين تضبطها؟

افتح لوحة الإدارة، ثم انتقل إلى:

```text
System Settings -> Security Settings -> Upload Management -> User Rate Limits
```

![إعدادات حدود معدل المستخدم](../../image/other/用户频控截图.png)

## تفعيل الحدود

بعد تفعيل `Enable Rate Limits` يتتبع ImgBed الرفعات الأخيرة حسب IP الرافع.

القيم الافتراضية:

| الإعداد | الافتراضي | الوصف |
| --- | --- | --- |
| Detection window | 1.5 hours | المدة السابقة التي تُحسب ضمنها سجلات الرفع. |
| Max file count | 20 | الحد الأقصى لعدد الملفات داخل نافذة الكشف. |
| Single file size limit | 20 MB | الحد الأقصى لحجم ملف واحد. |
| Total upload size limit | 200 MB | الحد الأقصى لإجمالي حجم الرفع داخل نافذة الكشف. |

مثلًا، مع نافذة 1.5 ساعة و20 ملفًا و20 MB لكل ملف و200 MB إجماليًا، تُحظر الرفعات من IP نفسه عند تجاوز أي حد مضبوط.

## استثناء أنواع الملفات

يقوم `Excluded upload file types` بمنع المستخدمين العاديين أو الزوار من رفع فئات ملفات محددة.

الفئات المتاحة:

| النوع | الوصف |
| --- | --- |
| Images | ملفات الصور مثل jpg و png و webp و gif وما شابه |
| Videos | ملفات الفيديو مثل mp4 و webm و mov وما شابه |
| Audio | ملفات الصوت مثل mp3 و flac و wav وما شابه |
| Documents | ملفات المستندات مثل pdf و txt و md و docx وما شابه |
| Other | ملفات خارج الفئات السابقة، مثل zip و rar و exe و apk |

افتراضيًا لا تكون الفئة محددة، وهذا يعني أنها مسموحة.

النقر على فئة يبرزها، وهذا يعني أن ذلك النوع محظور.

إذا تم تحديد `Other` فسيُحظر رفع ملفات zip أو rar وسيُبلغ الزائر أن هذا النوع غير مدعوم.

## رسائل الحظر

عند تفعيل حد معين يرى المستخدم رسالة مناسبة:

![رسالة الرفع المتكرر](../../image/other/频繁报错提示.png)

| الحالة | معنى الرسالة |
| --- | --- |
| ملف واحد كبير جدًا | الملف كبير جدًا ويجب ضغطه قبل الرفع. |
| نوع الملف محظور | هذا النوع غير مدعوم. أزله وحاول مرة أخرى. |
| الرفع متكرر جدًا | الرفعات الأخيرة كثيرة جدًا، ويظهر وقت إعادة المحاولة. |
| الحجم الإجمالي مرتفع جدًا | الحجم الإجمالي للرفعات الأخيرة مرتفع، ويظهر وقت إعادة المحاولة. |

## متى تفعّلها؟

فعّل user rate limits إذا كانت صفحة الرفع الرئيسية متاحة للعامة.

أسباب شائعة:

- القلق من الرفع الجماعي عبر سكربتات.
- الرغبة في الحد من رفع الملفات الكبيرة من الزوار.
- السماح للمستخدمين العاديين برفع الصور فقط، وليس الأرشيفات أو ملفات التثبيت.
- إبقاء الرفع العام متاحًا مع التحكم في استهلاك الموارد.

إذا كان الموقع لك وحدك، أو كان الرفع مقتصرًا على المسؤولين، يمكنك تركها معطلة.
