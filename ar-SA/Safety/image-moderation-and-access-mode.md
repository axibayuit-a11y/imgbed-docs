# مراجعة الصور ووضع الوصول

تسند مراجعة الصور تصنيفات عمرية إلى الصور المرفوعة. ويتحكم وضع الوصول في التصنيفات التي يمكن عرضها عبر الوصول العام.

يؤثر ذلك في المعرض العام، وروابط الملفات العامة، وواجهة API للصور العشوائية. ولا يقيّد لوحة الإدارة؛ إذ يستطيع المسؤولون دائمًا عرض جميع الملفات وإدارتها.

## مكان الإعداد

افتح لوحة الإدارة، ثم انتقل إلى:

```text
System Settings -> Security Settings -> Upload Management -> Image Moderation
```

الإعدادات الرئيسية هي:

- وضع الوصول
- تفعيل المراجعة
- مزود المراجعة

## وظيفة وضع الوصول

يحدد وضع الوصول التصنيفات العمرية التي يمكن عرضها للعامة.

الأوضاع الحالية:

| وضع الوصول | التصنيفات المرئية للعامة |
| --- | --- |
| وضع البالغين | عام، R12، R16، R18 |
| وضع الشباب | عام، R12، R16 |
| وضع المراهقين | عام، R12 |
| وضع الأطفال | عام فقط |

الوضع الافتراضي هو وضع البالغين.

قد يكون وضع البالغين مناسبًا للمواقع الخاصة أو المواقع التي تحتوي على محتوى موجه للبالغين. أما إذا أردت معرضًا عامًا أكثر تحفظًا، فاختر وضع الشباب أو المراهقين أو الأطفال.

## وظيفة تفعيل المراجعة

عند تفعيل المراجعة، يستدعي ImgBed مزود المراجعة المحدد أثناء الرفع ويحفظ التصنيف العمري الذي تم اكتشافه.

التصنيفات الرئيسية:

| التصنيف | المعنى |
| --- | --- |
| عام | محتوى عام آمن |
| R12 | محتوى حساس بدرجة خفيفة |
| R16 | محتوى حساس بدرجة متوسطة |
| R18 | محتوى للبالغين |

تُستخدم نتيجة المراجعة عند تحديد الوصول العام.

إذا لم تكن المراجعة مفعّلة، أو لم تكن لدى الملفات القديمة تصنيفات، تُعامل هذه الملفات كملفات غير مصنفة. ولا تُزال الملفات غير المصنفة تلقائيًا من المعرض العام أو واجهة API للصور العشوائية لمجرد عدم وجود تصنيف.

## اختيار مزود المراجعة

تشمل المزودات المتاحة:

- moderatecontent.com
- nsfwjs
- Sightengine

لكل مزود متطلبات مختلفة:

- يحتاج moderatecontent.com عادةً إلى API Key.
- يحتاج nsfwjs عادةً إلى عنوان URL لنقطة API.
- يحتاج Sightengine إلى API user وAPI secret.

اختر المزود وفق حسابك، وتوفر الخدمة، وجودة الكشف. ما دامت المراجعة مفعّلة ومضبوطة بشكل صحيح، يحاول ImgBed كتابة تصنيف للصورة أثناء الرفع.

## التأثير في المعرض العام

يرشّح المعرض العام الملفات وفق وضع الوصول.

أمثلة:

- وضع البالغين: يمكن أن تظهر صور R18.
- وضع الشباب: تُخفى صور R18.
- وضع المراهقين: تُخفى صور R16 وR18.
- وضع الأطفال: تظهر الصور المصنفة عام فقط.

ينطبق ذلك على الوصول العام العادي فقط. وتظل لوحة الإدارة تعرض جميع الملفات.

## التأثير في روابط الملفات العامة

روابط الملفات العامة هي روابط الصور المباشرة التي يفتحها الزوار.

إذا كان تصنيف الملف مسموحًا في وضع الوصول الحالي، يعيد ImgBed الصورة الأصلية.

إذا كان التصنيف أعلى من المستوى المسموح، لا يعيد الوصول العام العادي الصورة الأصلية. وبدلًا من ذلك، يعيد ImgBed نتيجة الحجب المضبوطة أو صورة العنصر النائب.

مثال:

- الوضع الحالي هو وضع الأطفال.
- الصورة مصنفة R18.
- يفتح زائر الرابط العام مباشرة.
- لا يعيد ImgBed الصورة الأصلية المصنفة R18 إلى ذلك الزائر.

![صورة ملف مقيّد](../../image/Safety/文件受限图.png)

لا يتأثر المسؤولون بهذا القيد عند عرض الملفات من لوحة الإدارة.

## التأثير في واجهة API للصور العشوائية

ترشّح واجهة API للصور العشوائية مجموعة الملفات المرشحة وفق وضع الوصول أيضًا.

في وضع الأطفال، لا تُختار الصور العشوائية إلا من الملفات المصنفة عام.

في وضع الشباب، قد تأتي الصور العشوائية من ملفات مصنفة عام أو R12 أو R16، لكنها لا تأتي من ملفات R18.

يمنع ذلك واجهة API للصور العشوائية من تجاوز قيود المعرض العام.

## العلاقة مع قواعد القوائم

وضع الوصول ليس قاعدة الوصول العام الوحيدة. فهو يعمل مع قواعد قوائم السماح والحظر.

بعبارة مبسطة:

- يكون المحتوى الموجود في قائمة السماح عامًا أولًا.
- لا يمكن للزوار العاديين عرض المحتوى الموجود في قائمة الحظر مباشرة.
- يُفحص المحتوى غير الموجود في أي من القائمتين وفق وضع الوصول بعد ذلك.

إذا كانت الصورة مقيّدة في الوقت نفسه بسبب التصنيف العمري وقواعد القوائم، فلن يستطيع الزوار العاديون عرض الملف الأصلي مباشرة.

## الإعدادات الموصى بها

للمواقع العامة:

- فعّل المراجعة.
- اختر وضع الوصول المناسب لجمهور الموقع.
- استخدم وضع الأطفال أو وضع المراهقين للزوار من جميع الأعمار.
- تجنب وضع البالغين إذا كنت لا تريد عرض محتوى موجه للبالغين للعامة.
- راجع تصنيفات الملفات في لوحة الإدارة وعدّلها يدويًا عند الحاجة.

للمواقع الخاصة أو الشخصية:

- يكون وضع البالغين مناسبًا عادةً.
- فعّل المراجعة إذا كانت مفيدة لك.
- راجع التصنيفات وعدّلها في لوحة الإدارة حسب الحاجة.

## الأسئلة الشائعة

### هل تختفي الملفات من لوحة الإدارة بعد تغيير وضع الوصول؟

لا.

يؤثر وضع الوصول في الوصول العام العادي فقط، ولا يؤثر في لوحة الإدارة.

### لماذا يعرض المعرض العام صورًا أقل بعد التبديل إلى وضع الأطفال؟

يسمح وضع الأطفال بعرض الملفات المصنفة عام فقط. لذلك تُستبعد ملفات R12 وR16 وR18.

### هل تستطيع الروابط العامة فتح صور للبالغين؟

إذا كان وضع الوصول الحالي لا يسمح بذلك التصنيف، فلن تعيد الروابط العامة العادية الصورة الأصلية.

### هل يمكن أن تعيد واجهة API للصور العشوائية صورًا مقيّدة؟

لا.

ترشّح واجهة API للصور العشوائية المرشحين وفق وضع الوصول الحالي.

### ماذا يحدث للصور القديمة غير المصنفة؟

لا تُخفى الصور غير المصنفة تلقائيًا لمجرد عدم وجود نتائج مراجعة لها. يمكنك تعديل تصنيفاتها لاحقًا في لوحة الإدارة.
