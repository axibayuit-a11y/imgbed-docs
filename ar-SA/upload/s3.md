# إضافة قناة S3

## متى تكون مناسبة؟

استخدم قناة S3 عندما:

- تريد ربط أي خدمة تخزين كائنات متوافقة مع S3، مثل Cloudflare R2 أو Backblaze B2 أو MinIO أو AWS S3 أو مزود متوافق آخر.
- تفضل نموذج إعداد S3 المعتاد: `Endpoint` و Access Key و Secret Key.
- لا تريد استخدام قناة ربط Cloudflare R2 الأصلية، أو كان مزود التخزين ليس Cloudflare R2.

## ما الذي تحتاجه قبل البدء؟

| المتطلب | الغرض |
| --- | --- |
| S3 Endpoint | نقطة نهاية S3 API لخدمة تخزين الكائنات. |
| اسم الحاوية | الحاوية التي ستُحفظ فيها الملفات. |
| Region | منطقة التخزين. في Cloudflare R2 تكون غالبًا `auto`. |
| Access Key ID | معرّف مفتاح وصول S3. |
| Secret Access Key | مفتاح وصول S3 السري. |
| نطاق CDN | نطاق مخصص اختياري للوصول إلى الملفات. |

مثال لاستخدام Cloudflare R2 عبر S3 API:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## أين تضيفها؟

1. افتح إعدادات النظام.
2. انتقل إلى إعدادات الرفع.
3. اضغط إضافة قناة في الزاوية العلوية اليمنى.
4. اختر `S3`.

## إنشاء مفاتيح Cloudflare R2 S3 API

1. افتح لوحة معلومات Cloudflare.
2. انتقل إلى `R2 Object Storage`.
3. ابحث عن `Manage R2 API Tokens` أو مدخل إدارة رموز API.

![مدخل رمز R2 API](../../image/upload/s3/r2%20api%20token入口.png)

4. أنشئ رمز R2 API يستطيع الوصول إلى الحاوية الهدف.

![إنشاء رمز API للحساب](../../image/upload/s3/创建账户api令牌.png)

5. انسخ قيم `Access Key ID` و `Secret Access Key`.

![نسخ قيم الرمز](../../image/upload/s3/复制令牌值.png)

6. ارجع إلى صفحة حاوية R2 وتأكد من اسم الحاوية.
7. سجّل نقطة نهاية S3 API الخاصة بالحساب. غالبًا تكون بالشكل:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## خطوات الإعداد

1. افتح إعدادات الرفع.
2. اضغط إضافة قناة.
3. اختر `S3`.
4. أدخل اسم قناة واضحًا، مثل `s3test`.
5. أدخل نقطة نهاية S3 API في `Endpoint`.
6. إذا كنت تستخدم CDN مخصصًا فأدخله في `Custom CDN domain`، وإلا اتركه فارغًا.
7. أدخل اسم الحاوية.
8. أدخل المنطقة. في مثال Cloudflare R2 استخدم `auto`.
9. أدخل Access Key ID و Secret Access Key.
10. اترك الوصول بنمط المسار مغلقًا ما لم يطلبه المزود صراحة.
11. إذا أردت التحكم في السعة، فعّل حد الحصة وأدخل حد السعة والعتبة.
12. اضغط حفظ.

![إضافة إعداد S3](../../image/upload/s3/添加s3配置.png)

## كيف تتحقق؟

| الفحص | طريقة التحقق |
| --- | --- |
| ظهور بطاقة القناة | بعد الحفظ يجب أن تظهر بطاقة قناة S3 في إعدادات الرفع. |
| القناة مفعلة | يجب أن يبقى المفتاح في أعلى يمين البطاقة مفعّلًا. |
| الحقول الأساسية محفوظة | يجب أن تظهر في التفاصيل قيم Endpoint و Bucket و Region والوصول بنمط المسار والحقول المرتبطة. |
| نجاح الرفع | ارفع صورة تجريبية وتأكد من ظهور الكائن في الحاوية الهدف. |
| فتح الرابط | يجب أن يفتح رابط الصورة الناتج بعد الرفع بشكل طبيعي. |
| عرض السعة يعمل | إذا كان حد الحصة مفعّلًا، يجب أن تعرض البطاقة السعة المستخدمة والحد المضبوط. |

إحصاءات سعة S3 تُحسب من سجلات الملفات المحلية في ImgBed، وليس من استعلام مباشر للحاوية لدى المزود في الوقت الحقيقي. بعد حفظ إعداد S3، يعيد النظام حساب دفتر الحصة من سجلات D1 الحالية.

## FAQ

### هل أفعّل الوصول بنمط المسار؟

غالبًا لا.

فعّله فقط إذا كان مزود S3 المتوافق لا يدعم عناوين URL بنمط الاستضافة الافتراضية، أو إذا طلبت وثائق المزود الوصول بنمط المسار صراحة. قد تحتاجه بعض نقاط MinIO أو S3 ذاتية الاستضافة أو OpenList.

### ما فائدة نطاق CDN؟

إذا وضعت CDN أو نطاق وصول مخصصًا أمام الحاوية، أدخله هنا.

بعد ضبطه ستفضل روابط الملفات الناتجة هذا النطاق. إذا لم تستخدم CDN، اتركه فارغًا.

### لماذا يفشل الرفع؟

تحقق أولًا من التالي:

1. يحتوي Endpoint على عنوان `https://` كامل.
2. اسم الحاوية صحيح.
3. Region يطابق متطلبات المزود.
4. تم نسخ Access Key ID و Secret Access Key بالكامل.
5. المفتاح يملك صلاحية الكتابة على الـ bucket الهدف.
6. المزود لا يتطلب الوصول بنمط المسار، أو تم تفعيله إذا كان مطلوبًا.

