# إضافة قناة Cloudflare R2

## متى تكون مناسبة؟

استخدم Cloudflare R2 عندما:

- يكون موقع ImgBed منشورًا أصلًا على Cloudflare وتريد حفظ الملفات في حاوية R2 ضمن الحساب نفسه.
- لا ترغب في ضبط نقطة نهاية S3 ومفتاح الوصول والمفتاح السري بشكل منفصل.
- تريد أن تتم القراءة والكتابة عبر ربط R2 في Worker أو Pages بأقل إعداد ممكن.

باختصار:

لا تُنشأ قناة R2 يدويًا داخل لوحة إدارة ImgBed. يجب أولًا ربط حاوية R2 بمشروع Cloudflare، ويجب أن يكون اسم متغير الربط بالضبط `img_r2`.

## ما الذي تحتاجه قبل البدء؟

- حساب Cloudflare.
- حاوية R2 موجودة مسبقًا.
- صلاحية إدارة مشروع Cloudflare الذي يعمل عليه ImgBed.

## الإعداد في Cloudflare

### 1. أنشئ حاوية R2

1. سجّل الدخول إلى لوحة معلومات Cloudflare.
2. افتح `R2 Object Storage`.
3. اضغط إنشاء حاوية.
4. اختر اسمًا للحاوية، مثل `imgbed`.

سيتم حفظ الملفات المرفوعة داخل هذه الحاوية.

![إنشاء حاوية R2](../../image/upload/cloudflare-r2/创建一个存储桶img-r2.png)

### 2. اربط الحاوية بمشروع ImgBed

اختر مكان الربط حسب نوع النشر:

| نوع النشر | مكان الربط |
| --- | --- |
| Pages | مشروع Pages الحالي -> الإعدادات -> الدوال -> روابط حاويات R2 |
| Worker | Worker الحالي -> الإعدادات -> الروابط -> روابط حاويات R2 |

عند إضافة الربط، الحقول المهمة هي:

| الحقل | القيمة |
| --- | --- |
| اسم المتغير | `img_r2` |
| حاوية R2 | اختر الحاوية التي أنشأتها. |

يجب أن يكون اسم المتغير `img_r2` بالضبط. تعتمد عمليات الرفع والقراءة والحذف في R2 على اسم الربط هذا.

### 3. أعد نشر المشروع

بعد حفظ الربط، أعد نشر ImgBed حتى تستطيع بيئة تشغيل Worker أو Pages الوصول إلى `img_r2`.

## ماذا سترى في ImgBed؟

بعد توفر ربط R2، افتح:

1. إعدادات النظام.
2. إعدادات الرفع.
3. قناة `Cloudflare R2`.

ينشئ النظام قناة ثابتة واحدة تلقائيًا:

| الحقل | القيمة الثابتة |
| --- | --- |
| اسم القناة | `Cloudflare R2` |
| نوع القناة | `cfr2` |
| وضع التخزين | `binding` |
| مصدر الإعداد | ربط بيئة التشغيل |

هذه قناة ربط ثابتة. لا تحتاج إلى الضغط على إضافة قناة لإنشائها، ولا يمكن حذفها مثل القنوات العادية.

## الحقول القابلة للتعديل في لوحة الإدارة

| الحقل | وظيفته | مطلوب |
| --- | --- | --- |
| تفعيل القناة | يحدد هل تشارك R2 في اختيار قناة الرفع. | نعم |
| Account ID | يستخدم فقط عند تفعيل حدود الحصة والحاجة إلى قراءة الاستخدام الرسمي لـ R2. | موصى به عند تفعيل الحصص |
| اسم الحاوية | يستخدم فقط عند تفعيل حدود الحصة والحاجة إلى قراءة الاستخدام الرسمي لـ R2. | موصى به عند تفعيل الحصص |
| حد الحصة | يحدد هل تُؤخذ سعة هذه القناة في الاعتبار عند اختيار قناة الرفع. | لا |
| العتبة | يوقف الكتابة إلى هذه القناة بعد بلوغ نسبة الاستخدام المحددة. | مطلوب عند تفعيل الحصص |

يمكن نسخ Account ID من لوحة معلومات الحساب في Cloudflare. املأه فقط إذا أردت من ImgBed قراءة استخدام حصة R2 وتطبيقه.

![الحصول على Account ID](../../image/upload/cloudflare-r2/获取账户id.png)

## خطوات الإعداد

1. أنشئ حاوية R2 في Cloudflare.
2. افتح إعدادات Cloudflare الخاصة بمشروع ImgBed.
3. أضف ربطًا لحاوية R2.
4. اجعل اسم المتغير مساويًا لـ `img_r2`.
5. اختر حاوية R2 التي أنشأتها.
6. احفظ الربط وأعد نشر ImgBed.
7. ارجع إلى ImgBed -> إعدادات النظام -> إعدادات الرفع.
8. تأكد من ظهور قناة `Cloudflare R2` وأنها مفعلة.

إذا أردت أن تشارك R2 في اختيار قناة الرفع حسب السعة، فعّل حد الحصة ثم أدخل Account ID واسم الحاوية وحد الحصة والعتبة قبل الحفظ.

![ضبط حدود الحصة](../../image/upload/cloudflare-r2/配置容量限制.png)

## كيف تتحقق؟

- تظهر قناة `Cloudflare R2` الثابتة في إعدادات الرفع.
- تظهر بطاقة القناة على أنها مفعلة.
- ينجح رفع ملف صغير، ويفتح الرابط الناتج بشكل طبيعي.
- إذا أعاد فتح الملف رسالة `R2 database binding is not configured` فهذا يعني أن بيئة التشغيل لم تستلم الربط `img_r2`. تحقق من اسم الربط في Cloudflare وأعد نشر المشروع.
