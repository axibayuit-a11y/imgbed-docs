# النسخ الاحتياطي الزائد وتبديل مصدر القراءة

يحفظ Redundant backup نسخة إضافية من ملف تم رفعه مسبقًا.

يمكن استخدام الملف الأساسي وملف النسخة الاحتياطية كمصادر قراءة. لن يلاحظ الزائر عادة أي فرق؛ الاختلاف الوحيد هو قناة التخزين التي تخدم الملف.

## ماذا يستطيع Redundant Backup فعله؟

| الميزة | الوصف |
| --- | --- |
| حفظ نسخة إضافية | نسخ الملفات احتياطيًا إلى قناة رفع أخرى لتقليل خطر تعطل قناة واحدة. |
| تبديل read source | بعد نجاح النسخ الاحتياطي، يمكن التبديل بين القناة الأساسية وقناة النسخة الاحتياطية للقراءة. |
| Single-file backup | نسخ ملف واحد احتياطيًا من صفحة تفاصيله. |
| Batch backup | اختيار عدة ملفات في لوحة الإدارة ونسخها معًا. |
| Global redundant backup | نسخ الملفات حسب المجلد من Other Settings. |

## مدخل Redundant Backup

افتح:

```text
System Settings -> Other Settings -> Redundant Backup
```

![Redundant backup](../../image/other/冗余备份截图.png)

هذا المدخل مناسب لإضافة نسخ احتياطية لمجلد أو لكل الملفات دفعة واحدة.

يمكن اختيار backup channel يدويًا، أو اختيار automatic switching ليجد ImgBed قناة احتياطية مناسبة.

## النسخ من تفاصيل الملف

افتح صفحة تفاصيل ملف في لوحة الإدارة واضغط backup.

![Backup في تفاصيل الملف](../../image/other/文件详情里文件备份.png)

هذا مناسب لنسخ ملف مهم واحد عند الحاجة.

بعد نجاح النسخ تعرض صفحة تفاصيل الملف مصادر القراءة المتاحة.

## Batch Backup بالتحديد

في لوحة الإدارة، حدد عدة ملفات وشغّل batch backup.

![Batch backup](../../image/other/批量备份截图.png)

هذا مناسب لمعالجة مجموعة ملفات.

Selection backup و file details backup و redundant backup ضمن Other Settings تستخدم النظام نفسه، لكنها مداخل مختلفة.

## تبديل Read Source بعد النسخ

بعد اكتمال النسخ، تتيح صفحة تفاصيل الملف تبديل read source:

| Read Source | الوصف |
| --- | --- |
| Primary channel | القراءة من قناة الرفع الأصلية. |
| Backup channel | القراءة من قناة النسخة الاحتياطية. |

![تبديل read source بعد النسخ](../../image/other/备份成功切换读取源.png)

لا يحتاج الزوار إلى معرفة هل الملف يُخدم من القناة الأساسية أو الاحتياطية.

مصدر القراءة الذي تختاره يصبح المصدر المفضل للوصول اللاحق إلى الملف.

## متى يتم تجاوز النسخ؟

الحالات التالية تُتجاوز أثناء النسخ وليست أخطاء:

| الحالة | سبب التجاوز |
| --- | --- |
| Already backed up | الملف الذي لديه نسخة احتياطية لا يُنسخ مرة أخرى. |
| Primary and backup channels are the same | يجب أن تحفظ النسخة الاحتياطية في قناة أخرى حتى تكون مفيدة. |
| No usable backup channel | لا توجد قناة بديلة مناسبة. |

باختصار: يجب أن تذهب النسخ الاحتياطية إلى قناة أخرى، والملفات المنسوخة مسبقًا لا تستهلك مساحة إضافية مرة أخرى.

## Primary Channel و Backup Channel

| الاسم | المعنى |
| --- | --- |
| Primary channel | القناة المستخدمة عند رفع الملف لأول مرة. |
| Backup channel | القناة التي تحفظ النسخة الزائدة. |
| Primary read source | الملف يُقرأ حاليًا من القناة الأساسية. |
| Backup read source | الملف يُقرأ حاليًا من قناة النسخة الاحتياطية. |

مصادر القراءة الأساسية والاحتياطية تقدم السلوك نفسه للمستخدم.

ما دام backup file متاحًا، تستمر الصور والفيديوهات وروابط التنزيل في العمل بعد التبديل إلى backup read source.

## ماذا يحدث عند حذف ملف؟

عند حذف ملف، يحذف ImgBed الملف الأساسي وملف النسخة الاحتياطية معًا.
