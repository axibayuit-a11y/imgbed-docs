# 圖片審查與存取模式

「圖片審查」用來替上傳圖片設定年齡分級，並透過「存取模式」控制公開訪問時能看到哪些內容。

這個設定會影響公開圖庫、公開 URL 連結和隨機圖 API。後台管理頁不受這個存取模式限制，管理員仍然可以查看和管理全部檔案。

## 功能入口

進入後台後打開：

```text
System Settings -> Security Settings -> Upload Management -> Image Moderation
```

主要有三個設定：

- 存取模式
- 啟用審查
- 審查渠道

## 存取模式是做什麼的

存取模式決定公開訪問時允許顯示到哪個年齡分級。

目前有四種：

| 存取模式 | 公開可見範圍 |
| --- | --- |
| 成人模式 | 全年齡、R12、R16、R18 |
| 青年模式 | 全年齡、R12、R16 |
| 少年模式 | 全年齡、R12 |
| 兒童模式 | 僅全年齡 |

預設是成人模式。

如果站台自用或成人內容較多，可以維持成人模式。如果希望公開圖庫更保守，可以改成青年、少年或兒童模式。

## 啟用審查是做什麼的

啟用審查後，系統會在上傳時呼叫你選擇的審查渠道，辨識圖片並寫入年齡分級。

分級主要有：

| 分級 | 含義 |
| --- | --- |
| 全年齡 | 一般公開內容 |
| R12 | 輕度敏感內容 |
| R16 | 中度敏感內容 |
| R18 | 成人內容 |

審查結果會參與公開存取判斷。

如果沒有啟用審查，或某些舊檔案還沒有分級，這些檔案會被視為未分級。未分級檔案不會只因為沒有審查結果，就自動從公開圖庫和隨機圖消失。

## 審查渠道怎麼選

目前審查渠道包括：

- moderatecontent.com
- nsfwjs
- Sightengine

不同渠道需要的設定不同：

- moderatecontent.com 通常需要 API Key。
- nsfwjs 通常需要填 API 端點。
- Sightengine 需要 API User 和 API Secret。

選哪個渠道取決於你的帳號、服務可用性和辨識效果。只要啟用審查並設定正確，上傳時就會嘗試寫入圖片分級。

## 對公開圖庫的影響

公開圖庫列表會依存取模式過濾內容。

例如：

- 成人模式：公開圖庫可以顯示 R18 圖片。
- 青年模式：公開圖庫不會顯示 R18 圖片。
- 少年模式：公開圖庫不會顯示 R16 和 R18 圖片。
- 兒童模式：公開圖庫只顯示全年齡圖片。

這只影響普通公開訪問。後台管理頁仍可看到全部檔案。

## 對公開 URL 連結的影響

公開 URL 連結就是使用者直接打開圖片連結的情境。

如果圖片分級在目前存取模式允許範圍內，系統會正常回傳原圖。

如果圖片分級超過目前存取模式，普通訪問不會回傳原圖，而是回傳攔截結果或提示圖。

例如：

- 目前是兒童模式。
- 某張圖是 R18。
- 訪客直接打開這張圖的公開連結。
- 系統不會把 R18 原圖回傳給普通訪客。

## 對隨機圖的影響

隨機圖 API 也會依存取模式過濾候選池。

如果目前是兒童模式，隨機圖只會從全年齡圖片裡抽取。

如果目前是青年模式，隨機圖可以從全年齡、R12、R16 圖片裡抽取，但不會抽到 R18。

## 和名單規則的關係

存取模式不是唯一的公開訪問規則，它會和名單規則一起生效。

簡單理解：

- 白名單內容優先公開。
- 黑名單內容普通使用者不能直接看到原圖。
- 不在名單裡的內容，再依存取模式判斷是否允許公開。

如果一張圖片同時被年齡分級和名單規則限制，普通使用者仍不能直接看到原圖。

## 建議設定

公開站台建議：

- 啟用審查。
- 依站台定位選擇存取模式。
- 面向全年齡使用者時選兒童模式或少年模式。
- 不希望公開展示成人內容時，不要使用成人模式。
- 定期檢查後台檔案分級，必要時手動調整。

私人或自用站台可以：

- 使用成人模式。
- 視需求啟用審查。
- 在後台管理頁查看和調整分級。

## 常見問題

### 改存取模式，後台檔案會消失嗎？

不會。存取模式只影響普通公開訪問，不影響後台管理頁。

### 改成兒童模式後，公開圖庫為什麼少很多圖片？

因為兒童模式只允許全年齡圖片公開顯示。R12、R16、R18 會被過濾。

### 公開 URL 還會直接打開成人圖片嗎？

如果目前存取模式不允許該分級，普通公開 URL 不會回傳原圖。

### 隨機圖會抽到被限制的圖片嗎？

不會。隨機圖會依目前存取模式過濾候選圖片。

### 沒有分級的舊圖片會怎樣？

未分級圖片不會只因為沒有審查結果就自動隱藏。你可以之後在後台手動調整分級。
