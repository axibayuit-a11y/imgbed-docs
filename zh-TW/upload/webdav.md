# WebDAV 渠道新增說明

## 適合什麼場景

- 你有 NAS、網盤或對象存儲服務提供的 WebDAV 地址。
- 你希望把圖片上傳到自己的 WebDAV 目錄里。
- 你希望設定和憑據保存在 D1 上傳渠道表 `upload_channels` 中，不直接放在前端頁面里長期暴露。


## 新增前要準備什麼

| 需要準備 | 用途 |
| --- | --- |
| WebDAV Endpoint | 服務端 WebDAV 地址，例如 `https://nas.example.com/dav`. |
| 使用者名 | 登入 WebDAV 服務使用 |
| 密碼 | 登入 WebDAV 服務使用 |
| 鑒權模式 | 預設 `Basic`，也可以按服務端要求選擇 `Digest` 或自動協商 |
| 存儲目錄 | 檔案保存目錄，預設 `imgbed` |

## 入口位置

1. 進入系統設定。
2. 打開上傳設定。
3. 點擊右上角“新增渠道”。
4. 選擇 `WebDAV`。

## 欄位說明

| 欄位 | 作用 | 是否必填 |
| --- | --- | --- |
| 渠道名稱 | 用來區分當前 WebDAV 渠道，例如 `koofr`、`nas` | 是 |
| Endpoint | WebDAV 服務地址，需要填寫完整 `https://` 地址 | 是 |
| 使用者名 | WebDAV 登入使用者名 | 是 |
| 密碼 | WebDAV 登入密碼 | 是 |
| 鑒權模式 | 一般選 `Basic`；服務端要求摘要認證時選 `Digest` | 是 |
| 存儲目錄 | 檔案保存目錄，預設 `imgbed` | 否 |

## 大檔案上傳能力

WebDAV 渠道現在走真會話分塊。

小檔案會直接上傳為一個完整檔案；超過 64 MiB 的檔案會自動切成 10 MiB 左右的分塊，分別上傳到遠端分塊目錄。

這里不要求 WebDAV 服務支援 `partial update`（部分更新）或按偏移寫入。系統不會在遠端把分塊重新合成一個檔案，而是保存分塊清單，讀取檔案時再按清單順序讀取分塊并拼接返回。

簡單理解：

| 檔案大小 | 上傳方式 | 遠端保存形態 |
| --- | --- | --- |
| 小于等于 64 MiB | 普通上傳 | 一個完整檔案 |
| 大于 64 MiB | 真會話分塊上傳 | 一個分塊目錄，里面保存多個分塊檔案 |

分塊目錄只影響遠端存儲結構，不會改變圖床里的檔案連結。使用者訪問時仍然使用原來的 `/file/...` 地址。

## 新增步驟

1. 打開上傳設定頁面。
2. 點擊“新增渠道”。
3. 選擇 `WebDAV`。
4. 渠道名稱填一個自己能看懂的名字，例如 `koofr`。
5. `Endpoint` 填 WebDAV 地址，例如 `https://app.koofr.net/dav/Koofr`.
6. 填寫使用者名和密碼。
7. 鑒權模式預設選 `Basic`。
8. 存儲目錄預設 `imgbed`，也可以改成自己的目錄。
9. 點擊保存。
10. 保存後查看渠道卡片，再查詢容量或上傳測試檔案確認連接可用。

## 新增完成後怎么檢查

| 檢查項 | 檢查方式 |
| --- | --- |
| 渠道卡片是否出現 | 保存後，上傳設定頁面應顯示 WebDAV 渠道卡片 |
| 渠道是否啟用 | 卡片右上角開關應保持開啟 |
| 憑據是否保存 | 詳情頁能看到 Endpoint、使用者名、鑒權模式、存儲目錄 |
| 小檔案上傳是否正常 | 上傳一張測試圖片，確認 WebDAV 目錄里出現檔案 |
| 大檔案上傳規則 | 超過 64 MiB 會走分塊上傳，遠端會保存分塊目錄和分塊檔案 |
| 容量查詢是否可用 | 點擊查詢，如果服務端支援容量資訊，會顯示已用容量和總容量 |

## 常見問題

### WebDAV 大檔案為什麼會出現分塊目錄？

這是現在的大檔案保存方式。

超過 64 MiB 的檔案不會在遠端合成一個大檔案，而是保存為分塊目錄。圖床會記錄分塊清單，訪問時按順序讀取分塊并返回完整內容。

### 大檔案上傳失敗先檢查什麼？

先檢查 Endpoint、使用者名、密碼和存儲目錄是否正確，再檢查 WebDAV 服務是否允許建立目錄、寫入檔案、讀取檔案。

如果容量查詢失敗，但小檔案上傳正常，通常是服務端不支援或限制容量資訊查詢，不一定代表上傳不可用。


### 鑒權模式怎么選？

一般先用 `Basic`。

如果服務端明確要求摘要認證，就選 `Digest`。

不確定時可以選自動協商。
