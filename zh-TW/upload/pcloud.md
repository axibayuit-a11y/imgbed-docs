# 新增 pCloud 渠道

## 適合什麼情境

- 你有 pCloud 帳號，想把圖片存到 pCloud 空間。
- 你可以接受用帳號信箱和密碼作為渠道憑證。

## 新增前要準備什麼

| 需要準備 | 用途 |
| --- | --- |
| pCloud 帳號信箱 | 登入 pCloud API |
| pCloud 密碼 | 登入 pCloud API |
| 主機網域 | 預設 `api.pcloud.com`，歐洲區可用 `eapi.pcloud.com` |
| 儲存目錄 | 檔案存放目錄，預設 `imgbed` |

## 入口位置

1. 進入系統設定。
2. 打開上傳設定。
3. 點右上角「新增渠道」。
4. 選擇 `pCloud`。

## 欄位說明

| 欄位 | 作用 | 是否必填 |
| --- | --- | --- |
| 渠道名稱 | 用來辨識這個 pCloud 渠道，例如 `pCloud主帳號` | 是 |
| 帳號信箱 | pCloud 登入信箱 | 是 |
| 密碼 | pCloud 登入密碼 | 是 |
| 主機網域 | pCloud API 網域，預設 `api.pcloud.com` | 否 |
| 儲存目錄 | 檔案存放目錄，預設 `imgbed` | 否 |

主機網域通常這樣選：

| 帳號區域 | 主機網域 |
| --- | --- |
| 預設 / 美國區 | `api.pcloud.com` |
| 歐洲區 | `eapi.pcloud.com` |

## 新增步驟

1. 打開上傳設定頁。
2. 點「新增渠道」。
3. 選擇 `pCloud`。
4. 渠道名稱填一個自己看得懂的名字。
5. 帳號信箱填 pCloud 登入信箱。
6. 密碼填 pCloud 密碼。
7. 主機網域預設 `api.pcloud.com`；歐洲區帳號改成 `eapi.pcloud.com`。
8. 儲存目錄預設 `imgbed`，也可以改成自己的資料夾。
9. 儲存。

## 新增完成後怎麼檢查

| 檢查項目 | 檢查方式 |
| --- | --- |
| 渠道卡片是否出現 | 儲存後，上傳設定頁應該看得到 pCloud 渠道卡 |
| 渠道是否啟用 | 卡片右上角開關維持開啟 |
| 信箱是否顯示 | 卡片裡能看到目前 pCloud 信箱 |
| 容量查詢是否正常 | 點查詢後會顯示已用容量和總容量 |
| 上傳是否正常 | 上傳測試圖片，確認 pCloud 儲存目錄裡出現檔案 |

## 常見問題

### 為什麼不用 OAuth2？

pCloud 的 OAuth2 不是預設自助開通，需要寫信請官方啟用。

而且目前 OAuth2 授權方式不支援 ImgBed 需要的短期上傳連結流程，所以這裡直接使用帳號信箱和密碼登入。

### 主機網域要填什麼？

預設填：

```text
api.pcloud.com
```

如果你的帳號是歐洲區，可以填：

```text
eapi.pcloud.com
```

## 快速流程

```text
準備 pCloud 信箱和密碼
-> 打開上傳設定
-> 新增渠道
-> 選 pCloud
-> 填渠道名稱 / 信箱 / 密碼
-> 主機網域預設 api.pcloud.com
-> 儲存目錄預設 imgbed
-> 儲存
-> 查詢容量
-> 上傳測試圖片
```
