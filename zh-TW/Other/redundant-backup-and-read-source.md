# 備援備份與讀取來源切換

備援備份會替已上傳檔案再存一份副本。

主檔案和備份檔案都可以作為讀取來源，使用者看起來沒有差別；差別只在檔案實際從哪個儲存渠道讀取。

## 能做什麼

| 功能 | 說明 |
| --- | --- |
| 多存一份檔案 | 備份到另一個上傳渠道，降低單一渠道故障風險 |
| 切換讀取來源 | 備份成功後，可在主渠道與備份渠道之間切換 |
| 單檔備份 | 在檔案詳情裡替單一檔案備份 |
| 批次備份 | 在管理頁框選多個檔案後批次備份 |
| 全域備援備份 | 在其他設定裡依資料夾批次補備份 |

## 備援備份入口

```text
系統設定 -> 其他設定 -> 備援備份
```

![備援備份截圖](../../image/other/冗余备份截图.png)

這裡適合替某個資料夾或全部檔案統一補備份。

備份渠道可以手動選，也可以選自動切換，讓系統自己找合適的備份渠道。

## 檔案詳情裡備份

在管理後台打開檔案詳情，可以直接點備份檔案。

![檔案詳情裡備份](../../image/other/文件详情里文件备份.png)

這個入口適合臨時替單一重要檔案補備份。

備份成功後，檔案詳情裡會出現可切換的讀取來源。

## 框選批次備份

在管理後台可以框選多個檔案，然後執行批次備份。

![批次備份截圖](../../image/other/批量备份截图.png)

框選備份、檔案詳情備份、其他設定裡的備援備份，本質上都是同一套備份功能，只是入口不同。

## 備份成功後切換讀取來源

| 讀取來源 | 說明 |
| --- | --- |
| 主渠道 | 從原始上傳渠道讀取檔案 |
| 備份渠道 | 從備份渠道讀取檔案 |

![備份成功後切換讀取來源](../../image/other/备份成功切换读取源.png)

切到哪個讀取來源，後續訪問檔案時就優先從那個渠道讀取。

## 哪些情況會跳過

| 情況 | 為什麼跳過 |
| --- | --- |
| 已經有備份 | 不重複備份，避免浪費空間 |
| 主備渠道相同 | 備份必須放到另一個渠道才有意義 |
| 沒有可用備份渠道 | 目前找不到適合的其他渠道 |

## 主渠道和備份渠道

| 名稱 | 說明 |
| --- | --- |
| 主渠道 | 檔案第一次上傳時使用的渠道 |
| 備份渠道 | 後來透過備援備份產生副本的渠道 |
| 主讀取來源 | 目前從主渠道讀取 |
| 備份讀取來源 | 目前從備份渠道讀取 |

只要備份檔案可用，切到備份讀取來源後，圖片、影片、下載連結都會照常使用。

## 刪除檔案時會發生什麼

刪除檔案時，系統會同時刪除主檔案和備份檔案。
