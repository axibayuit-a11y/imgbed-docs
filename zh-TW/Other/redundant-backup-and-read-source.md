# 冗餘備份與讀取來源切換

冗餘備份會為已上傳的檔案額外儲存一份副本。

主檔案和備份檔案都可以作為讀取來源。訪客通常看不出差異，唯一不同的是由哪個儲存管道提供檔案。

## 冗餘備份能做什麼

| 功能 | 說明 |
| --- | --- |
| 儲存額外副本 | 將檔案備份到另一個上傳管道，降低單一管道故障的風險。 |
| 切換讀取來源 | 備份成功後，在主要管道與備份管道之間切換檔案讀取。 |
| 單一檔案備份 | 從檔案詳情頁面備份單一檔案。 |
| 批次備份 | 在管理頁面選取多個檔案，並一起備份。 |
| 全域冗餘備份 | 從其他設定依資料夾備份檔案。 |

## 冗餘備份入口

開啟：

```text
System Settings -> Other Settings -> Redundant Backup
```

![冗餘備份](../../image/other/冗余备份截图.png)

此入口最適合為某個資料夾或所有檔案批次新增備份。

備份管道可以手動選擇，也可以選擇自動切換，讓 ImgBed 尋找合適的備份管道。

## 從檔案詳情備份

在管理面板開啟檔案詳情頁面，然後點擊備份。

![檔案詳情中的備份](../../image/other/文件详情里文件备份.png)

此入口最適合依需求備份單一重要檔案。

備份成功後，檔案詳情頁面會顯示可用的讀取來源。

## 透過選取批次備份

在管理面板中選取多個檔案，然後執行批次備份。

![批次備份](../../image/other/批量备份截图.png)

此入口最適合處理一組檔案。

選取備份、檔案詳情備份，以及其他設定下的冗餘備份，都使用同一套備份系統。它們只是不同的入口。

## 備份後切換讀取來源

備份完成後，檔案詳情頁面可讓你切換讀取來源：

| 讀取來源 | 說明 |
| --- | --- |
| 主要管道 | 從原始上傳管道讀取。 |
| 備份管道 | 從備份管道讀取。 |

![備份後切換讀取來源](../../image/other/备份成功切换读取源.png)

訪客不需要知道檔案是由主要管道還是備份管道提供。

你選擇的讀取來源會成為後續檔案存取的優先來源。

## 何時會跳過備份

下列情況會在備份期間跳過。這些情況不是錯誤。

| 情況 | 跳過原因 |
| --- | --- |
| 已有備份 | 已經有備份的檔案不會再次備份。 |
| 主要管道與備份管道相同 | 備份必須儲存在另一個管道才有意義。 |
| 沒有可用的備份管道 | 沒有合適的替代管道可用。 |

簡而言之：備份必須寫入另一個管道，已經備份過的檔案也不會再次佔用額外空間。

## 主要管道與備份管道

| 名稱 | 含義 |
| --- | --- |
| 主要管道 | 檔案首次上傳時使用的管道。 |
| 備份管道 | 儲存冗餘副本的管道。 |
| 主要讀取來源 | 檔案目前從主要管道讀取。 |
| 備份讀取來源 | 檔案目前從備份管道讀取。 |

主要讀取來源和備份讀取來源提供相同的使用者體驗。

只要備份檔案可用，切換到備份讀取來源後，圖片、影片和下載連結仍會繼續運作。

## 刪除檔案時會發生什麼

刪除檔案時，ImgBed 會同時刪除主檔案和備份檔案。
