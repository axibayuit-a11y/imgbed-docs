# Резервное дублирование и переключение источника чтения

Резервное дублирование хранит дополнительную копию уже загруженного файла.

И основной файл, и файл резервной копии могут использоваться как источники чтения. Для посетителей обычно нет разницы. Отличается только канал хранения, который отдает файл.

## Что может резервное дублирование

| Возможность | Описание |
| --- | --- |
| Хранить дополнительную копию | Создает резервную копию файлов в другом канале загрузки, чтобы снизить риск отказа одного канала. |
| Переключать источник чтения | После успешного резервного копирования переключает чтение файлов между основным каналом и каналом резервной копии. |
| Резервное копирование одного файла | Создает резервную копию одного файла со страницы его сведений. |
| Пакетное резервное копирование | Выбирает несколько файлов на странице администрирования и создает их резервные копии вместе. |
| Глобальное резервное дублирование | Создает резервные копии файлов по папкам в других настройках. |

## Раздел резервного дублирования

Откройте:

```text
System Settings -> Other Settings -> Redundant Backup
```

![Резервное дублирование](../../image/other/冗余备份截图.png)

Этот раздел лучше всего подходит для добавления резервных копий к папке или ко всем файлам в пакетном режиме.

Канал резервной копии можно выбрать вручную, либо можно выбрать автоматическое переключение и позволить ImgBed найти подходящий канал резервной копии.

## Резервное копирование со страницы сведений о файле

Откройте страницу сведений о файле в панели администратора и нажмите кнопку резервного копирования.

![Резервное копирование в сведениях о файле](../../image/other/文件详情里文件备份.png)

Это лучше всего подходит для резервного копирования одного важного файла по требованию.

После успешного резервного копирования страница сведений о файле показывает доступные источники чтения.

## Пакетное резервное копирование по выбору

В панели администратора выберите несколько файлов и запустите пакетное резервное копирование.

![Пакетное резервное копирование](../../image/other/批量备份截图.png)

Это лучше всего подходит для обработки группы файлов.

Резервное копирование по выбору, резервное копирование со страницы сведений о файле и резервное дублирование в других настройках используют одну и ту же систему резервного копирования. Это просто разные точки входа.

## Переключение источника чтения после резервного копирования

После завершения резервного копирования страница сведений о файле позволяет переключить источник чтения:

| Источник чтения | Описание |
| --- | --- |
| Основной канал | Читает из исходного канала загрузки. |
| Канал резервной копии | Читает из канала резервной копии. |

![Переключение источника чтения после резервного копирования](../../image/other/备份成功切换读取源.png)

Посетителям не нужно знать, отдается ли файл из основного канала или из канала резервной копии.

Выбранный источник чтения становится предпочтительным источником для последующих обращений к файлу.

## Когда резервное копирование пропускается

Следующие случаи пропускаются во время резервного копирования. Это не ошибки.

| Случай | Почему пропускается |
| --- | --- |
| Уже есть резервная копия | Файл, у которого уже есть резервная копия, не копируется повторно. |
| Основной канал и канал резервной копии совпадают | Резервная копия должна храниться в другом канале, чтобы иметь смысл. |
| Нет доступного канала резервной копии | Подходящий альтернативный канал недоступен. |

Кратко: резервные копии должны уходить в другой канал, а файлы, у которых уже есть резервная копия, не занимают место повторно.

## Основной канал и канал резервной копии

| Название | Значение |
| --- | --- |
| Основной канал | Канал, использованный при первой загрузке файла. |
| Канал резервной копии | Канал, в котором хранится избыточная копия. |
| Основной источник чтения | Сейчас файл читается из основного канала. |
| Источник чтения резервной копии | Сейчас файл читается из канала резервной копии. |

Основной источник чтения и источник чтения резервной копии ведут себя одинаково для пользователя.

Пока файл резервной копии доступен, изображения, видео и ссылки на скачивание продолжают работать после переключения на источник чтения резервной копии.

## Что происходит при удалении файла

Когда файл удаляется, ImgBed удаляет и основной файл, и файл резервной копии.
