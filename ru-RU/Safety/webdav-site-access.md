# WebDAV-доступ к сайту (Beta)

Настройка WebDAV в разделе настроек безопасности открывает ваш сайт ImgBed как конечную точку WebDAV.

После включения можно использовать Windows, macOS, мобильные файловые менеджеры или любой WebDAV-совместимый клиент, чтобы просматривать, загружать, удалять и управлять файлами ImgBed как удаленной папкой.

Это вход WebDAV-доступа именно к сайту. Он отличается от канала хранения WebDAV в настройках загрузки. Канал загрузки хранит файлы в стороннем WebDAV-сервисе. Эта настройка позволяет вашему сайту ImgBed предоставлять WebDAV-доступ клиентам.

## Где это настраивается

Откройте админ-панель и перейдите в:

```text
System Settings -> Security Settings -> WebDAV
```

Доступные настройки:

- Включить
- Имя пользователя
- Пароль
- Режим загрузки изображений
- Канал по умолчанию

## Что делает эта функция

После включения WebDAV ImgBed предоставляет фиксированный URL доступа:

```text
https://your-domain.com/dav
```

Используйте этот URL для подключения к файловой директории ImgBed.

Полезные сценарии:

- просматривать файлы ImgBed напрямую из файлового менеджера компьютера;
- перетаскивать изображения в WebDAV-папку для загрузки;
- упорядочивать папки ImgBed из локального файлового менеджера;
- использовать WebDAV-совместимое ПО для синхронизации или управления изображениями;
- получать доступ к содержимому ImgBed без открытия админ-панели.

## Настройки

### Включить

Включает конечную точку WebDAV.

Когда выключено, клиенты не могут подключиться через WebDAV.

### Имя пользователя и пароль

Эти учетные данные используются WebDAV-клиентами при подключении.

Используйте отдельные имя пользователя и пароль WebDAV. Не используйте повторно пароль администратора или пароль загрузки.

Если имя пользователя или пароль пустые, WebDAV-клиенты не смогут подключаться корректно.

### Режим загрузки изображений

Режим загрузки изображений определяет, какой URL изображения WebDAV-клиенты предпочитают при чтении изображений.

Распространенные варианты:

| Режим | Описание |
| --- | --- |
| Умная загрузка | ImgBed выбирает по контексту. Рекомендуется для обычного использования. |
| Оригинал | Предпочитать оригинальные изображения. |
| Миниатюра | Предпочитать миниатюры. Полезно для быстрого предварительного просмотра. |

Если не уверены, оставьте `Умная загрузка`.

### Канал по умолчанию

Канал по умолчанию используется для WebDAV-загрузок.

Когда вы копируете файлы в WebDAV-директорию из Windows или другого клиента, ImgBed загружает их через выбранный канал загрузки по умолчанию.

Если канал по умолчанию не выбран, просмотр может работать, но загрузка может падать.

## Доступ к WebDAV в Windows 11

Windows 11 может добавить WebDAV как сетевое расположение.

1. Откройте `Этот компьютер`.
2. Выберите `Добавить сетевое расположение`.
3. Введите `https://your-domain.com/dav`.
4. Когда появится запрос, введите имя пользователя и пароль WebDAV.
5. Завершите мастер. После этого WebDAV-директорию можно открыть в Проводнике.

![Добавление WebDAV в Windows 11](../../image/Safety/webdav在win11配置.png)

После добавления WebDAV-директория появится в Проводнике Windows. Можно открывать, копировать и управлять файлами как в обычной папке.

![WebDAV в Windows](../../image/Safety/webdav在win显示效果.png)

## Поддерживаемые операции

После успешного WebDAV-подключения обычно можно:

- просматривать файлы и папки;
- загружать файлы;
- создавать папки;
- переименовывать файлы или папки;
- перемещать файлы;
- удалять файлы.

WebDAV лучше подходит для повседневного доступа и небольшого управления файлами. Для крупных перемещений, массовых удалений или сложной организации лучше использовать админ-панель.

## Управление устройствами входа

Успешные WebDAV-подключения также появляются на вкладке WebDAV в управлении устройствами входа.

Там можно просматривать WebDAV-клиенты и при необходимости принудительно выводить старые устройства офлайн.

Если изменить имя пользователя или пароль WebDAV, старым клиентам нужно будет войти снова.

## FAQ

### Windows постоянно спрашивает имя пользователя и пароль

Проверьте:

- URL должен быть `https://your-domain.com/dav`.
- Имя пользователя и пароль совпадают с настройками WebDAV.
- WebDAV включен.
- Сайт доступен через HTTPS.

### Просмотр работает, но загрузка нет

Проверьте `Канал по умолчанию`.

Для WebDAV-загрузок нужен канал загрузки по умолчанию. Если его нет, он выключен или неправильно настроен, загрузка может не работать.

### Скорость доступа нестабильна

Производительность WebDAV зависит от клиента, сети, количества файлов и канала загрузки по умолчанию.

Если в директории много файлов, разложите их по папкам, а не держите слишком много файлов в одной директории.

## Рекомендации по безопасности

- Используйте HTTPS для WebDAV-доступа.
- Установите надежный пароль.
- Не передавайте пароль WebDAV людям, которым не доверяете.
- Выключайте WebDAV, когда не пользуетесь им.
- Периодически очищайте неиспользуемые устройства WebDAV в управлении устройствами входа.
