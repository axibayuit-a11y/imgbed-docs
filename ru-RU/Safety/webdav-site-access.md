# WebDAV-доступ к сайту (Beta)

Настройка WebDAV в Security Settings открывает ваш сайт ImgBed как WebDAV endpoint.

После включения можно использовать Windows, macOS, мобильные файловые менеджеры или любой WebDAV-совместимый клиент, чтобы просматривать, загружать, удалять и управлять файлами ImgBed как удаленной папкой.

Это вход WebDAV-доступа именно к сайту. Он отличается от WebDAV storage channel в настройках загрузки. Канал загрузки хранит файлы в стороннем WebDAV-сервисе. Эта настройка позволяет вашему сайту ImgBed предоставлять WebDAV-доступ клиентам.

## Где это настраивается

Откройте админ-панель и перейдите в:

```text
System Settings -> Security Settings -> WebDAV
```

Доступные настройки:

- Enable
- Username
- Password
- Image loading mode
- Default channel

## Что делает эта функция

После включения WebDAV ImgBed предоставляет фиксированный URL доступа:

```text
https://your-domain.com/dav
```

Используйте этот URL для подключения к файловой директории ImgBed.

Полезные сценарии:

- просматривать файлы ImgBed напрямую из файлового менеджера компьютера;
- перетаскивать изображения в WebDAV-папку для загрузки;
- упорядочивать папки ImgBed из локального файлового менеджера;
- использовать WebDAV-совместимое ПО для синхронизации или управления изображениями;
- получать доступ к содержимому ImgBed без открытия админ-панели.

## Настройки

### Enable

Включает WebDAV endpoint.

Когда выключено, клиенты не могут подключиться через WebDAV.

### Username и Password

Эти учетные данные используются WebDAV-клиентами при подключении.

Используйте отдельные WebDAV username и password. Не используйте повторно admin password или upload password.

Если username или password пустые, WebDAV-клиенты не смогут подключаться корректно.

### Image Loading Mode

Image loading mode определяет, какой URL изображения WebDAV-клиенты предпочитают при чтении изображений.

Распространенные варианты:

| Mode | Описание |
| --- | --- |
| Smart loading | ImgBed выбирает по контексту. Рекомендуется для обычного использования. |
| Original | Предпочитать оригинальные изображения. |
| Thumbnail | Предпочитать миниатюры. Полезно для быстрого preview. |

Если не уверены, оставьте `Smart loading`.

### Default Channel

Default channel используется для WebDAV-загрузок.

Когда вы копируете файлы в WebDAV-директорию из Windows или другого клиента, ImgBed загружает их через выбранный default upload channel.

Если default channel не выбран, просмотр может работать, но загрузка может падать.

## Доступ к WebDAV в Windows 11

Windows 11 может добавить WebDAV как network location.

1. Откройте `This PC`.
2. Выберите `Add a network location`.
3. Введите `https://your-domain.com/dav`.
4. Когда появится запрос, введите WebDAV username и password.
5. Завершите мастер. После этого WebDAV-директорию можно открыть в File Explorer.

![Добавление WebDAV в Windows 11](../../image/Safety/webdav在win11配置.png)

После добавления WebDAV-директория появится в Windows File Explorer. Можно открывать, копировать и управлять файлами как в обычной папке.

## Поддерживаемые операции

После успешного WebDAV-подключения обычно можно:

- просматривать файлы и папки;
- загружать файлы;
- создавать папки;
- переименовывать файлы или папки;
- перемещать файлы;
- удалять файлы.

WebDAV лучше подходит для повседневного доступа и небольшого управления файлами. Для крупных перемещений, массовых удалений или сложной организации лучше использовать админ-панель.

## Login Device Management

Успешные WebDAV-подключения также появляются на вкладке WebDAV в Login Device Management.

Там можно просматривать WebDAV-клиенты и при необходимости принудительно выводить старые устройства офлайн.

Если изменить WebDAV username или password, старым клиентам нужно будет войти снова.

## FAQ

### Windows постоянно спрашивает username и password

Проверьте:

- URL должен быть `https://your-domain.com/dav`.
- Username и password совпадают с настройками WebDAV.
- WebDAV включен.
- Сайт доступен через HTTPS.

### Просмотр работает, но загрузка нет

Проверьте `Default channel`.

Для WebDAV-загрузок нужен default upload channel. Если его нет, он выключен или неправильно настроен, загрузка может не работать.

### Скорость доступа нестабильна

Производительность WebDAV зависит от клиента, сети, количества файлов и default upload channel.

Если в директории много файлов, разложите их по папкам, а не держите слишком много файлов в одной директории.

## Рекомендации по безопасности

- Используйте HTTPS для WebDAV-доступа.
- Установите надежный пароль.
- Не передавайте WebDAV password людям, которым не доверяете.
- Выключайте WebDAV, когда не пользуетесь им.
- Периодически очищайте неиспользуемые WebDAV devices в Login Device Management.

## Размер файлов для WebDAV-загрузки

WebDAV-клиенты не используют browser upload page's large-file chunking flow. Для файлов больше рекомендованных ниже лимитов используйте веб-страницу загрузки.

| Default Upload Channel | Рекомендуемый лимит одного файла для WebDAV |
| --- | ---: |
| Telegram | 20 MB |
| Discord | 10 MB |
| Cloudflare R2 | 30 MB |
| S3 | 30 MB |
| GitHub Releases | 64 MB |
| GitLab Packages | 64 MB |
| Hugging Face | 20 MB |
| OneDrive | 30 MB |
| Google Drive | 30 MB |
| Dropbox | 30 MB |
| Yandex Disk | 30 MB |
| pCloud | 30 MB |
| WebDAV | 64 MB |
