# Настройки загрузки

Настройки загрузки подключают ImgBed к вашим собственным каналам хранения. После настройки загруженные изображения и файлы сохраняются в выбранном сервисе, а ImgBed управляет ссылками доступа, записями файлов, предпросмотром, публичной галереей, API случайного изображения, WebDAV-доступом и связанными возможностями.

Разным пользователям подходят разные каналы. Для простого старта можно выбрать Telegram, Discord или GitHub Releases. Если важнее объем, скорость и долгосрочная стабильность, рассмотрите Cloudflare R2, S3, OneDrive, Google Drive, Dropbox, Yandex, pCloud или собственный WebDAV.

## Перед началом

> Перед первым использованием ImgBed необходимо открыть страницу инициализации и нажать «Перестроить индекс», чтобы добавить необходимые таблицы D1 и избежать ошибок в последующих функциях.
>
> ![Нажать Перестроить индекс при инициализации](../../image/初始化点击重建索引.png)

- Подготовьте учетную запись хранилища или API-данные, которые будете использовать.
- Убедитесь, что домен ImgBed доступен, потому что OAuth-каналам нужны URL обратного вызова.
- После добавления канала сначала загрузите тестовое изображение и проверьте, что файл сохраняется и открывается.
