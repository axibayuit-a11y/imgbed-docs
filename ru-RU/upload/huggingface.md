# Добавление канала Hugging Face

## Что подготовить заранее

Нужны всего три вещи:

| Что нужно | Зачем это нужно |
| --- | --- |
| Аккаунт Hugging Face | Для создания access token и владения репозиторием. |
| Hugging Face User Access Token | ImgBed использует его для доступа к Hugging Face API, создания репозиториев и загрузки файлов. |
| Имя репозитория | Можно ввести только имя репозитория, например `image`. |

## Настройка

### Шаг 1. Войдите в Hugging Face и создайте Access Token

1. Войдите в Hugging Face.
2. Нажмите аватар в правом верхнем углу и откройте `Settings`.
3. В левой боковой панели откройте `Access Tokens`.
4. Создайте новый token.
5. Дайте token понятное имя.
6. Выберите разрешение `write`.
7. Сразу после создания скопируйте и сохраните token.

![Создание token](../../image/upload/huggingface/创建令牌.png)

## Шаг 2. Заполните канал Hugging Face в ImgBed

После выбора `Hugging Face` в настройках загрузки заполните поля:

| Поле в интерфейсе | Что ввести |
| --- | --- |
| Channel name | Имя на ваш выбор, например `hf-primary`. |
| Repository name | Короткое имя repo, например `image`, или полный путь вроде `username/image`. |
| Access Token | Hugging Face User Access Token, который вы только что создали. |
| Private repository | Включите или выключите по своим потребностям. |
| Remark | Необязательно, например `Primary upload channel`. |

![Добавление канала](../../image/upload/huggingface/添加渠道.png)

## Шаг 3. Сохраните канал

После заполнения полей нажмите Save.

Система сама обработает такие детали:

| Поведение системы | Описание |
| --- | --- |
| Короткое имя репозитория | ImgBed определяет текущий аккаунт Hugging Face и разворачивает значение в полный путь репозитория. |
| Полный путь репозитория | ImgBed использует путь `username/repository` ровно в том виде, как он введен. |
| Проверка репозитория | Если используется путь текущего личного аккаунта, ImgBed попробует создать репозиторий, если его еще нет. Если полный путь введен вручную, ImgBed использует его напрямую. |
| Тип репозитория | Этот канал использует репозиторий типа `dataset`. |
| Публичный или приватный статус | Видимость репозитория синхронизируется с текущим переключателем. |

## Короткий чек-лист

```text
Войти в Hugging Face
-> Создать Access Token
-> Выбрать разрешение write
-> Вернуться в ImgBed и ввести token и имя репозитория
-> Сохранить
-> Если введено только имя repo, ImgBed автоматически добавит текущий username
-> Если введено username/repo, ImgBed использует это как есть
-> ImgBed проверит или создаст dataset repository
-> Загрузить тестовое изображение
```
