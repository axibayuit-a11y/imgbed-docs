# اضافي شاتړ او د لوستلو سرچینې بدلول

اضافي شاتړ د هغه فایل لپاره یوه بله کاپي ساتي چې مخکې اپلوډ شوی وي.

اصلي فایل او د شاتړ فایل دواړه د لوستلو سرچینو په توګه کارېدای شي. لیدونکي عموما توپیر نه ویني. یوازینی توپیر دا دی چې فایل له کوم ذخیره چینل څخه وړاندې کېږي.

## اضافي شاتړ څه کولی شي

| ځانګړنه | تشریح |
| --- | --- |
| اضافي کاپي ساتل | فایلونه بل اپلوډ چینل ته شاتړ کوي، څو د یوه چینل د ناکامۍ خطر کم شي. |
| د لوستلو سرچینه بدلول | له بریالي شاتړ وروسته، د فایل لوستل د اصلي چینل او شاتړ چینل تر منځ بدلوي. |
| د یو فایل شاتړ | د فایل د جزئیاتو له پاڼې څخه د یو فایل شاتړ جوړوي. |
| ډله ییز شاتړ | په اداري پاڼه کې څو فایلونه وټاکئ او یو ځای یې شاتړ کړئ. |
| عمومي اضافي شاتړ | له نورو تنظیماتو څخه فایلونه د فولډر له مخې شاتړ کوي. |

## د اضافي شاتړ ځای

دا پرانیزئ:

```text
System Settings -> Other Settings -> Redundant Backup
```

![اضافي شاتړ](../../image/other/冗余备份截图.png)

دا ځای د فولډر یا ټولو فایلونو لپاره په ډله ییز ډول د شاتړ زیاتولو لپاره مناسب دی.

د شاتړ چینل په لاس ټاکل کېدای شي. تاسو کولای شئ اتومات بدلول هم وټاکئ او ImgBed ته پرېږدئ چې مناسب شاتړ چینل ومومي.

## د فایل له جزئیاتو څخه شاتړ

په اداري پنل کې د فایل د جزئیاتو پاڼه پرانیزئ او د شاتړ تڼۍ کلیک کړئ.

![د فایل په جزئیاتو کې شاتړ](../../image/other/文件详情里文件备份.png)

دا د اړتیا پر وخت د یوه مهم فایل د شاتړ لپاره ډېر مناسب دی.

له بریالي شاتړ وروسته، د فایل جزئیاتو پاڼه شته د لوستلو سرچینې ښيي.

## د ټاکنې له لارې ډله ییز شاتړ

په اداري پنل کې څو فایلونه وټاکئ او ډله ییز شاتړ اجرا کړئ.

![ډله ییز شاتړ](../../image/other/批量备份截图.png)

دا د فایلونو د یوې ډلې د پروسس لپاره مناسب دی.

د ټاکنې شاتړ، د فایل جزئیاتو شاتړ او په نورو تنظیماتو کې اضافي شاتړ ټول هماغه د شاتړ سیستم کاروي. یوازې د کارولو ځایونه یې بېلابېل دي.

## له شاتړ وروسته د لوستلو سرچینه بدلول

کله چې شاتړ بشپړ شي، د فایل جزئیاتو پاڼه د لوستلو سرچینې بدلولو اجازه درکوي:

| د لوستلو سرچینه | تشریح |
| --- | --- |
| اصلي چینل | فایل له اصلي اپلوډ چینل څخه لولي. |
| شاتړ چینل | فایل له شاتړ چینل څخه لولي. |

![له شاتړ وروسته د لوستلو سرچینه بدلول](../../image/other/备份成功切换读取源.png)

لیدونکو ته اړتیا نشته پوه شي چې فایل له اصلي چینل څخه وړاندې کېږي که له شاتړ چینل څخه.

هره د لوستلو سرچینه چې وټاکئ، د راتلونکو فایل لاسرسیو لپاره غوره سرچینه ګرځي.

## کله شاتړ پرېښودل کېږي

لاندې حالتونه د شاتړ پر مهال پرېښودل کېږي. دا تېروتنې نه دي.

| حالت | ولې پرېښودل کېږي |
| --- | --- |
| مخکې شاتړ شوی | هغه فایل چې شاتړ لري، بیا شاتړ نه کېږي. |
| اصلي او شاتړ چینلونه یو شان دي | شاتړ باید په بل چینل کې وساتل شي څو معنا ولري. |
| کارېدونکی شاتړ چینل نشته | مناسب بدیل چینل نشته. |

په لنډه توګه: شاتړونه باید بل چینل ته لاړ شي، او هغه فایلونه چې مخکې شاتړ لري بیا اضافي ځای نه نیسي.

## اصلي چینل او شاتړ چینل

| نوم | معنا |
| --- | --- |
| اصلي چینل | هغه چینل چې فایل لومړی پرې اپلوډ شوی و. |
| شاتړ چینل | هغه چینل چې اضافي کاپي ساتي. |
| اصلي د لوستلو سرچینه | فایل اوس له اصلي چینل څخه لوستل کېږي. |
| د شاتړ د لوستلو سرچینه | فایل اوس له شاتړ چینل څخه لوستل کېږي. |

اصلي او د شاتړ د لوستلو سرچینې د کاروونکي له نظره یو شان چلند لري.

تر څو د شاتړ فایل شته وي، انځورونه، ویډیوګانې او د ډاونلوډ لینکونه د شاتړ د لوستلو سرچینې ته له بدلون وروسته هم کار کوي.

## کله فایل ړنګ شي څه کېږي

کله چې فایل ړنګ شي، ImgBed اصلي فایل او د شاتړ فایل دواړه ړنګوي.
