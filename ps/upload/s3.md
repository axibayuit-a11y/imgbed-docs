# د S3 چینل اضافه کول

## کله مناسب دی

S3 چینل هغه وخت وکاروئ چې:

- غواړئ هر S3-compatible object زېرمه خدمت وصل کړئ، لکه Cloudflare R2، Backblaze B2، MinIO، AWS S3 یا بل compatible provider.
- د standard S3 تنظیمات model غوره کوئ: `Endpoint`، لاسرسی Key او Secret Key.
- نه غواړئ native Cloudflare R2 binding چینل وکاروئ، یا ستاسې زېرمه provider Cloudflare R2 نه وي.

## د پیل مخکې اړتیاوې

| اړتیا | موخه |
| --- | --- |
| S3 Endpoint | د object زېرمه خدمت S3 API endpoint. |
| Bucket name | هغه bucket چې فایلونه پکې ساتل کېږي. |
| Region | زېرمه region. د Cloudflare R2 لپاره عموماً `auto`. |
| لاسرسی Key ID | S3 لاسرسی key ID. |
| Secret لاسرسی Key | S3 secret لاسرسی key. |
| CDN ډومېن | د فایل لاسرسی لپاره اختیاري custom ډومېن. |

د Cloudflare R2 لپاره د S3 API له لارې مثال:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## چېرته یې اضافه کړو

1. د سیسټم تنظیمات پرانیزئ.
2. د اپلوډ تنظیمات ته ولاړ شئ.
3. په پورته ښي لوري کې چینل اضافه کړئ کلیک کړئ.
4. `S3` وټاکئ.

## د فیلډونو لارښود

| فیلډ | څه کوي | اړین |
| --- | --- | --- |
| د چینل نوم | د دې S3 چینل لپاره پېژندل کېدونکی نوم، لکه `s3test` یا `R2-S3`. | Yes |
| Enable چینل | کنټرولوي چې دا چینل په اپلوډ selection کې ګډون وکړي که نه. | سپارښتنه کېږي |
| Endpoint | بشپړ S3 خدمت endpoint د `https://` سره. | Yes |
| Custom CDN ډومېن | اختیاري. که set شي، generated فایل links به دا ډومېن غوره کړي. | No |
| Bucket name | bucket name، لکه `s3test` یا `img-r2`. | Yes |
| Bucket region | Region. د Cloudflare R2 لپاره عموماً `auto`. | Yes |
| لاسرسی Key ID | S3 لاسرسی key ID. | Yes |
| Secret لاسرسی Key | S3 secret لاسرسی key. | Yes |
| Path-style لاسرسی | compatibility switch. اصلي off دی. ځینې MinIO، OpenList یا self-hosted S3 خدمتs یې غواړي. | No |
| Quota limit | کنټرولوي چې دا S3 چینل د capacity له مخې اپلوډ selection کې ګډون وکړي که نه. | No |
| Capacity limit | د کوټه limit له enable وروسته اړین، لکه `10 GB`. | اړین دی کله چې د کوټې محدودیت فعال وي |
| Threshold | کله چې usage دې percentage ته ورسېږي write بندوي، لکه `90%`. | اړین دی کله چې د کوټې محدودیت فعال وي |
| یادښت | د خپل maintenance لپاره یادښتونه. | No |

## Cloudflare R2 S3 API Keys جوړ کړئ

1. Cloudflare Dashboard پرانیزئ.
2. `R2 Object Storage` ته ولاړ شئ.
3. `Manage R2 API Tokens` یا API ټوکن management entry ومومئ.

![R2 API ټوکن entry](../../image/upload/s3/r2%20api%20token入口.png)

4. داسې R2 API ټوکن جوړ کړئ چې target bucket ته لاسرسی ولري.

![Create an حساب API ټوکن](../../image/upload/s3/创建账户api令牌.png)

5. generated `Access Key ID` او `Secret Access Key` کاپي کړئ.

![کاپي the ټوکن values](../../image/upload/s3/复制令牌值.png)

6. R2 bucket page ته بېرته ولاړ شئ او bucket name confirm کړئ.
7. د حساب لپاره S3 API endpoint ولیکئ. عموماً داسې ښکاري:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## د تنظیم ګامونه

1. د اپلوډ تنظیمات پرانیزئ.
2. چینل اضافه کړئ کلیک کړئ.
3. `S3` وټاکئ.
4. یو پېژندل کېدونکی چینل name ولیکئ، لکه `s3test`.
5. S3 API endpoint په `Endpoint` کې ولیکئ.
6. که custom CDN لرئ، په `Custom CDN domain` کې یې ولیکئ؛ که نه، خالي یې پرېږدئ.
7. bucket name ولیکئ.
8. region ولیکئ. د Cloudflare R2 مثال لپاره `auto` وکاروئ.
9. لاسرسی Key ID او Secret لاسرسی Key ولیکئ.
10. path-style لاسرسی off پرېږدئ، مګر provider یې صریحاً وغواړي.
11. که capacity control غواړئ، کوټه limit enable کړئ او capacity limit او threshold ولیکئ.
12. خوندي کلیک کړئ.

## FAQ

### path-style لاسرسی enable کړم؟

عموماً نه.

یوازې هغه وخت یې enable کړئ چې S3-compatible provider مو virtual-hosted-style URLs نه support کوي، یا provider documentation صریحاً path-style لاسرسی غواړي.

### CDN ډومېن د څه لپاره دی؟

که د bucket مخې ته CDN یا custom لاسرسی ډومېن لرئ، دلته یې ولیکئ.

له تنظیم وروسته جوړ شوي فایل لینکونه به دا ډومېن غوره کړي. که CDN نه کاروئ، فیلډ خالي پرېږدئ.

### ولې اپلوډ fail کېږي؟

لومړی دا وګورئ:

1. Endpoint د `https://` سره بشپړ URL لري.
2. bucket name سم دی.
3. Region د provider اړتیا سره برابر دی.
4. لاسرسی Key ID او Secret لاسرسی Key بشپړ کاپي شوي.
5. key د target bucket لپاره write permission لري.
6. provider path-style لاسرسی نه غواړي، یا که غواړي enabled دی.
