# S3 चैनल जोड़ना

## कब उपयुक्त है

S3 चैनल का उपयोग करें जब:

- आप किसी भी S3-संगत ऑब्जेक्ट स्टोरेज सेवा को जोड़ना चाहते हैं, जैसे Cloudflare R2, Backblaze B2, MinIO, AWS S3 या कोई अन्य संगत प्रदाता।
- आप मानक S3 कॉन्फ़िगरेशन मॉडल पसंद करते हैं: `Endpoint`, एक्सेस कुंजी और गोपनीय कुंजी।
- आप Cloudflare R2 के मूल बाइंडिंग चैनल का उपयोग नहीं करना चाहते, या आपका स्टोरेज प्रदाता Cloudflare R2 नहीं है।

## शुरू करने से पहले क्या चाहिए

| आवश्यकता | उद्देश्य |
| --- | --- |
| S3 एंडपॉइंट | ऑब्जेक्ट स्टोरेज सेवा का S3 API एंडपॉइंट। |
| बकेट नाम | वह बकेट जहाँ फ़ाइलें सहेजी जाएँगी। |
| क्षेत्र | स्टोरेज क्षेत्र। Cloudflare R2 के लिए यह आम तौर पर `auto` होता है। |
| Access Key ID | S3 एक्सेस कुंजी ID। |
| Secret Access Key | S3 गोपनीय एक्सेस कुंजी। |
| CDN डोमेन | फ़ाइलों तक पहुँच के लिए वैकल्पिक कस्टम डोमेन। |

S3 API के माध्यम से Cloudflare R2 का उदाहरण:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## कहाँ जोड़ें

1. सिस्टम सेटिंग्स खोलें।
2. अपलोड सेटिंग्स पर जाएँ।
3. ऊपर दाएँ कोने में चैनल जोड़ें पर क्लिक करें।
4. `S3` चुनें।

| चैनल सक्षम करें | तय करता है कि यह चैनल अपलोड चयन में शामिल होगा या नहीं। | सुझाया गया |

| बकेट नाम | बकेट नाम, जैसे `s3test` या `img-r2`। | हाँ |

| Access Key ID | S3 एक्सेस कुंजी ID। | हाँ |

| path-style एक्सेस | संगतता स्विच। डिफ़ॉल्ट रूप से बंद। कुछ MinIO, OpenList या स्वयं-होस्टेड S3 सेवाओं को इसकी आवश्यकता हो सकती है। | नहीं |
| कोटा सीमा | तय करती है कि यह S3 चैनल क्षमता के आधार पर अपलोड चयन में शामिल होगा या नहीं। | नहीं |
| क्षमता सीमा | कोटा सीमा सक्षम होने के बाद आवश्यक, उदाहरण के लिए `10 GB`। | कोटा सीमा सक्षम हो तो आवश्यक |
| थ्रेशोल्ड | उपयोग इस प्रतिशत तक पहुँचने पर लिखना रोकता है, उदाहरण के लिए `90%`। | कोटा सीमा सक्षम हो तो आवश्यक |
| टिप्पणी | अपने रखरखाव के लिए नोट्स। | नहीं |

## Cloudflare R2 के लिए S3 API कुंजियाँ बनाना

1. Cloudflare Dashboard खोलें।
2. `R2 Object Storage` पर जाएँ।
3. `Manage R2 API Tokens` या API टोकन प्रबंधन प्रविष्टि खोजें।

![R2 API टोकन प्रविष्टि](../../image/upload/s3/r2%20api%20token入口.png)

4. ऐसा R2 API टोकन बनाएँ जो लक्षित बकेट तक पहुँच सके।

![खाता API टोकन बनाएँ](../../image/upload/s3/创建账户api令牌.png)

5. बनाए गए `Access Key ID` और `Secret Access Key` मानों को कॉपी करें।

![टोकन मान कॉपी करें](../../image/upload/s3/复制令牌值.png)

6. R2 बकेट पेज पर लौटें और बकेट नाम की पुष्टि करें।
7. खाते का S3 API एंडपॉइंट नोट करें। यह आम तौर पर ऐसा दिखता है:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## कॉन्फ़िगरेशन चरण

1. अपलोड सेटिंग्स खोलें।
2. चैनल जोड़ें पर क्लिक करें।
3. `S3` चुनें।
4. पहचानने योग्य चैनल नाम दर्ज करें, उदाहरण के लिए `s3test`।
5. S3 API एंडपॉइंट को `Endpoint` में दर्ज करें।
6. यदि आप कस्टम CDN उपयोग करते हैं, तो उसे कस्टम CDN डोमेन फ़ील्ड में दर्ज करें; अन्यथा फ़ील्ड खाली छोड़ दें।
7. बकेट नाम दर्ज करें।
8. क्षेत्र दर्ज करें। Cloudflare R2 उदाहरण के लिए `auto` उपयोग करें।
9. Access Key ID और Secret Access Key दर्ज करें।
10. path-style एक्सेस बंद रखें, जब तक आपका प्रदाता स्पष्ट रूप से इसकी माँग न करे।
11. यदि क्षमता नियंत्रण चाहिए, तो कोटा सीमा सक्षम करें और क्षमता सीमा तथा थ्रेशोल्ड दर्ज करें।
12. सहेजें पर क्लिक करें।

## FAQ

### क्या path-style एक्सेस सक्षम करना चाहिए?

आम तौर पर नहीं।

इसे केवल तब सक्षम करें जब आपका S3-संगत प्रदाता virtual-hosted-style URL का समर्थन नहीं करता, या प्रदाता का दस्तावेज़ स्पष्ट रूप से path-style एक्सेस माँगता है। कुछ MinIO, स्वयं-होस्टेड S3 और OpenList-संगत एंडपॉइंट में इसकी आवश्यकता हो सकती है।

### CDN डोमेन किसलिए है?

यदि आपने बकेट के आगे CDN या कस्टम एक्सेस डोमेन लगाया है, तो उसे यहाँ दर्ज करें।

सेट करने के बाद जनरेट किए गए फ़ाइल लिंक इस डोमेन को प्राथमिकता देंगे। यदि CDN उपयोग नहीं करते, तो फ़ील्ड खाली छोड़ दें।

### अपलोड क्यों विफल होता है?

पहले ये जाँचें:

1. Endpoint में `https://` सहित पूरा URL हो।
2. बकेट नाम सही हो।
3. Region प्रदाता की आवश्यकता से मेल खाता हो।
4. Access Key ID और Secret Access Key पूरी तरह कॉपी किए गए हों।
5. कुंजी के पास लक्षित बकेट में लिखने की अनुमति हो।
6. प्रदाता path-style एक्सेस नहीं माँगता, या यदि माँगता है तो path-style एक्सेस सक्षम हो।
