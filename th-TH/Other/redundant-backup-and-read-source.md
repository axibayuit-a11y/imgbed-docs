# การสำรองซ้ำซ้อนและการสลับแหล่งอ่าน

การสำรองซ้ำซ้อนจะเก็บสำเนาเพิ่มเติมของไฟล์ที่อัปโหลดแล้ว

ทั้งไฟล์หลักและไฟล์สำรองสามารถใช้เป็นแหล่งอ่านได้ สำหรับผู้เยี่ยมชมโดยทั่วไปจะไม่เห็นความแตกต่าง ความแตกต่างมีเพียงช่องทางพื้นที่จัดเก็บที่ให้บริการไฟล์

## การสำรองซ้ำซ้อนทำอะไรได้บ้าง

| ฟีเจอร์ | คำอธิบาย |
| --- | --- |
| เก็บสำเนาเพิ่มเติม | สำรองไฟล์ไปยังช่องทางอัปโหลดอื่น เพื่อลดความเสี่ยงจากความล้มเหลวของช่องทางเดียว |
| สลับแหล่งอ่าน | หลังสำรองสำเร็จ สลับการอ่านไฟล์ระหว่างช่องทางหลักและช่องทางสำรอง |
| สำรองไฟล์เดียว | สำรองไฟล์หนึ่งไฟล์จากหน้ารายละเอียดไฟล์ |
| สำรองแบบชุด | เลือกหลายไฟล์ในหน้าผู้ดูแลแล้วสำรองพร้อมกัน |
| สำรองซ้ำซ้อนทั่วทั้งระบบ | สำรองไฟล์ตามโฟลเดอร์จากการตั้งค่าอื่น |

## ทางเข้าการสำรองซ้ำซ้อน

เปิด:

```text
System Settings -> Other Settings -> Redundant Backup
```

![การสำรองซ้ำซ้อน](../../image/other/冗余备份截图.png)

ทางเข้านี้เหมาะที่สุดสำหรับเพิ่มการสำรองให้โฟลเดอร์หนึ่งหรือไฟล์ทั้งหมดแบบชุด

สามารถเลือกช่องทางสำรองด้วยตนเอง หรือเลือกการสลับอัตโนมัติแล้วให้ ImgBed ค้นหาช่องทางสำรองที่เหมาะสม

## สำรองจากรายละเอียดไฟล์

เปิดหน้ารายละเอียดไฟล์ในแผงผู้ดูแลแล้วคลิกสำรอง

![สำรองในรายละเอียดไฟล์](../../image/other/文件详情里文件备份.png)

เหมาะที่สุดสำหรับสำรองไฟล์สำคัญหนึ่งไฟล์ตามต้องการ

หลังสำรองสำเร็จ หน้ารายละเอียดไฟล์จะแสดงแหล่งอ่านที่ใช้ได้

## สำรองแบบชุดจากการเลือก

ในแผงผู้ดูแล เลือกหลายไฟล์แล้วเรียกใช้การสำรองแบบชุด

![สำรองแบบชุด](../../image/other/批量备份截图.png)

เหมาะที่สุดสำหรับประมวลผลกลุ่มไฟล์

การสำรองไฟล์ที่เลือก การสำรองจากรายละเอียดไฟล์ และการสำรองซ้ำซ้อนในการตั้งค่าอื่นใช้ระบบสำรองเดียวกัน ต่างกันเพียงจุดเข้าใช้งาน

## สลับแหล่งอ่านหลังสำรอง

หลังสำรองเสร็จ หน้ารายละเอียดไฟล์ให้คุณสลับแหล่งอ่านได้:

| แหล่งอ่าน | คำอธิบาย |
| --- | --- |
| ช่องทางหลัก | อ่านจากช่องทางอัปโหลดเดิม |
| ช่องทางสำรอง | อ่านจากช่องทางสำรอง |

![สลับแหล่งอ่านหลังสำรอง](../../image/other/备份成功切换读取源.png)

ผู้เยี่ยมชมไม่จำเป็นต้องรู้ว่าไฟล์ถูกให้บริการจากช่องทางหลักหรือช่องทางสำรอง

แหล่งอ่านที่คุณเลือกจะกลายเป็นแหล่งที่มีลำดับความสำคัญสำหรับการเข้าถึงไฟล์ครั้งต่อไป

## เมื่อใดที่การสำรองจะถูกข้าม

กรณีต่อไปนี้จะถูกข้ามระหว่างการสำรอง ไม่ใช่ข้อผิดพลาด

| กรณี | เหตุผลที่ถูกข้าม |
| --- | --- |
| มีไฟล์สำรองอยู่แล้ว | ไฟล์ที่มีไฟล์สำรองอยู่แล้วจะไม่ถูกสำรองซ้ำ |
| ช่องทางหลักและช่องทางสำรองเหมือนกัน | การสำรองจะมีความหมายก็ต่อเมื่อเก็บไว้ในช่องทางอื่น |
| ไม่มีช่องทางสำรองที่ใช้ได้ | ไม่มีช่องทางอื่นที่เหมาะสม |

สรุปคือ การสำรองต้องไปยังอีกช่องทางหนึ่ง และไฟล์ที่สำรองแล้วจะไม่ใช้พื้นที่เพิ่มซ้ำอีก

## ช่องทางหลักและช่องทางสำรอง

| ชื่อ | ความหมาย |
| --- | --- |
| ช่องทางหลัก | ช่องทางที่ใช้เมื่อไฟล์ถูกอัปโหลดครั้งแรก |
| ช่องทางสำรอง | ช่องทางที่เก็บสำเนาซ้ำซ้อน |
| แหล่งอ่านหลัก | ขณะนี้ไฟล์ถูกอ่านจากช่องทางหลัก |
| แหล่งอ่านสำรอง | ขณะนี้ไฟล์ถูกอ่านจากช่องทางสำรอง |

แหล่งอ่านหลักและแหล่งอ่านสำรองมีพฤติกรรมเหมือนกันสำหรับผู้ใช้

ตราบใดที่ไฟล์สำรองยังใช้ได้ รูปภาพ วิดีโอ และลิงก์ดาวน์โหลดจะยังทำงานต่อหลังสลับไปใช้แหล่งอ่านสำรอง

## เกิดอะไรขึ้นเมื่อไฟล์ถูกลบ

เมื่อไฟล์ถูกลบ ImgBed จะลบทั้งไฟล์หลักและไฟล์สำรอง
