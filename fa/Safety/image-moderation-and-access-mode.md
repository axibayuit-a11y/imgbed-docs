# بازبینی تصویر و حالت دسترسی

بازبینی تصویر به تصاویر آپلودشده رده‌بندی سنی اختصاص می‌دهد. حالت دسترسی کنترل می‌کند کدام رده‌بندی‌ها از طریق دسترسی عمومی قابل مشاهده باشند.

این موضوع روی گالری عمومی، URLهای عمومی فایل‌ها و API تصویر تصادفی اثر می‌گذارد. پنل مدیریت را محدود نمی‌کند؛ مدیران همچنان می‌توانند همه فایل‌ها را ببینند و مدیریت کنند.

## محل پیکربندی

پنل مدیریت را باز کنید و به مسیر زیر بروید:

```text
System Settings -> Security Settings -> Upload Management -> Image Moderation
```

تنظیمات اصلی:

- حالت دسترسی
- فعال کردن بازبینی
- ارائه‌دهنده بازبینی

## حالت دسترسی چه کاری انجام می‌دهد

حالت دسترسی مشخص می‌کند کدام رده‌بندی‌های سنی می‌توانند به‌صورت عمومی نمایش داده شوند.

حالت‌های فعلی:

| حالت دسترسی | رده‌بندی‌های قابل مشاهده برای عموم |
| --- | --- |
| حالت بزرگسال | عمومی، R12، R16، R18 |
| حالت جوانان | عمومی، R12، R16 |
| حالت نوجوانان | عمومی، R12 |
| حالت کودکان | فقط عمومی |

حالت پیش‌فرض، حالت بزرگسال است.

برای سایت‌های خصوصی یا سایت‌هایی با محتوای بزرگسال، حالت بزرگسال می‌تواند مناسب باشد. برای گالری عمومی محتاطانه‌تر، حالت جوانان، نوجوانان یا کودکان را انتخاب کنید.

## فعال کردن بازبینی چه کاری انجام می‌دهد

وقتی بازبینی فعال باشد، ImgBed هنگام آپلود ارائه‌دهنده بازبینی انتخاب‌شده را فراخوانی می‌کند و رده‌بندی سنی تشخیص‌داده‌شده را ذخیره می‌کند.

رده‌بندی‌های اصلی:

| رده‌بندی | معنی |
| --- | --- |
| عمومی | محتوای عمومی امن |
| R12 | محتوای کمی حساس |
| R16 | محتوای نسبتاً حساس |
| R18 | محتوای بزرگسال |

نتیجه بازبینی هنگام تصمیم‌گیری درباره دسترسی عمومی استفاده می‌شود.

اگر بازبینی فعال نباشد، یا فایل‌های قدیمی رده‌بندی نداشته باشند، آن فایل‌ها بدون رده‌بندی در نظر گرفته می‌شوند. فایل‌های بدون رده‌بندی فقط به دلیل نداشتن نتیجه بازبینی به‌صورت خودکار از گالری عمومی یا API تصویر تصادفی حذف نمی‌شوند.

## انتخاب ارائه‌دهنده بازبینی

ارائه‌دهندگان در دسترس شامل این موارد هستند:

- moderatecontent.com
- nsfwjs
- Sightengine

هر ارائه‌دهنده نیازمندی‌های متفاوتی دارد:

- moderatecontent.com معمولاً به API Key نیاز دارد.
- nsfwjs معمولاً به URL نقطه پایانی API نیاز دارد.
- Sightengine به API user و API secret نیاز دارد.

بر اساس حساب، دسترس‌پذیری و کیفیت تشخیص انتخاب کنید. تا زمانی که بازبینی فعال و درست پیکربندی شده باشد، ImgBed تلاش می‌کند هنگام آپلود رده‌بندی تصویر را ثبت کند.

## اثر روی گالری عمومی

گالری عمومی فایل‌ها را بر اساس حالت دسترسی فیلتر می‌کند.

نمونه‌ها:

- حالت بزرگسال: تصاویر R18 می‌توانند نمایش داده شوند.
- حالت جوانان: تصاویر R18 پنهان می‌شوند.
- حالت نوجوانان: تصاویر R16 و R18 پنهان می‌شوند.
- حالت کودکان: فقط تصاویر با رده‌بندی عمومی نمایش داده می‌شوند.

این فقط روی دسترسی عمومی عادی اثر دارد. پنل مدیریت همچنان همه فایل‌ها را نشان می‌دهد.

## اثر روی URLهای عمومی فایل

URLهای عمومی فایل، لینک‌های مستقیم تصویر هستند که بازدیدکنندگان باز می‌کنند.

اگر رده‌بندی فایل در حالت دسترسی فعلی مجاز باشد، ImgBed تصویر اصلی را برمی‌گرداند.

اگر رده‌بندی بالاتر از سطح مجاز باشد، دسترسی عمومی عادی تصویر اصلی را برنمی‌گرداند. در عوض، ImgBed نتیجه مسدودشده پیکربندی‌شده یا تصویر جایگزین را برمی‌گرداند.

نمونه:

- حالت فعلی، حالت کودکان است.
- تصویر رده‌بندی R18 دارد.
- بازدیدکننده URL عمومی را مستقیم باز می‌کند.
- ImgBed تصویر اصلی R18 را به آن بازدیدکننده برنمی‌گرداند.

![تصویر فایل محدودشده](../../image/Safety/文件受限图.png)

مدیرانی که فایل‌ها را در پنل مدیریت می‌بینند تحت تأثیر این محدودیت نیستند.

## اثر روی API تصویر تصادفی

API تصویر تصادفی نیز مجموعه نامزدهای خود را بر اساس حالت دسترسی فیلتر می‌کند.

در حالت کودکان، تصاویر تصادفی فقط از فایل‌های دارای رده‌بندی عمومی انتخاب می‌شوند.

در حالت جوانان، تصاویر تصادفی می‌توانند از فایل‌های عمومی، R12 و R16 باشند، اما از فایل‌های R18 نه.

این کار از دور زدن محدودیت‌های گالری عمومی توسط API تصویر تصادفی جلوگیری می‌کند.

## رابطه با قوانین فهرست‌ها

حالت دسترسی تنها قانون دسترسی عمومی نیست. این حالت همراه با قوانین فهرست مجاز و فهرست مسدود کار می‌کند.

به‌طور ساده:

- محتوای فهرست مجاز ابتدا عمومی است.
- محتوای فهرست مسدود را بازدیدکنندگان عادی نمی‌توانند مستقیم ببینند.
- محتوایی که در هیچ‌کدام از دو فهرست نیست، سپس بر اساس حالت دسترسی بررسی می‌شود.

اگر تصویر هم از نظر رده‌بندی سنی و هم بر اساس قوانین فهرست‌ها محدود شده باشد، بازدیدکنندگان عادی همچنان نمی‌توانند فایل اصلی را مستقیم ببینند.

## تنظیمات پیشنهادی

برای سایت‌های عمومی:

- بازبینی را فعال کنید.
- حالت دسترسی متناسب با مخاطبان سایت را انتخاب کنید.
- برای بازدیدکنندگان همه سنین، از حالت کودکان یا حالت نوجوانان استفاده کنید.
- اگر نمی‌خواهید محتوای بزرگسال به‌صورت عمومی نمایش داده شود، از حالت بزرگسال پرهیز کنید.
- رده‌بندی فایل‌ها را در پنل مدیریت بررسی کنید و در صورت نیاز دستی تنظیم کنید.

برای سایت‌های خصوصی یا شخصی:

- حالت بزرگسال معمولاً مناسب است.
- اگر مفید است، بازبینی را فعال کنید.
- رده‌بندی‌ها را در پنل مدیریت بررسی و در صورت نیاز تنظیم کنید.

## پرسش‌های متداول

### آیا پس از تغییر حالت دسترسی فایل‌ها از پنل مدیریت ناپدید می‌شوند؟

خیر.

حالت دسترسی فقط روی دسترسی عمومی عادی اثر دارد. روی پنل مدیریت اثر ندارد.

### چرا پس از تغییر به حالت کودکان، گالری عمومی تصاویر کمتری نشان می‌دهد؟

حالت کودکان فقط اجازه می‌دهد فایل‌های دارای رده‌بندی عمومی به‌صورت عمومی نمایش داده شوند. فایل‌های R12، R16 و R18 فیلتر می‌شوند.

### آیا URLهای عمومی هنوز می‌توانند تصاویر بزرگسال را باز کنند؟

اگر حالت دسترسی فعلی آن رده‌بندی را مجاز نداند، URLهای عمومی عادی تصویر اصلی را برنمی‌گردانند.

### آیا API تصویر تصادفی می‌تواند تصاویر محدودشده برگرداند؟

خیر.

API تصویر تصادفی نامزدها را بر اساس حالت دسترسی فعلی فیلتر می‌کند.

### برای تصاویر قدیمی بدون رده‌بندی چه اتفاقی می‌افتد؟

تصاویر بدون رده‌بندی فقط به دلیل نداشتن نتیجه بازبینی به‌صورت خودکار پنهان نمی‌شوند. می‌توانید بعداً رده‌بندی آن‌ها را در پنل مدیریت تنظیم کنید.
