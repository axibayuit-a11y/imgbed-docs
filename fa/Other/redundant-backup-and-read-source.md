# پشتیبان‌گیری افزون و تغییر منبع خواندن

پشتیبان‌گیری افزون یک کپی اضافی از فایلی که قبلاً بارگذاری شده ذخیره می‌کند.

هم فایل اصلی و هم فایل پشتیبان می‌توانند به‌عنوان منبع خواندن استفاده شوند. بازدیدکنندگان معمولاً تفاوتی نمی‌بینند. تنها تفاوت این است که فایل از کدام کانال ذخیره‌سازی ارائه می‌شود.

## پشتیبان‌گیری افزون چه کارهایی انجام می‌دهد

| قابلیت | توضیح |
| --- | --- |
| ذخیره کپی اضافی | از فایل‌ها در یک کانال بارگذاری دیگر پشتیبان می‌گیرد تا خطر خرابی یک کانال کاهش یابد. |
| تغییر منبع خواندن | پس از موفقیت پشتیبان‌گیری، امکان جابه‌جایی خواندن فایل بین کانال اصلی و کانال پشتیبان را فراهم می‌کند. |
| پشتیبان‌گیری از یک فایل | از صفحه جزئیات فایل، از یک فایل پشتیبان می‌گیرد. |
| پشتیبان‌گیری دسته‌ای | با انتخاب چند فایل در صفحه مدیریت، از آن‌ها به‌صورت دسته‌ای پشتیبان می‌گیرد. |
| پشتیبان‌گیری افزون سراسری | از فایل‌ها بر اساس پوشه و از بخش تنظیمات دیگر پشتیبان می‌گیرد. |

## ورودی پشتیبان‌گیری افزون

باز کنید:

```text
System Settings -> Other Settings -> Redundant Backup
```

![پشتیبان‌گیری افزون](../../image/other/冗余备份截图.png)

این ورودی برای افزودن پشتیبان به یک پوشه یا همه فایل‌ها به‌صورت دسته‌ای مناسب‌تر است.

کانال پشتیبان را می‌توان دستی انتخاب کرد، یا می‌توانید جابه‌جایی خودکار را انتخاب کنید تا ImgBed یک کانال پشتیبان مناسب پیدا کند.

## پشتیبان‌گیری از جزئیات فایل

یک صفحه جزئیات فایل را در پنل مدیریت باز کنید و روی پشتیبان‌گیری کلیک کنید.

![پشتیبان‌گیری در جزئیات فایل](../../image/other/文件详情里文件备份.png)

این روش برای پشتیبان‌گیری از یک فایل مهم در زمان نیاز مناسب‌تر است.

پس از موفقیت پشتیبان‌گیری، صفحه جزئیات فایل منابع خواندن موجود را نشان می‌دهد.

## پشتیبان‌گیری دسته‌ای با انتخاب

در پنل مدیریت، چند فایل را انتخاب کنید و پشتیبان‌گیری دسته‌ای را اجرا کنید.

![پشتیبان‌گیری دسته‌ای](../../image/other/批量备份截图.png)

این روش برای پردازش گروهی از فایل‌ها مناسب‌تر است.

پشتیبان‌گیری با انتخاب، پشتیبان‌گیری از جزئیات فایل و پشتیبان‌گیری افزون در تنظیمات دیگر، همگی از یک سامانه پشتیبان‌گیری استفاده می‌کنند. فقط نقطه ورود آن‌ها متفاوت است.

## تغییر منبع خواندن پس از پشتیبان‌گیری

پس از کامل شدن پشتیبان‌گیری، صفحه جزئیات فایل اجازه می‌دهد منبع خواندن را تغییر دهید:

| منبع خواندن | توضیح |
| --- | --- |
| کانال اصلی | از کانال بارگذاری اصلی می‌خواند. |
| کانال پشتیبان | از کانال پشتیبان می‌خواند. |

![تغییر منبع خواندن پس از پشتیبان‌گیری](../../image/other/备份成功切换读取源.png)

بازدیدکنندگان لازم نیست بدانند فایل از کانال اصلی ارائه می‌شود یا از کانال پشتیبان.

هر منبع خواندنی که انتخاب کنید، برای دسترسی‌های بعدی فایل به منبع ترجیحی تبدیل می‌شود.

## چه زمانی پشتیبان‌گیری رد می‌شود

موارد زیر هنگام پشتیبان‌گیری رد می‌شوند. این‌ها خطا نیستند.

| مورد | دلیل رد شدن |
| --- | --- |
| قبلاً پشتیبان‌گیری شده است | فایلی که از قبل پشتیبان دارد دوباره پشتیبان‌گیری نمی‌شود. |
| کانال اصلی و پشتیبان یکسان هستند | برای معنادار بودن پشتیبان، کپی باید در کانال دیگری ذخیره شود. |
| کانال پشتیبان قابل استفاده وجود ندارد | کانال جایگزین مناسبی در دسترس نیست. |

خلاصه: پشتیبان‌ها باید در کانال دیگری ذخیره شوند، و فایل‌هایی که قبلاً پشتیبان دارند دوباره فضای اضافی مصرف نمی‌کنند.

## کانال اصلی در برابر کانال پشتیبان

| نام | معنی |
| --- | --- |
| کانال اصلی | کانالی که هنگام نخستین بارگذاری فایل استفاده شده است. |
| کانال پشتیبان | کانالی که کپی افزون را ذخیره می‌کند. |
| منبع خواندن اصلی | فایل در حال حاضر از کانال اصلی خوانده می‌شود. |
| منبع خواندن پشتیبان | فایل در حال حاضر از کانال پشتیبان خوانده می‌شود. |

منابع خواندن اصلی و پشتیبان از دید کاربر رفتار یکسانی دارند.

تا زمانی که فایل پشتیبان در دسترس باشد، تصاویر، ویدئوها و پیوندهای دانلود پس از تغییر به منبع خواندن پشتیبان همچنان کار می‌کنند.

## هنگام حذف فایل چه می‌شود

وقتی فایلی حذف می‌شود، ImgBed هم فایل اصلی و هم فایل پشتیبان را حذف می‌کند.
