# S3 চ্যানেল যোগ করা

## কখন সবচেয়ে উপযোগী

S3 চ্যানেল ব্যবহার করুন যখন:

- Cloudflare R2, Backblaze B2, MinIO, AWS S3 অথবা অন্য কোনো S3-compatible অবজেক্ট স্টোরেজ পরিষেবা যুক্ত করতে চান।
- স্ট্যান্ডার্ড S3 কনফিগারেশন মডেল পছন্দ করেন: `Endpoint`, Access Key এবং Secret Key।
- নেটিভ Cloudflare R2 binding চ্যানেল ব্যবহার করতে চান না, অথবা আপনার স্টোরেজ প্রদানকারী Cloudflare R2 নয়।

## শুরু করার আগে যা লাগবে

| প্রয়োজনীয় তথ্য | উদ্দেশ্য |
| --- | --- |
| S3 Endpoint | অবজেক্ট স্টোরেজ পরিষেবার S3 API endpoint। |
| Bucket name | ফাইল যেখানে সংরক্ষিত হবে সেই বাকেট। |
| Region | সংরক্ষণ অঞ্চল। Cloudflare R2-এর জন্য সাধারণত `auto`। |
| Access Key ID | S3 access key ID। |
| Secret Access Key | S3 secret access key। |
| CDN ডোমেইন | ফাইল অ্যাক্সেসের জন্য ঐচ্ছিক কাস্টম ডোমেইন। |

Cloudflare R2-কে S3 API দিয়ে ব্যবহারের উদাহরণ:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## কোথায় যোগ করবেন

1. সিস্টেম সেটিংস খুলুন।
2. আপলোড সেটিংসে যান।
3. উপরের ডান পাশে চ্যানেল যোগ করুন ক্লিক করুন।
4. `S3` নির্বাচন করুন।

## ক্ষেত্রের বিবরণ

| ক্ষেত্র | কাজ | প্রয়োজনীয় |
| --- | --- | --- |
| চ্যানেলের নাম | এই S3 চ্যানেলের সহজে চেনা যায় এমন নাম, যেমন `s3test` বা `R2-S3`। | হ্যাঁ |
| চ্যানেল চালু করুন | এই চ্যানেল আপলোড নির্বাচনে অংশ নেবে কি না নিয়ন্ত্রণ করে। | প্রস্তাবিত |
| Endpoint | `https://`. পূর্ণ S3 পরিষেবা endpoint। | হ্যাঁ |
| Custom CDN domain | ঐচ্ছিক। সেট করা হলে তৈরি হওয়া ফাইল লিংকে এই ডোমেইন-কে অগ্রাধিকার দেওয়া হবে। | না |
| Bucket name | বাকেট নাম, যেমন `s3test` বা `img-r2`। | হ্যাঁ |
| Bucket region | অঞ্চল। Cloudflare R2-এর জন্য সাধারণত `auto`। | হ্যাঁ |
| Access Key ID | S3 access key ID। | হ্যাঁ |
| Secret Access Key | S3 secret access key। | হ্যাঁ |
| Path-style access | সামঞ্জস্যতার সুইচ। ডিফল্টভাবে বন্ধ। কিছু MinIO, OpenList অথবা self-hosted S3 পরিষেবায় এটি দরকার হতে পারে। | না |
| Quota limit | ধারণক্ষমতার ভিত্তিতে এই S3 চ্যানেল আপলোড নির্বাচনে অংশ নেবে কি না নিয়ন্ত্রণ করে। | না |
| Capacity limit | Quota limit চালু করার পর প্রয়োজনীয়, যেমন `10 GB`। | Quota limit চালু থাকলে প্রয়োজনীয় |
| Threshold | ব্যবহার এই শতাংশে পৌঁছালে লেখা বন্ধ করে, যেমন `90%`। | Quota limit চালু থাকলে প্রয়োজনীয় |
| Remark | নিজস্ব রক্ষণাবেক্ষণের নোট। | না |

## Cloudflare R2 S3 API Keys তৈরি করা

1. Cloudflare ড্যাশবোর্ড খুলুন।
2. `R2 Object Storage`-এ যান।
3. `Manage R2 API Tokens` অথবা API টোকেন ব্যবস্থাপনার এন্ট্রি খুঁজুন।

![R2 API টোকেন এন্ট্রি](../../image/upload/s3/r2%20api%20token入口.png)

4. লক্ষ্য বাকেট-এ প্রবেশ করতে পারে এমন R2 API টোকেন তৈরি করুন।

![অ্যাকাউন্ট API টোকেন তৈরি করা](../../image/upload/s3/创建账户api令牌.png)

5. তৈরি হওয়া `Access Key ID` এবং `Secret Access Key` কপি করুন।

![টোকেনের মান কপি করা](../../image/upload/s3/复制令牌值.png)

6. R2 বাকেট পৃষ্ঠায় ফিরে বাকেট নাম নিশ্চিত করুন।
7. অ্যাকাউন্টের S3 API endpoint লিখে রাখুন। সাধারণত এটি এমন দেখায়:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## সেটআপ ধাপ

1. আপলোড সেটিংস খুলুন।
2. চ্যানেল যোগ করুন ক্লিক করুন।
3. `S3` নির্বাচন করুন।
4. সহজে চেনা যায় এমন চ্যানেলের নাম দিন, যেমন `s3test`।
5. `Endpoint`-এ S3 API endpoint দিন।
6. কাস্টম CDN ব্যবহার করলে `Custom CDN domain`-এ দিন; না হলে খালি রাখুন।
7. বাকেট নাম দিন।
8. Region দিন। Cloudflare R2 উদাহরণের ক্ষেত্রে `auto` ব্যবহার করুন।
9. Access Key ID এবং Secret Access Key দিন।
10. প্রদানকারী স্পষ্টভাবে না চাইলে path-style access বন্ধ রাখুন।
11. ধারণক্ষমতা নিয়ন্ত্রণ চাইলে quota limit চালু করে capacity limit এবং threshold দিন।
12. সংরক্ষণ ক্লিক করুন।

| পরীক্ষা | কীভাবে যাচাই করবেন |

| চ্যানেল চালু আছে | কার্ডের উপরের ডান পাশের সুইচ চালু থাকা উচিত। |
| মূল ক্ষেত্রগুলো সংরক্ষিত হয়েছে | বিস্তারিত দৃশ্যে Endpoint, Bucket, Region, path-style access এবং সংশ্লিষ্ট ক্ষেত্রগুলো দেখা উচিত। |
| আপলোড কাজ করছে | একটি পরীক্ষামূলক ছবি আপলোড করে লক্ষ্য বাকেট-এ অবজেক্ট এসেছে কি না নিশ্চিত করুন। |
| লিংক খোলে | আপলোডের পর ফেরত পাওয়া ছবির লিংক স্বাভাবিকভাবে খোলা উচিত। |
| ধারণক্ষমতা প্রদর্শন কাজ করছে | Quota limit চালু থাকলে কার্ডে ব্যবহৃত ধারণক্ষমতা এবং সেট করা সীমা দেখা উচিত। |

S3 ধারণক্ষমতার পরিসংখ্যান প্রদানকারীর বাকেট real time-এ কোয়েরি করে নয়, ImgBed-এর local file records থেকে হিসাব করা হয়। S3 কনফিগারেশন সংরক্ষণের পর সিস্টেম বর্তমান D1 records থেকে quota ledger আবার হিসাব করে।

## FAQ

### Path-style access চালু করব?

সাধারণত নয়।

শুধু তখনই চালু করুন যখন আপনার S3-compatible প্রদানকারী virtual-hosted-style URLs সমর্থন করে না, অথবা প্রদানকারীর documentation স্পষ্টভাবে path-style access চাইছে। কিছু MinIO, self-hosted S3 এবং OpenList-compatible endpoint-এ এটি লাগতে পারে।

### CDN ডোমেইন কী কাজে লাগে?

বাকেট-এর সামনে CDN বা কাস্টম access ডোমেইন থাকলে এখানে দিন।

সেট করার পর তৈরি হওয়া ফাইল লিংকে এই ডোমেইন-কে অগ্রাধিকার দেওয়া হবে। CDN ব্যবহার না করলে ক্ষেত্রটি খালি রাখুন।

### আপলোড ব্যর্থ হলে কী দেখব?

আগে এগুলো পরীক্ষা করুন:

1. Endpoint পূর্ণ `https://` URL-সহ আছে।
2. বাকেট নাম সঠিক।
3. Region প্রদানকারীর শর্তের সঙ্গে মেলে।
4. Access Key ID এবং Secret Access Key সম্পূর্ণভাবে কপি করা হয়েছে।
5. key-এর লক্ষ্য বাকেটে লেখার অনুমতি আছে।
6. প্রদানকারী path-style access চায় না, অথবা প্রয়োজন হলে path-style access চালু করা হয়েছে।
