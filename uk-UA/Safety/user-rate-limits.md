# Обмеження частоти завантажень для користувачів

User rate limits визначають, як часто звичайні користувачі або відвідувачі можуть завантажувати файли з головної сторінки. Це допомагає захистити публічну сторінку завантаження від зловживань.

Функція впливає лише на завантаження з головної сторінки. Завантаження адміністратора та завантаження через API Tokens не обмежуються user rate limits.

## Де це налаштовується

Відкрийте адмін-панель і перейдіть до:

```text
System Settings -> Security Settings -> Upload Management -> User Rate Limits
```

![Налаштування user rate limit](../../image/other/用户频控截图.png)

## Увімкнення rate limits

Після ввімкнення `Enable Rate Limits` ImgBed відстежує нещодавні завантаження за IP-адресою того, хто завантажує.

Типові значення:

| Налаштування | Типово | Опис |
| --- | --- | --- |
| Detection window | 1.5 hours | Наскільки далеко назад рахуються записи завантажень. |
| Max file count | 20 | Максимальна кількість файлів у detection window. |
| Single file size limit | 20 MB | Максимальний розмір одного файлу. |
| Total upload size limit | 200 MB | Максимальний загальний розмір завантажень у detection window. |

Наприклад, із вікном 1,5 години, 20 файлами, 20 MB на файл і 200 MB загалом завантаження з тієї самої IP-адреси блокуються, щойно перевищено будь-який із налаштованих лімітів.

## Виключення типів файлів

`Excluded upload file types` блокує звичайним користувачам або відвідувачам завантаження вибраних категорій файлів.

Доступні категорії:

| Тип | Опис |
| --- | --- |
| Images | jpg, png, webp, gif та подібні файли зображень |
| Videos | mp4, webm, mov та подібні відеофайли |
| Audio | mp3, flac, wav та подібні аудіофайли |
| Documents | pdf, txt, md, docx та подібні документи |
| Other | Файли поза наведеними категоріями, наприклад zip, rar, exe, apk |

Типово категорія не вибрана, тобто вона дозволена.

Натискання на категорію підсвічує її, а це означає, що цей тип заблоковано.

Якщо вибрати `Other`, відвідувачі, які завантажують zip або rar, будуть заблоковані й побачать повідомлення, що цей тип файлу не підтримується.

## Повідомлення про блокування

Коли спрацьовує ліміт, користувачі бачать відповідне повідомлення:

![Повідомлення про занадто часте завантаження](../../image/other/频繁报错提示.png)

| Сценарій | Значення повідомлення |
| --- | --- |
| Один файл завеликий | Файл завеликий, його варто стиснути перед завантаженням. |
| Тип файлу заблоковано | Цей тип файлу не підтримується. Приберіть його й спробуйте ще раз. |
| Завантаження занадто часті | Нещодавні завантаження занадто часті, показано час повторної спроби. |
| Загальний розмір завеликий | Сумарний розмір нещодавніх завантажень занадто великий, показано час повторної спроби. |

## Коли вмикати

Увімкніть user rate limits, якщо ваша головна сторінка завантаження доступна публічно.

Типові причини:

- ви хвилюєтеся щодо масових автоматизованих завантажень;
- хочете обмежити великі завантаження від відвідувачів;
- хочете дозволити звичайним користувачам завантажувати лише зображення, а не архіви чи інсталятори;
- хочете залишити публічне завантаження доступним, але контролювати використання ресурсів.

Якщо сайт тільки для вас або завантажувати можуть лише адміністратори, це можна залишити вимкненим.
