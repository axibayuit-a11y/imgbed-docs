# Cloudflare API Token

Облікові дані Cloudflare API дозволяють ImgBed очищати кеш Cloudflare CDN після змін у файлах.

![Налаштування Cloudflare API Token](../../image/Safety/cloudflare%20api%20token截图.png)

## Де це налаштовується

Відкрийте адмін-панель і перейдіть до:

```text
System Settings -> Security Settings -> Cloudflare API Token
```

Потрібно заповнити:

- Zone ID
- Account email
- API Key

## Що робить це налаштування

Cloudflare може кешувати публічні URL зображень.

Кеш прискорює доставку зображень, але після видалення, блокування, заміни або переміщення файлу старий вміст може ще деякий час залишатися видимим.

Після налаштування облікових даних Cloudflare API ImgBed намагається очистити пов'язаний кеш Cloudflare, коли такі операції завершуються.

Це корисно, якщо:

- ви видаляєте зображення й хочете, щоб публічне посилання перестало працювати якомога швидше;
- ви блокуєте зображення й хочете, щоб відвідувачі більше не бачили оригінальний файл;
- ви замінюєте файл із тією самою назвою й хочете швидше показати відвідувачам нову версію;
- ви переміщуєте або перейменовуєте файли й хочете швидко оновити кеш старих шляхів;
- ви змінюєте правила публічного доступу й хочете швидше оновити кеш публічної галереї або random image API.

## Що буде, якщо залишити порожнім

ImgBed і далі працюватиме нормально без цього налаштування.

Єдина різниця: ImgBed не буде активно очищати кеш Cloudflare CDN. Відвідувачі можуть бачити старий вміст, доки кеш Cloudflare не завершиться природно.

## Як знайти Zone ID

Zone ID — це Cloudflare Zone ID сайту, який використовується вашим доменом ImgBed.

1. Увійдіть у Cloudflare dashboard.
2. Відкрийте сайт, у якому знаходиться ваш домен ImgBed.
3. Знайдіть `Zone ID` на overview-сторінці сайту.
4. Скопіюйте його в поле `Zone ID` в ImgBed.

Це саме Zone ID сайту, а не account ID.

## Account Email

Введіть email, який використовуєте для входу в Cloudflare.

Він має відповідати API Key, який ви вкажете нижче.

## API Key

Введіть Cloudflare Global API Key.

1. Увійдіть у Cloudflare dashboard.
2. Відкрийте свій profile.
3. Перейдіть на сторінку API Tokens.
4. Знайдіть `Global API Key`.
5. Перегляньте й скопіюйте його.
6. Вставте його в поле `API Key` в ImgBed.

![Перегляд global API key](../../image/Safety/查看全局令牌.png)

## Коли це починає працювати

Після заповнення полів збережіть налаштування.

Майбутні зміни файлів автоматично намагатимуться очистити кеш Cloudflare. Минулі операції не очищаються заднім числом. Якщо файл було видалено або замінено до цього налаштування, дочекайтеся завершення кешу Cloudflare або очистьте його вручну в Cloudflare.

## FAQ

### Це обов'язково?

Ні.

Якщо ваш домен не використовує Cloudflare або вас не турбує затримка CDN-кешу, поле можна залишити порожнім.

### Неправильні облікові дані зламають завантаження?

Зазвичай ні.

Неправильні облікові дані лише завадять ImgBed очищати кеш Cloudflare. Завантаження й звичайний доступ до файлів мають працювати далі.

### Чому видалене зображення все ще відкривається?

Найчастіша причина — Cloudflare усе ще має старий файл у кеші.

З правильними обліковими даними Cloudflare API ImgBed очищає кеш пов'язаного URL під час видалення файлу.

### Чому після заміни файлу я все ще бачу старе зображення?

Це також зазвичай спричинено CDN-кешем.

Після налаштування цього параметра ImgBed намагається очистити кеш старого URL, коли файл із тією самою назвою перезаписується.
