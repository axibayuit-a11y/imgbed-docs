# WebDAV-доступ до сайту (Beta)

Налаштування WebDAV у Security Settings відкриває ваш сайт ImgBed як WebDAV endpoint.

Після ввімкнення можна використовувати Windows, macOS, мобільні файлові менеджери або будь-який WebDAV-сумісний клієнт, щоб переглядати, завантажувати, видаляти й керувати файлами ImgBed як віддаленою папкою.

Це вхід WebDAV-доступу саме до сайту. Він відрізняється від WebDAV storage channel у налаштуваннях завантаження. Канал завантаження зберігає файли в сторонньому WebDAV-сервісі. Це налаштування дозволяє вашому сайту ImgBed надавати WebDAV-доступ клієнтам.

## Де це налаштовується

Відкрийте адмін-панель і перейдіть до:

```text
System Settings -> Security Settings -> WebDAV
```

Доступні налаштування:

- Enable
- Username
- Password
- Image loading mode
- Default channel

## Що робить ця функція

Після ввімкнення WebDAV ImgBed надає фіксований URL доступу:

```text
https://your-domain.com/dav
```

Використовуйте цей URL для підключення до файлової директорії ImgBed.

Корисні сценарії:

- переглядати файли ImgBed напряму з файлового менеджера комп'ютера;
- перетягувати зображення в WebDAV-папку для завантаження;
- упорядковувати папки ImgBed з локального файлового менеджера;
- використовувати WebDAV-сумісне ПЗ для синхронізації або керування зображеннями;
- отримувати доступ до вмісту ImgBed без відкриття адмін-панелі.

## Налаштування

### Enable

Увімкнення WebDAV endpoint.

Коли вимкнено, клієнти не можуть підключитися через WebDAV.

### Username і Password

Ці облікові дані використовуються WebDAV-клієнтами під час підключення.

Використовуйте окремі WebDAV username і password. Не використовуйте повторно admin password або upload password.

Якщо username або password порожні, WebDAV-клієнти не зможуть підключатися коректно.

### Image Loading Mode

Image loading mode визначає, який URL зображення WebDAV-клієнти віддають перевагу під час читання зображень.

Поширені варіанти:

| Mode | Опис |
| --- | --- |
| Smart loading | ImgBed вибирає на основі контексту. Рекомендовано для звичайного використання. |
| Original | Перевага оригінальним зображенням. |
| Thumbnail | Перевага мініатюрам. Корисно для швидкого попереднього перегляду. |

Якщо не впевнені, залиште `Smart loading`.

### Default Channel

Default channel використовується для WebDAV-завантажень.

Коли ви копіюєте файли в WebDAV-директорію з Windows або іншого клієнта, ImgBed завантажує їх через вибраний default upload channel.

Якщо default channel не вибраний, перегляд може працювати, але завантаження може падати.

## Доступ до WebDAV у Windows 11

Windows 11 може додати WebDAV як network location.

1. Відкрийте `This PC`.
2. Виберіть `Add a network location`.
3. Введіть `https://your-domain.com/dav`.
4. Коли з'явиться запит, введіть WebDAV username і password.
5. Завершіть майстер. Після цього WebDAV-директорію можна відкривати у File Explorer.

![Додавання WebDAV у Windows 11](../../image/Safety/webdav在win11配置.png)

Після додавання WebDAV-директорія з'явиться у Windows File Explorer. Можна відкривати, копіювати й керувати файлами як у звичайній папці.

## Підтримувані операції

Після успішного WebDAV-підключення зазвичай можна:

- переглядати файли й папки;
- завантажувати файли;
- створювати папки;
- перейменовувати файли або папки;
- переміщувати файли;
- видаляти файли.

WebDAV найкраще підходить для щоденного доступу й невеликого керування файлами. Для великих переміщень, масових видалень або складної організації краще використовувати адмін-панель.

## Login Device Management

Успішні WebDAV-підключення також з'являються у вкладці WebDAV в Login Device Management.

Там можна переглядати WebDAV-клієнти й за потреби примусово виводити старі пристрої офлайн.

Якщо змінити WebDAV username або password, старим клієнтам потрібно буде увійти знову.

## FAQ

### Windows постійно запитує username і password

Перевірте:

- URL має бути `https://your-domain.com/dav`.
- Username і password збігаються з налаштуваннями WebDAV.
- WebDAV увімкнений.
- Сайт доступний через HTTPS.

### Перегляд працює, але завантаження ні

Перевірте `Default channel`.

Для WebDAV-завантажень потрібен default upload channel. Якщо його немає, він вимкнений або неправильно налаштований, завантаження можуть не працювати.

### Швидкість доступу нестабільна

Продуктивність WebDAV залежить від клієнта, мережі, кількості файлів і default upload channel.

Якщо в директорії багато файлів, розкладіть їх по папках, а не тримайте надто багато файлів в одній директорії.

## Рекомендації з безпеки

- Використовуйте HTTPS для WebDAV-доступу.
- Встановіть надійний пароль.
- Не передавайте WebDAV password людям, яким не довіряєте.
- Вимикайте WebDAV, коли не користуєтеся ним.
- Періодично очищайте невикористані WebDAV devices у Login Device Management.

## Розмір файлів для WebDAV-завантаження

WebDAV-клієнти не використовують browser upload page's large-file chunking flow. Для файлів, більших за рекомендовані нижче ліміти, використовуйте вебсторінку завантаження.

| Default Upload Channel | Рекомендований ліміт одного файлу для WebDAV |
| --- | ---: |
| Telegram | 20 MB |
| Discord | 10 MB |
| Cloudflare R2 | 30 MB |
| S3 | 30 MB |
| GitHub Releases | 64 MB |
| GitLab Packages | 64 MB |
| Hugging Face | 20 MB |
| OneDrive | 30 MB |
| Google Drive | 30 MB |
| Dropbox | 30 MB |
| Yandex Disk | 30 MB |
| pCloud | 30 MB |
| WebDAV | 64 MB |
