# Модерація зображень і режим доступу

Модерація зображень присвоює завантаженим файлам вікові рейтинги. Режим доступу визначає, які рейтинги можна показувати через публічний доступ.

Це впливає на публічну галерею, публічні URL файлів і random image API. На адмін-панель обмеження не поширюються: адміністратори все одно можуть переглядати й керувати всіма файлами.

## Де це налаштовується

Відкрийте адмін-панель і перейдіть до:

```text
System Settings -> Security Settings -> Upload Management -> Image Moderation
```

Основні налаштування:

- Access mode
- Enable moderation
- Moderation provider

## Що робить Access Mode

Access mode визначає, які вікові рейтинги можуть бути видимі публічно.

Поточні режими:

| Access Mode | Рейтинги, видимі публічно |
| --- | --- |
| Adult mode | General, R12, R16, R18 |
| Youth mode | General, R12, R16 |
| Teen mode | General, R12 |
| Child mode | Тільки General |

Типово використовується Adult mode.

Для приватних сайтів або сайтів із дорослим контентом Adult mode може бути доречним. Для більш обережної публічної галереї виберіть Youth, Teen або Child mode.

## Що дає ввімкнення модерації

Коли модерація ввімкнена, ImgBed під час завантаження викликає вибраного moderation provider і зберігає визначений віковий рейтинг.

Основні рейтинги:

| Рейтинг | Значення |
| --- | --- |
| General | Безпечний публічний контент |
| R12 | Помірно чутливий контент |
| R16 | Більш чутливий контент |
| R18 | Контент для дорослих |

Результат модерації використовується під час визначення публічного доступу.

Якщо модерацію не ввімкнено або старі файли не мають рейтингу, такі файли вважаються unrated. Unrated-файли не прибираються автоматично з публічної галереї або random image API лише через відсутність рейтингу.

## Вибір moderation provider

Доступні провайдери:

- moderatecontent.com
- nsfwjs
- Sightengine

У кожного провайдера свої вимоги:

- moderatecontent.com зазвичай потребує API Key.
- nsfwjs зазвичай потребує API endpoint URL.
- Sightengine потребує API user і API secret.

Вибирайте за своїм акаунтом, доступністю й якістю розпізнавання. Якщо модерацію ввімкнено й налаштовано правильно, ImgBed намагається записати рейтинг зображення під час завантаження.

## Вплив на публічну галерею

Публічна галерея фільтрує файли відповідно до access mode.

Приклади:

- Adult mode: R18-зображення можуть з'являтися.
- Youth mode: R18-зображення приховані.
- Teen mode: R16 і R18 приховані.
- Child mode: показуються тільки General-зображення.

Це впливає лише на звичайний публічний доступ. Адмін-панель і далі показує всі файли.

## Вплив на публічні URL файлів

Публічні URL файлів — це прямі посилання на зображення, які відкривають відвідувачі.

Якщо рейтинг файлу дозволений поточним access mode, ImgBed повертає оригінальне зображення.

Якщо рейтинг вищий за дозволений рівень, звичайний публічний доступ не повертає оригінальне зображення. Натомість ImgBed повертає налаштований blocked result або зображення-заглушку.

Приклад:

- Поточний режим — Child mode.
- Зображення має рейтинг R18.
- Відвідувач відкриває публічний URL напряму.
- ImgBed не повертає цьому відвідувачу оригінальне R18-зображення.

![Обмежене зображення файлу](../../image/Safety/文件受限图.png)

Адміністратори, які переглядають файли в адмін-панелі, цим обмеженням не зачіпаються.

## Вплив на Random Image API

Random image API також фільтрує пул кандидатів за access mode.

У Child mode випадкові зображення вибираються тільки з файлів із рейтингом General.

У Youth mode випадкові зображення можуть надходити з General, R12 і R16, але не з R18.

Так random image API не обходить обмеження публічної галереї.

## Зв'язок із правилами списків

Access mode — не єдине правило публічного доступу. Він працює разом із allow/block list rules.

Простіше кажучи:

- allowlisted-контент спершу вважається публічним;
- blocklisted-контент не може переглядатися напряму звичайними відвідувачами;
- контент, який не входить до жодного списку, потім перевіряється за access mode.

Якщо зображення обмежене і віковим рейтингом, і правилами списків, звичайні відвідувачі все одно не зможуть напряму побачити оригінальний файл.

## Рекомендовані налаштування

Для публічних сайтів:

- Увімкніть модерацію.
- Виберіть access mode відповідно до аудиторії сайту.
- Для відвідувачів будь-якого віку використовуйте Child mode або Teen mode.
- Уникайте Adult mode, якщо не хочете показувати дорослий контент публічно.
- Переглядайте рейтинги файлів в адмін-панелі й за потреби змінюйте їх вручну.

Для приватних або особистих сайтів:

- Adult mode зазвичай підходить.
- Увімкніть модерацію, якщо вона корисна.
- Переглядайте й коригуйте рейтинги в адмін-панелі за потреби.

## FAQ

### Чи зникнуть файли з адмін-панелі після зміни Access Mode?

Ні.

Access mode впливає лише на звичайний публічний доступ. На адмін-панель він не впливає.

### Чому після переходу на Child Mode у публічній галереї стало менше зображень?

Child mode дозволяє публічно показувати тільки файли з рейтингом General. R12, R16 і R18 відфільтровуються.

### Чи можуть публічні URL відкривати adult images?

Якщо поточний access mode не дозволяє цей рейтинг, звичайні публічні URL не повертають оригінальне зображення.

### Чи може Random Image API повернути обмежені зображення?

Ні.

Random image API фільтрує кандидатів відповідно до поточного access mode.

### Що буде зі старими unrated images?

Unrated-зображення не приховуються автоматично лише через відсутність результатів модерації. Їхні рейтинги можна пізніше змінити в адмін-панелі.
