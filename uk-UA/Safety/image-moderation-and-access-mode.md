# Модерація зображень і режим доступу

Модерація зображень призначає завантаженим файлам вікові рейтинги. Режим доступу визначає, які рейтинги можна показувати через публічний доступ.

Це впливає на публічну галерею, публічні URL файлів і API випадкових зображень. На адмін-панель це обмеження не поширюється. Адміністратори все одно можуть переглядати й керувати всіма файлами.

## Де це налаштовується

Відкрийте адмін-панель і перейдіть до:

```text
System Settings -> Security Settings -> Upload Management -> Image Moderation
```

Основні налаштування:

- Режим доступу
- Увімкнути модерацію
- Постачальник модерації

## Що робить режим доступу

Режим доступу визначає, які вікові рейтинги можуть показуватися публічно.

Поточні режими:

| Режим доступу | Публічно видимі рейтинги |
| --- | --- |
| Режим для дорослих | General, R12, R16, R18 |
| Молодіжний режим | General, R12, R16 |
| Підлітковий режим | General, R12 |
| Дитячий режим | Лише General |

Типове значення — режим для дорослих.

Для приватних сайтів або сайтів із дорослим контентом режим для дорослих може бути доречним. Для обережнішої публічної галереї виберіть молодіжний, підлітковий або дитячий режим.

## Що дає ввімкнення модерації

Коли модерацію ввімкнено, ImgBed під час завантаження викликає вибраного постачальника модерації та зберігає визначений віковий рейтинг.

Основні рейтинги:

| Рейтинг | Значення |
| --- | --- |
| General | Безпечний публічний контент |
| R12 | Злегка чутливий контент |
| R16 | Більш чутливий контент |
| R18 | Контент для дорослих |

Результат модерації використовується під час визначення публічного доступу.

Якщо модерацію не ввімкнено або старі файли не мають рейтингу, ці файли вважаються нерейтингованими. Нерейтинговані файли не видаляються автоматично з публічної галереї або API випадкових зображень лише через відсутність рейтингу.

## Вибір постачальника модерації

Доступні постачальники:

- moderatecontent.com
- nsfwjs
- Sightengine

У кожного постачальника різні вимоги:

- moderatecontent.com зазвичай потребує API Key.
- nsfwjs зазвичай потребує URL кінцевої точки API.
- Sightengine потребує API user і API secret.

Вибирайте за своїм акаунтом, доступністю та якістю розпізнавання. Якщо модерацію ввімкнено й налаштовано правильно, ImgBed намагається записати рейтинг зображення під час завантаження.

## Вплив на публічну галерею

Публічна галерея фільтрує файли відповідно до режиму доступу.

Приклади:

- Режим для дорослих: R18-зображення можуть з'являтися.
- Молодіжний режим: R18-зображення приховано.
- Підлітковий режим: R16 і R18 приховано.
- Дитячий режим: показуються лише зображення з рейтингом General.

Це впливає лише на звичайний публічний доступ. Адмін-панель і надалі показує всі файли.

## Вплив на публічні URL файлів

Публічні URL файлів — це прямі посилання на зображення, які відкривають відвідувачі.

Якщо рейтинг файла дозволений поточним режимом доступу, ImgBed повертає оригінальне зображення.

Якщо рейтинг перевищує дозволений рівень, звичайний публічний доступ не повертає оригінальне зображення. Натомість ImgBed повертає налаштований результат блокування або зображення-заглушку.

Приклад:

- Поточний режим — дитячий.
- Зображення має рейтинг R18.
- Відвідувач відкриває публічний URL напряму.
- ImgBed не повертає цьому відвідувачу оригінальне R18-зображення.

![Обмежене зображення файла](../../image/Safety/文件受限图.png)

Адміністратори, які переглядають файли в адмін-панелі, не зачіпаються цим обмеженням.

## Вплив на API випадкових зображень

API випадкових зображень також фільтрує пул кандидатів за режимом доступу.

У дитячому режимі випадкові зображення вибираються лише з файлів із рейтингом General.

У молодіжному режимі випадкові зображення можуть бути з файлів General, R12 і R16, але не з R18.

Так API випадкових зображень не обходить обмеження публічної галереї.

## Зв'язок із правилами списків

Режим доступу — не єдине правило публічного доступу. Він працює разом із правилами дозволених і заблокованих списків.

Простіше кажучи:

- контент зі списку дозволених спочатку вважається публічним;
- контент зі списку заблокованих не може переглядатися напряму звичайними відвідувачами;
- контент, якого немає в жодному списку, потім перевіряється за режимом доступу.

Якщо зображення обмежене і віковим рейтингом, і правилами списків, звичайні відвідувачі все одно не зможуть напряму побачити оригінальний файл.

## Рекомендовані налаштування

Для публічних сайтів:

- Увімкніть модерацію.
- Виберіть режим доступу відповідно до аудиторії сайту.
- Для відвідувачів будь-якого віку використовуйте дитячий або підлітковий режим.
- Уникайте режиму для дорослих, якщо не хочете показувати дорослий контент публічно.
- Переглядайте рейтинги файлів в адмін-панелі й за потреби змінюйте їх вручну.

Для приватних або особистих сайтів:

- Режим для дорослих зазвичай підходить.
- Увімкніть модерацію, якщо вона корисна.
- Переглядайте й коригуйте рейтинги в адмін-панелі за потреби.

## FAQ

### Чи зникнуть файли з адмін-панелі після зміни режиму доступу?

Ні.

Режим доступу впливає лише на звичайний публічний доступ. На адмін-панель він не впливає.

### Чому після переходу на дитячий режим у публічній галереї стало менше зображень?

Дитячий режим дозволяє публічно показувати лише файли з рейтингом General. R12, R16 і R18 фільтруються.

### Чи можуть публічні URL відкривати зображення для дорослих?

Якщо поточний режим доступу не дозволяє цей рейтинг, звичайні публічні URL не повертають оригінальне зображення.

### Чи може API випадкових зображень повернути обмежені зображення?

Ні.

API випадкових зображень фільтрує кандидатів відповідно до поточного режиму доступу.

### Що буде зі старими зображеннями без рейтингу?

Зображення без рейтингу не приховуються автоматично лише через відсутність результатів модерації. Їхні рейтинги можна пізніше змінити в адмін-панелі.
