# Magnet Transfer

Magnet transfer завантажує файли з magnet link і автоматично передає їх у вибраний cloud storage channel.

Це зручно для перенесення anime-серій, відео, архівів та схожих файлів. Вставте magnet link, і ImgBed створить фонове завдання завантаження. Після завершення файл буде завантажено в ImgBed, а фінальне посилання з'явиться у списку завантажень.

![Magnet transfer](../../image/other/磁力链接/磁力链接.png)

## Де це використовувати

Вхід для magnet transfer знаходиться в зоні завантаження на головній сторінці.

Вставте magnet link у поле введення, виберіть `Transfer`, потім завантажте.

![Завантаження anime](../../image/other/磁力链接/上传番剧.png)

## Перед першим використанням

Спочатку налаштуйте magnet transfer в адмін-панелі.

Зазвичай потрібні:

1. GitHub-акаунт для запуску download task.
2. Cloud upload channel, наприклад Google Drive або OneDrive.
3. Цільова upload directory.
4. Task timeout.

Коли налаштування готові, поверніться на головну сторінку й вставте magnet link, щоб почати transfer.

## Завантаження magnet link

1. Вставте magnet link у поле завантаження на головній сторінці.
2. Переконайтеся, що режим встановлено як `Transfer`.
3. Натисніть upload.
4. Дочекайтеся, поки ImgBed створить magnet task.
5. Після запуску завдання використовуйте плаваючу панель `Magnet Tasks` у правому нижньому куті, щоб стежити за прогресом.

Download і upload можуть зайняти час. Швидкість залежить від magnet-ресурсу, середовища GitHub runtime і вибраного cloud storage channel.

![Magnet завантажується](../../image/other/磁力链接/磁力链接下载中.png)

## Після завершення

Після завершення завдання у списку завантажень з'являться назва файлу та посилання.

Для відео показується video preview, для зображень — image preview, а для інших файлів — звичайна іконка файлу.

![Завантажене відео](../../image/other/磁力链接/下载好后的视频.png)

Можна скопіювати:

| Тип посилання | Для чого |
| --- | --- |
| Original link | Прямий доступ до файлу |
| Markdown | Markdown-дописи або нотатки |
| HTML | Код вебсторінки |
| BBCode | Форуми з підтримкою BBCode |

## Панель Magnet Task

Панель magnet task у правому нижньому куті показує кількість завдань, назву, прогрес і фінальний статус.

Типові стани:

| Статус | Значення |
| --- | --- |
| Waiting | Завдання створене й очікує запуску. |
| Downloading | Magnet-ресурс завантажується. |
| Uploading | Файл завантажено й він передається в cloud storage. |
| Completed | Upload успішний, посилання можна копіювати. |
| Failed | Завдання не завершилося успішно. Перевірте повідомлення й спробуйте ще раз. |

## Поради

- Якщо magnet link містить кілька файлів, ImgBed намагається показати основний завершений файл.
- Великі файли потребують більше часу. Дочекайтеся завершення завдання перед оновленням сторінки.
- Якщо magnet-ресурс не має доступних peers, він може бути дуже повільним або не завантажитися.
- Якщо у cloud account закінчилася квота, authorization прострочена або upload directory неправильна, завдання може впасти.
- Video preview може з'явитися через кілька секунд після завершення upload.

## FAQ

### Після вставлення Magnet Link нічого не запускається

Переконайтеся, що magnet transfer увімкнений в адмін-панелі, а також вибрані придатний GitHub-акаунт і cloud channel.

### Завантаження завжди повільне

Швидкість magnet залежить від самого ресурсу. Якщо немає доступних peers, завантаження може бути дуже повільним або неможливим.

### Після upload немає preview

Спершу перевірте, чи відкривається посилання на файл. Відеофайлам може знадобитися трохи часу для завантаження в браузері; також можна відкрити посилання напряму.

### Що перевірити, якщо task failed?

Перевірте, чи magnet link дійсний, чи працює cloud channel і чи правильна upload directory. Потім надішліть завдання ще раз.
