# API випадкових зображень і публічна галерея

Обидві функції налаштовуються тут:

```text
System Settings -> Other Settings
```

## API випадкових зображень

API випадкових зображень повертає один випадковий файл із вибраних каталогів. Це корисно для фонів сайту, ротації аватарів або викликів випадкових зображень із зовнішніх сторінок.

Після ввімкнення використовуйте:

```text
https://your-domain.com/random
```

## Налаштування API випадкових зображень

| Параметр | Призначення |
| --- | --- |
| Увімкнути | Увімкнення або вимкнення кінцевої точки `/random`. Коли вимкнено, доступ заборонено. |
| Каталоги | Обмежує, які каталоги може використовувати API випадкових зображень. Каталоги, не включені тут, не можуть використовуватися API. |
| Демонстрація виклику | Створює посилання API випадкових зображень, які можна скопіювати напряму. |

Можна вибрати кілька каталогів. Наприклад, якщо дозволено лише `/landscape/` і `/portrait/`, API випадкових зображень зможе вибирати файли лише з цих каталогів та їхніх підкаталогів.

## Параметри API випадкових зображень

| Параметр | Приклад | Призначення |
| --- | --- | --- |
| `dir` | `/landscape/` | Указує випадковий каталог. |
| `content` | `image` | Указує тип медіа. Використовуйте `image`, `video`, `audio` або комбінації через кому. |
| `orientation` | `auto` | Фільтрує орієнтацію зображення. Використовуйте `portrait`, `landscape` або `auto`. |
| `type` | `url` | Формат повернення. Порожнє значення означає перенаправлення, `url` повертає URL простим текстом, `json` повертає JSON. |
| `origin` | `1` | Використовується з `type=url`, щоб повернути повний URL. |
| `age` | `all-ages,r12` | Фільтрує за віковим рейтингом. |
| `tag` | `wallpaper,sky` | Повертає лише файли, що містять ці мітки. |
| `ex` | `private` | Виключає файли, що містять ці мітки. |

## Формати повернення

Без `type` API напряму перенаправляє на URL випадкового файлу.

З `type=url` він повертає текстовий URL.

З `type=json` він повертає інформацію про файл, зокрема URL файлу, ID файлу, назву файлу, тип файлу, мітки, рейтинг і пов'язані метадані.

## Правила доступу

API випадкових зображень дотримується правил публічного доступу:

| Правило | Ефект |
| --- | --- |
| Обмеження каталогів | Можна вибирати лише файли в дозволених каталогах. |
| Список блокування | Файли зі списку блокування виключаються з випадкового пулу. |
| Режим списку дозволів | Коли ввімкнено, повертаються лише файли, дозволені для публічного доступу. |
| Віковий рейтинг | Контент R12, R16, R18 і подібний фільтрується поточним режимом доступу. |

Якщо після фільтрації немає відповідного файлу, API не повертає відповідний результат.

## Кеш

API випадкових зображень кешує набори кандидатів каталогів, щоб підвищити швидкість.

Після зміни файлів ImgBed оновлює версію кешу каталогу, а наступні запити повторно будують набір кандидатів. Порожні каталоги кешуються на короткий час, щоб уникнути повторних запитів.

## Публічна галерея

Публічна галерея надає відкриту сторінку перегляду лише для читання для каталогів, які ви дозволяєте бачити відвідувачам.

Після ввімкнення відвідувачі можуть відкрити:

```text
https://your-domain.com/browse/directory-name
```

## Налаштування публічної галереї

| Параметр | Призначення |
| --- | --- |
| Увімкнути | Увімкнення або вимкнення публічної галереї. Коли вимкнено, відвідувачі не можуть її переглядати. |
| Режим завантаження зображень | Керує тим, чи попередні перегляди використовують оригінальні зображення або мініатюри. |
| Відкриті каталоги | Задає, до яких каталогів відвідувачі можуть мати доступ. |

## Режим завантаження зображень

| Режим | Призначення |
| --- | --- |
| Оригінал | Сторінка відвідувача завантажує оригінальні файли напряму. |
| Мініатюра | Сторінка відвідувача надає перевагу мініатюрам для швидшого завантаження. |

## Відкриті каталоги

Відкриті каталоги визначають, що можуть бачити відвідувачі.

Наприклад:

```text
/1/,/2/,/landscape/,/portrait/
```

Тоді відвідувачі можуть отримати доступ до цих каталогів:

```text
https://your-domain.com/browse/1
https://your-domain.com/browse/2
https://your-domain.com/browse/landscape
https://your-domain.com/browse/portrait
```

Підкаталоги також можна відкривати, наприклад `/2026/lucky/`. Доступ відвідувачів до невідкритих каталогів блокується.

## Можливості публічної галереї

| Можливість | Опис |
| --- | --- |
| Перегляд каталогів | Перегляд файлів і підкаталогів у відкритих каталогах. |
| Пошук | Пошук за назвою файлу, ID файлу або мітками. |
| Фільтр типу | Фільтрує зображення, відео, аудіо або інші файли. |
| Фільтр міток | Включає або виключає вибрані мітки. |
| Фільтр орієнтації | Фільтрує горизонтальні або вертикальні зображення. |
| Фільтр часу | Фільтрує за діапазоном часу завантаження. |
| Фільтр розширення | Фільтрує за розширенням файлу. |
| Копіювання посилання | Копіює посилання доступу до файлу. |
| Попередній перегляд медіа | Перегляд або відтворення зображень, відео та аудіо на сторінці відвідувача. |

## Правила доступу публічної галереї

Публічна галерея також дотримується правил публічного доступу:

| Правило | Ефект |
| --- | --- |
| Відкриті каталоги | Показуються лише дозволені каталоги. |
| Режим доступу | Вміст фільтрується за поточним режимом доступу вікового рейтингу. |
| Режим списку дозволів | Коли ввімкнено, показуються лише файли, дозволені для публічного доступу. |
| Список блокування | Файли зі списку блокування приховані. |
