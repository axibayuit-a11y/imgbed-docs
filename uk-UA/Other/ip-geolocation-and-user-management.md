# IP-геолокація та керування користувачами

IP-геолокація перетворює IP-адреси в записах про завантажувачів, пристроях входу та подібних журналах на приблизні місцезнаходження.

Після налаштування панель адміністратора може чіткіше показувати джерела завантажень і доступу. Керування користувачами також дає змогу блокувати або відновлювати доступ до завантаження для підозрілих IP-адрес.

## Де налаштовувати

Відкрийте:

```text
System Settings -> Other Settings -> IP Geolocation
```

![IP-геолокація](../../image/other/ip定位/ip定位.png)

## Доступні налаштування

Новий процес IP-геолокації підтримує кілька джерел, а не залежить від одного картографічного сервісу.

| Налаштування | Призначення |
| --- | --- |
| Мова IP-геолокації | Вибирає мову відображення, наприклад англійську, спрощену китайську, японську, французьку та інші. |
| MaxMind Account ID | ID облікового запису MaxMind для MaxMind GeoLite Web Service. |
| MaxMind License Key | MaxMind License Key. |
| Tencent Map Key | Ключ Tencent Location Service. Корисний для китайських адрес і IP-адрес материкового Китаю. |
| ipapi Key | Ключ APILayer ipapi. Підтримує багатомовну IP-геолокацію. |

Заповнюйте лише ті сервіси, які вам потрібні. Не потрібно налаштовувати кожне поле.

Якщо жодного ключа не надано, ImgBed все одно спробує вбудовані безкоштовні джерела, але стабільність, підтримка мов і точність можуть бути нижчими, ніж у сервісу, який ви налаштуєте самостійно.

## Рекомендований вибір

Якщо вам переважно потрібні китайські адреси:

1. Установіть мову IP-геолокації на спрощену китайську.
2. Налаштуйте Tencent Map Key.
3. За потреби додайте MaxMind або ipapi як резервні джерела.

Якщо вам переважно потрібні англійські або багатомовні адреси:

1. Виберіть потрібну мову.
2. Налаштуйте MaxMind Account ID і License Key.
3. Додайте ipapi Key, якщо потрібні кращі багатомовні результати.

## Налаштування MaxMind

Для MaxMind потрібні:

```text
MaxMind Account ID
MaxMind License Key
```

Знайдіть ID облікового запису на панелі MaxMind і створіть License Key на сторінці License Keys.

![Налаштування ключа MaxMind](../../image/other/ip定位/maxmind的key配置.png)

Після створення вставте Account ID і License Key в ImgBed та збережіть.

Безкоштовний план MaxMind підходить для повсякденного використання, але має ліміти запитів. Якщо квоту перевищено, ImgBed продовжить пробувати інші доступні джерела.

## Налаштування ipapi

ipapi використовує APILayer API Key.

Відкрийте консоль ipapi та скопіюйте показаний там API Key.

![Налаштування ipapi](../../image/other/ip定位/ipapi配置.png)

Вставте його в поле `ipapi Key` в ImgBed і збережіть.

ipapi підтримує багатомовну IP-геолокацію та корисний, коли потрібно показувати адреси вибраною мовою. Його безкоштовний план також має ліміти запитів. Якщо квота закінчиться, ImgBed продовжить пробувати інші доступні джерела.

## Налаштування Tencent Map Key

Tencent Map Key корисний для китайських адрес, особливо для IP-адрес материкового Китаю.

Під час створення ключа в Tencent Location Service увімкніть:

```text
WebServiceAPI
```

Після створення вставте ключ у `Tencent Map Key` і збережіть.

Якщо вам потрібна лише базова китайська IP-геолокація, Tencent Map Key достатньо для початку.

## Що перевіряти в керуванні користувачами

Керування користувачами доступне у верхній частині панелі адміністратора.

![Керування користувачами](../../image/other/用户管理显示.png)

Керування користувачами показує активність завантажень за IP:

| Поле | Опис |
| --- | --- |
| Джерело IP | IP-адреса джерела завантажувача. |
| Адреса | Приблизне місцезнаходження, визначене за IP. |
| Загальний розмір завантажень | Загальний розмір файлів, завантажених із цього IP. |
| Кількість завантажень | Кількість завантажень із цього IP. |
| Завантаження дозволено | Увімкнено означає, що завантаження дозволено. Вимкнено означає, що завантаження заблоковано. |

Натисніть стрілку ліворуч, щоб розгорнути список файлів, завантажених із цього IP.

Список файлів показує назву файлу, попередній перегляд, розмір файлу, результат модерації, стан файлу та час завантаження. Якщо завантаження виглядають підозрілими, спочатку розгорніть IP, перегляньте файли, а потім вирішіть, чи блокувати подальші завантаження.

Якщо IP підозрілий, вимкніть `Upload allowed`. Майбутні завантаження з цього IP буде заблоковано.

## Пошук, сортування та розширені фільтри

У верхній частині керування користувачами можна шукати за джерелом IP або адресою.

Сортуйте за часом, кількістю завантажень або загальним розміром завантажень, щоб знайти нещодавніх завантажувачів, частих завантажувачів або IP з великим використанням.

Для глибшої перевірки відкрийте розширені фільтри.

![Розширені фільтри](../../image/other/用户管理高级筛选.png)

Розширені фільтри підтримують:

| Фільтр | Використання |
| --- | --- |
| Діапазон часу | Показує IP, які завантажували файли протягом вибраного періоду. |
| Стан доступу | Фільтрує за нормальним, заблокованим і подібними станами. |
| Список дозволу/блокування | Фільтрує за списком дозволу, списком блокування або відсутністю налаштування. |
| Тип файлу | Показує IP, які завантажували зображення, відео, аудіо, документи, код або інші файли. |
| Розмір файлу | Фільтрує за діапазоном розміру завантажених файлів. |
| Віковий рейтинг | Фільтрує за відсутністю налаштування, General, R12+, R16+, R18 і подібними рейтингами. |
| Стан файлу | Фільтрує за поточним станом файлу для перевірки аномальних файлів. |

Натисніть `Apply Filters`, щоб застосувати. Використовуйте `Reset`, щоб повернутися до всіх даних.

## Мобільний вигляд

На мобільних пристроях керування користувачами перемикається на картковий макет.

![Мобільне керування користувачами](../../image/other/手机端显示用户管理效果.png)

Кожна картка показує IP, адресу, загальний розмір завантажень, кількість завантажень і перемикач дозволу завантаження. Ви можете керувати користувачами без горизонтального прокручування таблиці.

## Якщо місцезнаходження виглядає неправильним

IP-геолокація приблизна. Це не точна адреса вулиці.

Якщо користувач перебуває за проксі, у дата-центрі, на хмарному сервері або в транскордонній мережі, показане місцезнаходження може відрізнятися від реального.

Використовуйте цю функцію, щоб розуміти приблизне джерело, знаходити аномальні завантаження та допомагати з рішеннями щодо блокування. Не розглядайте її як точне відстеження.

## Поширені випадки

| Випадок | Значення |
| --- | --- |
| Адреса порожня | IP, можливо, ще не визначено, або поточне джерело тимчасово недоступне. |
| Мова адреси неправильна | Перевірте мову IP-геолокації та чи налаштовано джерело, що підтримує цю мову. |
| Адреса показує дата-центр | Багато проксі, хмарних серверів і краулерів відображаються як адреси дата-центрів або ISP. |
| Кількість завантажень висока | Уважно перевірте цей IP і за потреби заблокуйте завантаження. |
| Загальний розмір завантажень великий | Відсортуйте або відфільтруйте, розгорніть IP і перевірте конкретні файли. |
| Потрібно відновити після блокування | Знову увімкніть `Upload allowed`. |

## Швидкий порядок дій

```text
Open IP Geolocation in Other Settings
-> Choose IP geolocation language
-> Fill MaxMind, Tencent Map, or ipapi credentials as needed
-> Save settings
-> Open User Management
-> Review IP source, address, total upload size, and upload count
-> Use search, sort, or advanced filters to find abnormal IPs
-> Allow or block uploads as needed
```
