# IP-геолокація та керування користувачами

IP-геолокація перетворює IP-адреси в записах завантажень, пристроях входу та схожих журналах на приблизні локації.

Після налаштування адмін-панель може чіткіше показувати, звідки йдуть завантаження та доступи. User Management також дозволяє блокувати або відновлювати доступ до завантаження для підозрілих IP-адрес.

## Де це налаштовується

Відкрийте:

```text
System Settings -> Other Settings -> IP Geolocation
```

![IP-геолокація](../../image/other/ip定位/ip定位.png)

## Доступні налаштування

Новіший процес IP geolocation підтримує кілька джерел, а не залежить від одного map service.

| Налаштування | Призначення |
| --- | --- |
| IP geolocation language | Вибирає мову відображення: English, Simplified Chinese, Japanese, French тощо. |
| MaxMind Account ID | Account ID MaxMind для MaxMind GeoLite Web Service. |
| MaxMind License Key | License Key MaxMind. |
| Tencent Map Key | Ключ Tencent Location Service. Корисний для китайських адрес і IP материкового Китаю. |
| ipapi Key | APILayer ipapi key. Підтримує багатомовну IP-геолокацію. |

Заповнюйте тільки ті сервіси, які потрібні. Немає потреби налаштовувати кожне поле.

Якщо ключів немає, ImgBed усе одно спробує вбудовані безплатні джерела, але стабільність, підтримка мов і точність можуть бути нижчими, ніж у сервісу, який ви налаштуєте самі.

## Рекомендований вибір

Якщо переважно потрібні китайські адреси:

1. Встановіть IP geolocation language як Simplified Chinese.
2. Налаштуйте Tencent Map Key.
3. За бажанням додайте MaxMind або ipapi як fallback sources.

Якщо потрібні англійські або багатомовні адреси:

1. Виберіть потрібну мову.
2. Налаштуйте MaxMind Account ID і License Key.
3. Додайте ipapi Key, якщо потрібні кращі багатомовні результати.

## Налаштування MaxMind

MaxMind потребує:

```text
MaxMind Account ID
MaxMind License Key
```

Знайдіть account ID у dashboard MaxMind і згенеруйте License Key на сторінці License Keys.

![Налаштування MaxMind key](../../image/other/ip定位/maxmind的key配置.png)

Після генерації вставте Account ID і License Key в ImgBed та збережіть.

Безплатний план MaxMind підходить для щоденного використання, але має request limits. Якщо квоту вичерпано, ImgBed продовжить пробувати інші доступні джерела.

## Налаштування ipapi

ipapi використовує APILayer API Key.

Відкрийте консоль ipapi й скопіюйте API Key, який там показано.

![Налаштування ipapi](../../image/other/ip定位/ipapi配置.png)

Вставте його в поле `ipapi Key` в ImgBed і збережіть.

ipapi підтримує багатомовну IP-геолокацію й корисний, коли адреси треба показувати вибраною мовою. Його безплатний план також має request limits. Якщо квота закінчиться, ImgBed продовжить пробувати інші доступні джерела.

## Налаштування Tencent Map Key

Tencent Map Key корисний для китайських адрес, особливо IP материкового Китаю.

Під час створення key у Tencent Location Service увімкніть:

```text
WebServiceAPI
```

Після створення вставте key у `Tencent Map Key` і збережіть.

Якщо вам потрібна тільки базова китайська IP-геолокація, Tencent Map Key достатньо для старту.

## Що перевіряти в User Management

User Management доступний у верхній частині адмін-панелі.

![User management](../../image/other/用户管理显示.png)

User Management показує активність завантажень за IP:

| Поле | Опис |
| --- | --- |
| IP source | IP-адреса джерела завантаження. |
| Address | Приблизна локація, визначена за IP. |
| Total upload size | Загальний розмір файлів, завантажених із цього IP. |
| Upload count | Кількість завантажень із цього IP. |
| Upload allowed | Увімкнено — завантаження дозволені. Вимкнено — завантаження заблоковані. |

Натисніть стрілку ліворуч, щоб розгорнути список файлів, завантажених із цього IP.

У списку файлів показуються назва файлу, preview, розмір, moderation result, статус файлу та час завантаження. Якщо завантаження виглядають підозріло, спершу розгорніть IP, перегляньте файли, а потім вирішіть, чи блокувати подальші завантаження.

Якщо IP підозрілий, вимкніть `Upload allowed`. Майбутні завантаження з цього IP будуть заблоковані.

## Пошук, сортування та розширені фільтри

Угорі User Management можна шукати за IP source або address.

Сортуйте за часом, upload count або total upload size, щоб знайти нещодавніх авторів завантажень, часті завантаження або IP з великим використанням.

Для глибшої перевірки відкрийте advanced filters.

![Розширені фільтри](../../image/other/用户管理高级筛选.png)

Advanced filters підтримують:

| Фільтр | Використання |
| --- | --- |
| Time range | Показати IP, які завантажували файли у вибраний період. |
| Access status | Фільтр за normal, blocked та схожими станами. |
| Allow/block list | Фільтр за allowlist, blocklist або unset. |
| File type | Показати IP, які завантажували images, videos, audio, documents, code або інші файли. |
| File size | Фільтр за діапазоном розміру завантажених файлів. |
| Age rating | Фільтр за unset, General, R12+, R16+, R18 та схожими рейтингами. |
| File status | Фільтр за поточним статусом файлу для розслідування аномалій. |

Натисніть `Apply Filters`, щоб застосувати. `Reset` повертає всі дані.

## Мобільний вигляд

На мобільних пристроях User Management переходить у картковий layout.

![Мобільний User Management](../../image/other/手机端显示用户管理效果.png)

Кожна картка показує IP, address, total upload size, upload count і перемикач upload allowed. Користувачами можна керувати без горизонтального прокручування таблиці.

## Якщо локація виглядає неправильно

IP-геолокація приблизна. Це не точна адреса до вулиці чи будинку.

Якщо користувач за proxy, data center, cloud server або cross-border network, показана локація може відрізнятися від реальної.

Використовуйте цю функцію, щоб розуміти приблизне походження, знаходити аномальні завантаження й допомагати з рішеннями про блокування. Не сприймайте її як точне відстеження.

## Типові випадки

| Випадок | Значення |
| --- | --- |
| Address порожній | IP ще міг не визначитися або поточне джерело тимчасово недоступне. |
| Мова address неправильна | Перевірте IP geolocation language і чи налаштовано джерело з підтримкою цієї мови. |
| Address показує data center | Багато proxy, cloud servers і crawlers відображаються як data center або ISP addresses. |
| Upload count високий | Уважно перевірте цей IP і за потреби заблокуйте завантаження. |
| Total upload size великий | Відсортуйте або відфільтруйте, розгорніть IP і перегляньте конкретні файли. |
| Потрібно відновити після блокування | Увімкніть `Upload allowed` назад. |

## Короткий сценарій

```text
Відкрити IP Geolocation в Other Settings
-> Вибрати IP geolocation language
-> За потреби заповнити MaxMind, Tencent Map або ipapi credentials
-> Зберегти налаштування
-> Відкрити User Management
-> Переглянути IP source, address, total upload size і upload count
-> Використати search, sort або advanced filters для пошуку аномальних IP
-> Дозволити або заблокувати завантаження за потреби
```
