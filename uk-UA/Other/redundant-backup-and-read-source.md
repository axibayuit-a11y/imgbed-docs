# Резервне дублювання та перемикання джерела читання

Резервне дублювання зберігає додаткову копію вже завантаженого файлу.

І основний файл, і файл резервної копії можна використовувати як джерела читання. Для відвідувачів зазвичай немає різниці. Єдина відмінність — який канал сховища віддає файл.

## Що може резервне дублювання

| Можливість | Опис |
| --- | --- |
| Зберігати додаткову копію | Резервує файли в іншому каналі завантаження, щоб зменшити ризик відмови одного каналу. |
| Перемикати джерело читання | Після успішного резервного копіювання перемикає читання файлів між основним каналом і каналом резервної копії. |
| Резервне копіювання одного файлу | Резервує один файл зі сторінки деталей файлу. |
| Пакетне резервне копіювання | Вибирає кілька файлів на сторінці адміністратора й резервує їх разом. |
| Глобальне резервне дублювання | Резервує файли за папками з інших налаштувань. |

## Розділ резервного дублювання

Відкрийте:

```text
System Settings -> Other Settings -> Redundant Backup
```

![Резервне дублювання](../../image/other/冗余备份截图.png)

Цей розділ найкраще підходить для додавання резервних копій до папки або до всіх файлів пакетно.

Канал резервної копії можна вибрати вручну, або можна вибрати автоматичне перемикання й дозволити ImgBed знайти відповідний канал резервної копії.

## Резервне копіювання з деталей файлу

Відкрийте сторінку деталей файлу в панелі адміністратора та натисніть резервне копіювання.

![Резервне копіювання в деталях файлу](../../image/other/文件详情里文件备份.png)

Це найкраще підходить для резервного копіювання одного важливого файлу за потреби.

Після успішного резервного копіювання сторінка деталей файлу показує доступні джерела читання.

## Пакетне резервне копіювання вибраних файлів

У панелі адміністратора виберіть кілька файлів і запустіть пакетне резервне копіювання.

![Пакетне резервне копіювання](../../image/other/批量备份截图.png)

Це найкраще підходить для обробки групи файлів.

Резервне копіювання вибраних файлів, резервне копіювання з деталей файлу та резервне дублювання в інших налаштуваннях використовують одну й ту саму систему резервного копіювання. Це лише різні точки входу.

## Перемикання джерела читання після резервного копіювання

Після завершення резервного копіювання сторінка деталей файлу дає змогу перемкнути джерело читання:

| Джерело читання | Опис |
| --- | --- |
| Основний канал | Читає з початкового каналу завантаження. |
| Канал резервної копії | Читає з каналу резервної копії. |

![Перемикання джерела читання після резервного копіювання](../../image/other/备份成功切换读取源.png)

Відвідувачам не потрібно знати, чи файл подається з основного каналу, чи з каналу резервної копії.

Вибране джерело читання стає пріоритетним джерелом для подальшого доступу до файлу.

## Коли резервне копіювання пропускається

Наведені нижче випадки пропускаються під час резервного копіювання. Це не помилки.

| Випадок | Чому пропускається |
| --- | --- |
| Уже є резервна копія | Файл, який уже має резервну копію, не резервується повторно. |
| Основний і резервний канали однакові | Резервна копія має зберігатися в іншому каналі, щоб мати сенс. |
| Немає придатного резервного каналу | Немає відповідного альтернативного каналу. |

Коротко: резервні копії мають іти в інший канал, а файли, які вже мають резервну копію, не споживають додатковий простір повторно.

## Основний канал і канал резервної копії

| Назва | Значення |
| --- | --- |
| Основний канал | Канал, використаний під час першого завантаження файлу. |
| Канал резервної копії | Канал, який зберігає дубльовану копію. |
| Основне джерело читання | Файл зараз читається з основного каналу. |
| Джерело читання резервної копії | Файл зараз читається з каналу резервної копії. |

Основне джерело читання й джерело читання резервної копії мають однакову поведінку для користувача.

Поки файл резервної копії доступний, зображення, відео та посилання для завантаження продовжують працювати після перемикання на джерело читання резервної копії.

## Що відбувається, коли файл видаляється

Коли файл видаляється, ImgBed видаляє і основний файл, і файл резервної копії.
