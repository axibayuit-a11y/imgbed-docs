# Автоматичне тегування

Автоматичне тегування налаштовується тут:

```text
System Settings -> Other Settings -> Auto Tagging
```

Воно автоматично створює мітки для зображень. Ці мітки корисні для пошуку, фільтрації випадкових зображень, фільтрації в публічній галереї та контролю доступу за віковим рейтингом.

## Що може автоматичне тегування

| Можливість | Опис |
| --- | --- |
| Створення змістових міток | Додає мітки для людей, сцен, об'єктів, художнього стилю та подібного візуального вмісту. |
| Створення міток персонажів | Корисно для аніме-зображень та ілюстрацій. |
| Додавання міток орієнтації | Додає `landscape`, `portrait` або `square`. |
| Додавання рейтингу зображення | Зберігає результати рейтингу `G/S/Q/E` для загального, чутливого, сумнівного або відвертого вмісту. |
| Автоматичне тегування під час завантаження | Нові завантажені зображення автоматично потрапляють у процес тегування. |
| Пакетне тегування | Додає мітки до старих зображень у всіх папках або вибраних папках. |

## Що потрібно підготувати спочатку

Підготуйте принаймні одну доступну URL-адресу Hugging Face Space.

Рекомендований спосіб — дублювати Space `wd-tagger` від SmilingWolf у власний обліковий запис Hugging Face:

```text
https://huggingface.co/spaces/SmilingWolf/wd-tagger
```

Ви можете тимчасово використовувати публічний Space, але публічні Space спільно використовуються багатьма користувачами, тому можуть потрапляти в чергу, сповільнюватися або ставати недоступними. Дубльований Space у вашому власному обліковому записі стабільніший для довгострокового автоматичного тегування.

## Дублювання Space SmilingWolf

1. Увійдіть у Hugging Face.
2. Відкрийте `https://huggingface.co/spaces/SmilingWolf/wd-tagger`.

![Публічний Space SmilingWolf](../../image/other/微笑狼的公开仓库.png)

3. Натисніть меню з трьома крапками у верхньому правому куті.
4. Виберіть `Duplicate this Space`.
5. Залиште назву Space за замовчуванням або виберіть власну, наприклад `wd-tagger`.
6. Установіть видимість `Public`. Публічні Space простіше викликати з ImgBed.
7. Спочатку залиште безкоштовне обладнання за замовчуванням. Оновлюйте його пізніше лише тоді, коли черги стануть помітними.
8. Створіть Space і дочекайтеся завершення збірки.

Після завершення збірки відкрийте сторінку свого Space. URL зазвичай має такий вигляд:

```text
https://huggingface.co/spaces/your-name/wd-tagger
```

Скопіюйте URL з браузера та вставте його в `Space URLs` в ImgBed.

## Заповнення кількох Space URL

Вводьте по одній URL-адресі Space в кожному рядку.

Приклади:

| Значення | Опис |
| --- | --- |
| `https://huggingface.co/spaces/SmilingWolf/wd-tagger` | Публічний Space SmilingWolf. Підходить для тимчасового тестування. |
| `https://huggingface.co/spaces/lintonxue00/wd-tagger` | URL сторінки скопійованого Space. |
| `https://huggingface.co/spaces/your-name/wd-tagger` | Ваша власна дубльована сторінка Space. |

Можна ввести кілька URL. ImgBed використовує кілька Space разом, що може підвищити швидкість.

Якщо один Space тимчасово недоступний, інші можуть продовжити обробку.

## Налаштування

| Параметр | Рекомендація |
| --- | --- |
| `Space URLs` | Введіть підготовлені URL Space. Використовуйте принаймні один. |
| Цільова папка | Залиште порожнім для всіх папок. Вибирайте папку лише тоді, коли потрібно обробити конкретний каталог. |
| Модель розпізнавання | За замовчуванням залиште `wd-swinv2-tagger-v3`. |
| Поріг загальних міток | Значення за замовчуванням підходить для більшості зображень. Нижчі значення дають більше міток, вищі — менше. |
| Поріг міток персонажів | Значення за замовчуванням обережне й допомагає уникати неправильних міток персонажів. |
| Автоматичний поріг `MCut` | Спочатку залиште вимкненим. Увімкніть, коли хочете, щоб модель сама визначала кількість міток. |
| Автоматичне тегування під час завантаження | Увімкніть, якщо нові зображення мають автоматично отримувати мітки. |
| Почати тегування | Запускає ручне пакетне тегування старих зображень. |

## Рекомендовані початкові значення

| Параметр | Рекомендоване значення |
| --- | --- |
| Модель розпізнавання | `wd-swinv2-tagger-v3` |
| Поріг загальних міток | `0.35` |
| Поріг міток персонажів | `0.85` |
| `MCut` | Спочатку вимкнено |
| Автоматичне тегування під час завантаження | Увімкніть за потреби |

Якщо міток забагато, трохи підвищте поріг загальних міток.

Якщо міток замало, трохи знизьте поріг загальних міток.

## Пакетне тегування

1. Заповніть `Space URLs`.
2. Виберіть цільову папку.
3. Натисніть запуск тегування.
4. Дочекайтеся завершення прогресу.

Якщо цільова папка порожня, ImgBed обробляє всі папки.

Пакетне тегування найкраще підходить для старих зображень. Для нових зображень увімкніть автоматичне тегування під час завантаження, щоб не запускати його вручну щоразу.

## Автоматичне тегування під час завантаження

Після ввімкнення автоматичного тегування під час завантаження нові зображення автоматично викликають налаштовані `Space URLs`.

Це підходить для довгострокового використання.

Якщо ваш Space стоїть у черзі, саме завантаження все одно може завершитися першим, а тегування продовжиться після цього.

## Які зображення обробляються

Автоматичне тегування переважно обробляє файли зображень.

Зображення, які вже мають повні мітки, орієнтацію, рейтинг, ширину та висоту, пропускаються, щоб уникнути зайвих викликів Space.

ImgBed за можливості заповнює лише відсутню інформацію. Наприклад, якщо бракує лише орієнтації, він спробує додати орієнтацію без виклику повного потоку змістового тегування.

## Поширені запитання

### Навіщо дублювати власний Space?

Публічні Space спільно використовуються багатьма користувачами. Ваш власний дубльований Space переважно використовується вашим сайтом ImgBed, тому зазвичай він швидший і надійніший.

### Space постійно запускається

Після першого створення або після тривалого простою Space може потребувати часу для запуску.

Спочатку відкрийте сторінку свого Space. Коли він зможе нормально розпізнати зображення, поверніться в ImgBed і почніть тегування.

### Як скопіювати URL Space?

Відкрийте сторінку Hugging Face Space і скопіюйте адресу з браузера.

Приклади:

```text
https://huggingface.co/spaces/lintonxue00/wd-tagger
https://huggingface.co/spaces/SmilingWolf/wd-tagger
```

### Чи можна додати кілька Space?

Так. Вводьте по одній URL-адресі Space в кожному рядку.

Кілька Space обробляють зображення разом і корисні, коли у вас багато зображень.

### Чому мітки англійською?

Моделі SmilingWolf виводять англійські мітки. Це очікувано.

Мітки переважно використовуються для пошуку, фільтрації, API випадкових зображень і фільтрів публічної галереї.

### Для чого потрібні рейтингові мітки?

Результати рейтингу працюють разом із режимом доступу в налаштуваннях безпеки.

Наприклад, коли доступ відвідувачів обмежено за віковим рейтингом, публічний перегляд і функції випадкових зображень фільтрують зображення за цими правилами.

## Швидкий порядок дій

```text
Sign in to Hugging Face
-> Open SmilingWolf/wd-tagger
-> Duplicate this Space
-> Wait for the Space to build
-> Copy your Space URL
-> Fill Space URLs in ImgBed
-> Choose model and thresholds
-> Start tagging or enable auto-tag on upload
```
