# Додавання каналу pCloud

## Найкраще підходить, якщо

- У вас є обліковий запис pCloud і ви хочете, щоб ImgBed зберігав зображення в pCloud.
- Вам підходить використання email і пароля pCloud як облікових даних каналу.

## Що підготувати спочатку

| Потрібно | Навіщо це потрібно |
| --- | --- |
| Email облікового запису pCloud | Для входу в pCloud API |
| Пароль pCloud | Для входу в pCloud API |
| API host | Типово `api.pcloud.com`. Для акаунтів у ЄС можна використовувати `eapi.pcloud.com`. |
| Storage directory | Місце зберігання файлів. Типово `imgbed`. |

## Де додати канал

1. Відкрийте системні налаштування.
2. Відкрийте налаштування завантаження.
3. Натисніть `Add Channel` у правому верхньому куті.
4. Виберіть `pCloud`.

## Поля налаштування

| Поле | Призначення | Обов'язково |
| --- | --- | --- |
| Channel name | Ідентифікує цей pCloud-канал, наприклад `Personal pCloud` | Так |
| Account email | Email для входу в pCloud | Так |
| Password | Ваш пароль pCloud | Так |
| API host | pCloud API host. Типово `api.pcloud.com`. | Ні |
| Storage directory | Директорія для зберігання файлів. Типово `imgbed`. | Ні |

Виберіть API host відповідно до регіону акаунта:

| Регіон акаунта | API Host |
| --- | --- |
| Default / US | `api.pcloud.com` |
| Europe | `eapi.pcloud.com` |

## Налаштування

1. Відкрийте налаштування завантаження.
2. Натисніть `Add Channel`.
3. Виберіть `pCloud`.
4. Введіть зрозумілу назву каналу.
5. Введіть email акаунта pCloud.
6. Введіть пароль pCloud.
7. Залиште API host як `api.pcloud.com` або використайте `eapi.pcloud.com` для акаунтів у ЄС.
8. Залиште storage directory як `imgbed` або змініть на потрібну папку.
9. Збережіть канал.

## Як перевірити

| Перевірка | Очікуваний результат |
| --- | --- |
| Картка каналу | Картка pCloud-каналу з'являється після збереження. |
| Перемикач каналу | Перемикач на картці залишається ввімкненим. |
| Показ email | На картці показано підключений email pCloud. |
| Запит квоти | Після успішного запиту показується використана й загальна місткість. |
| Тестове завантаження | Тестове зображення з'являється в налаштованій директорії pCloud. |

## Усунення несправностей

### Чому не OAuth2?

pCloud OAuth2 типово не є self-service. Потрібно написати в pCloud і попросити ввімкнути його.

Поточний OAuth2 flow pCloud також не підтримує workflow короткочасних upload links, потрібний ImgBed, тому цей канал використовує вхід через email і пароль акаунта.

### Який API Host використовувати?

Типово:

```text
api.pcloud.com
```

Для акаунтів у ЄС:

```text
eapi.pcloud.com
```

## Короткий сценарій

```text
Підготувати email і пароль pCloud
-> Відкрити налаштування завантаження
-> Додати канал
-> Вибрати pCloud
-> Заповнити назву каналу / email / пароль
-> Залишити API host як api.pcloud.com, якщо акаунт не в Європі
-> Залишити storage directory як imgbed, якщо не потрібна інша папка
-> Зберегти
-> Запитати квоту
-> Завантажити тестове зображення
```
