# Налаштування завантаження

Налаштування завантаження підключають ImgBed до ваших власних каналів зберігання. Після налаштування зображення й файли зберігаються у вибраному сервісі, а ImgBed керує посиланнями доступу, записами файлів, прев'ю, публічною галереєю, API випадкового зображення, WebDAV-доступом та пов'язаними функціями.

Різним користувачам підходять різні канали. Для простого старту можна вибрати Telegram, Discord або GitHub Releases. Якщо важливіші обсяг, швидкість і довгострокова стабільність, розгляньте Cloudflare R2, S3, OneDrive, Google Drive, Dropbox, Yandex, pCloud або власний WebDAV.

## Перед початком

> Перед першим використанням ImgBed потрібно відкрити сторінку ініціалізації та натиснути «Перебудувати індекс», щоб додати необхідні таблиці D1 і уникнути помилок у наступних функціях.
>
> ![Натиснути Перебудувати індекс під час ініціалізації](../../image/初始化点击重建索引.png)

- Підготуйте обліковий запис сховища або API-дані, які будете використовувати.
- Переконайтеся, що домен ImgBed доступний, бо OAuth-канали потребують URL зворотного виклику.
- Після додавання каналу спочатку завантажте тестове зображення й перевірте, що файл зберігається та відкривається.
