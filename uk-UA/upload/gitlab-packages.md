# Додавання каналу GitLab Packages

## Що підготувати заздалегідь

Потрібні лише три речі:

| Потрібно | Навіщо це потрібно |
| --- | --- |
| Обліковий запис GitLab | Для створення access token і володіння проєктом. |
| GitLab Personal Access Token | ImgBed використовує його для доступу до GitLab API, створення проєктів і завантаження файлів у Generic Packages. |
| Назва проєкту | Можна ввести тільки назву проєкту, наприклад `imgbed`. |

## Налаштування

### Крок 1. Увійдіть у GitLab і створіть Access Token

1. Увійдіть у GitLab.
2. Натисніть свій аватар у правому верхньому куті й відкрийте `Preferences`.
3. У лівій бічній панелі відкрийте `Access Tokens`.
4. Дайте token зрозумілу назву.
5. Виберіть дату закінчення дії відповідно до вашого підходу до обслуговування.
6. Виберіть scope `api`.
7. Одразу після створення скопіюйте й збережіть token.

![Вибір прав token](../../image/upload/gitlab-packages/勾选令牌权限.png)

## Крок 2. Заповніть канал GitLab Packages в ImgBed

Після вибору `GitLab Packages` у налаштуваннях завантаження заповніть поля так:

| Поле в інтерфейсі | Що ввести |
| --- | --- |
| Channel name | Назва на ваш вибір, наприклад `GitLabPrimary`. |
| Access Token | GitLab Personal Access Token, який ви щойно створили. |
| Project name | Або коротка назва проєкту, наприклад `imgbed`, або повний шлях на кшталт `username/imgbed`. |
| Private repository | Увімкніть або вимкніть відповідно до ваших потреб. |
| Remark | Необов'язково, наприклад `Primary upload channel`. |

![Налаштування каналу](../../image/upload/gitlab-packages/配置渠道内容.png)

## Крок 3. Збережіть канал

Після заповнення полів натисніть Save.

Система сама обробить такі деталі:

| Поведінка системи | Опис |
| --- | --- |
| Коротка назва проєкту | ImgBed визначає поточний обліковий запис GitLab і розгортає значення до повного шляху проєкту. |
| Повний шлях проєкту | ImgBed використовує шлях `username/project` саме в тому вигляді, у якому його введено. |
| Перевірка проєкту | Якщо використовується шлях поточного особистого акаунта, ImgBed автоматично створює проєкт, коли його ще немає. Якщо повний шлях введено вручну, ImgBed використовує його напряму. |
| Публічний або приватний стан | Видимість проєкту синхронізується відповідно до поточного перемикача. |

## Короткий чекліст

```text
Увійти в GitLab
-> Створити Access Token
-> Вибрати тільки scope api
-> Повернутися до ImgBed і ввести token та назву проєкту
-> Зберегти
-> Якщо введено тільки назву проєкту, ImgBed автоматично додасть поточний username
-> Якщо введено username/project, ImgBed використає це як є
-> Завантажити тестове зображення
```
