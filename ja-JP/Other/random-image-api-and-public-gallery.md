# ランダム画像 API と公開ギャラリー

どちらの機能も次の場所で設定します。

```text
System Settings -> Other Settings
```

## ランダム画像 API

ランダム画像 API は、選択したディレクトリからランダムに 1 つのファイルを返します。サイト背景、アバターのローテーション、外部ページからのランダム画像呼び出しに役立ちます。

有効化後は、次の URL を使用します。

```text
https://your-domain.com/random
```

## ランダム画像 API の設定

| オプション | 目的 |
| --- | --- |
| 有効化 | `/random` エンドポイントのオン/オフを切り替えます。無効の場合、アクセスは禁止されます。 |
| ディレクトリ | ランダム画像 API が使用できるディレクトリを制限します。ここに含まれていないディレクトリは API で使用できません。 |
| 呼び出しデモ | 直接コピーできるランダム画像 API リンクを生成します。 |

複数のディレクトリを選択できます。たとえば `/landscape/` と `/portrait/` だけを許可した場合、ランダム画像 API はその 2 つのディレクトリとそのサブディレクトリからのみファイルを選択できます。

## ランダム画像 API のパラメータ

| パラメータ | 例 | 目的 |
| --- | --- | --- |
| `dir` | `/landscape/` | ランダム対象のディレクトリを指定します。 |
| `content` | `image` | メディア種別を指定します。`image`、`video`、`audio`、またはカンマ区切りの組み合わせを使用します。 |
| `orientation` | `auto` | 画像の向きをフィルタします。`portrait`、`landscape`、`auto` を使用します。 |
| `type` | `url` | 返却形式。空の場合はリダイレクト、`url` はプレーンテキスト URL、`json` は JSON を返します。 |
| `origin` | `1` | `type=url` と併用し、完全な URL を返します。 |
| `age` | `all-ages,r12` | 年齢レーティングでフィルタします。 |
| `tag` | `wallpaper,sky` | これらのタグを含むファイルだけを返します。 |
| `ex` | `private` | これらのタグを含むファイルを除外します。 |

## 返却形式

`type` を指定しない場合、API はランダムファイルの URL へ直接リダイレクトします。

`type=url` の場合、テキスト URL を返します。

`type=json` の場合、ファイル URL、ファイル ID、ファイル名、ファイル種別、タグ、レーティング、関連メタデータを含むファイル情報を返します。

## アクセスルール

ランダム画像 API は公開アクセスルールに従います。

| ルール | 効果 |
| --- | --- |
| ディレクトリ制限 | 許可されたディレクトリ内のファイルだけが選択されます。 |
| ブロックリスト | ブロックリスト内のファイルはランダム候補から除外されます。 |
| 許可リストモード | 有効な場合、公開アクセスが許可されたファイルだけが返されます。 |
| 年齢レーティング | R12、R16、R18 などのコンテンツは、現在のアクセスモードに従ってフィルタされます。 |

フィルタ後に一致するファイルがない場合、API は一致する結果を返しません。

## キャッシュ

ランダム画像 API は、速度向上のためにディレクトリ候補プールをキャッシュします。

ファイルが変更されると、ImgBed はディレクトリキャッシュのバージョンを更新し、以降のリクエストで候補プールを再構築します。空のディレクトリは、繰り返しクエリを避けるため短時間キャッシュされます。

## 公開ギャラリー

公開ギャラリーは、訪問者に表示を許可したディレクトリを読み取り専用で閲覧できる公開ページを提供します。

有効にすると、訪問者は次の URL を開けます。

```text
https://your-domain.com/browse/directory-name
```

## 公開ギャラリーの設定

| オプション | 目的 |
| --- | --- |
| 有効化 | 公開ギャラリーのオン/オフを切り替えます。無効の場合、訪問者は閲覧できません。 |
| 画像読み込みモード | プレビューで元画像を使うかサムネイルを使うかを制御します。 |
| 公開ディレクトリ | 訪問者がアクセスできるディレクトリを設定します。 |

## 画像読み込みモード

| モード | 目的 |
| --- | --- |
| 元画像 | 訪問者ページが元ファイルを直接読み込みます。 |
| サムネイル | 訪問者ページが高速読み込みのためサムネイルを優先します。 |

## 公開ディレクトリ

公開ディレクトリは、訪問者が見られる内容を決定します。

例:

```text
/1/,/2/,/landscape/,/portrait/
```

訪問者は次の URL にアクセスできます。

```text
https://your-domain.com/browse/1
https://your-domain.com/browse/2
https://your-domain.com/browse/landscape
https://your-domain.com/browse/portrait
```

`/2026/lucky/` のようなサブディレクトリも公開できます。公開されていないディレクトリへのアクセスはブロックされます。

## 公開ギャラリーの機能

| 機能 | 説明 |
| --- | --- |
| ディレクトリ閲覧 | 公開ディレクトリ内のファイルとサブディレクトリを表示します。 |
| 検索 | ファイル名、ファイル ID、タグで検索します。 |
| 種別フィルタ | 画像、動画、音声、その他のファイルをフィルタします。 |
| タグフィルタ | 選択したタグを含める、または除外します。 |
| 向きフィルタ | 横長または縦長の画像をフィルタします。 |
| 時間フィルタ | アップロード時間の範囲でフィルタします。 |
| 拡張子フィルタ | ファイル拡張子でフィルタします。 |
| リンクコピー | ファイルアクセスリンクをコピーします。 |
| メディアプレビュー | 訪問者ページで画像、動画、音声を表示または再生します。 |

## 公開ギャラリーのアクセスルール

公開ギャラリーも公開アクセスルールに従います。

| ルール | 効果 |
| --- | --- |
| 公開ディレクトリ | 許可されたディレクトリだけが表示されます。 |
| アクセスモード | コンテンツは現在の年齢レーティングアクセスモードに従ってフィルタされます。 |
| 許可リストモード | 有効な場合、公開アクセスが許可されたファイルだけが表示されます。 |
| ブロックリスト | ブロックリスト内のファイルは非表示になります。 |
