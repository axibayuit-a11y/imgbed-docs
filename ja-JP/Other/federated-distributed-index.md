# フェデレーション分散インデックス

フェデレーション分散インデックスは、複数の ImgBed サイト同士でファイル一覧を共有するための機能です。

できることは大きく次の 3 つです。

- 自分の一部フォルダを他のサイトに共有する。
- 他のノードに参加し、相手が共有したファイル一覧を自分の管理画面に同期する。
- 共有ファイルを閲覧、検索、プレビュー、リンクコピーに使う。自分の保存先へ再アップロードする機能ではありません。

## 設定場所

```text
システム設定 -> その他の設定 -> フェデレーション分散インデックス
```

![ローカルノード](../../image/other/联盟图/联盟分布式索引本地节点.png)

画面は 3 つのタブに分かれています。

| タブ | 用途 |
| --- | --- |
| ローカルノード | 自分の共有設定、公開ドメイン、共有フォルダ、送信インデックスを管理 |
| 参加中のノード | 自分が参加している他サイトのノードを管理 |
| 参加申請 | 他の人から自分のノードへの参加申請を管理 |

## 初回設定

1. 「ローカルノード」を開きます。
2. 「有効」をオンにします。
3. 「同期フォルダ」で共有したいフォルダを選びます。
4. 「送信インデックスを更新」をクリックします。
5. ドメイン変更の確認が出た場合は、現在のドメインが本番用か確認して続行します。

同期フォルダは複数選択できます。空にした場合は、全フォルダを共有する扱いになります。

## ローカルノード

### 公開ドメイン

公開ドメインは、他のノードがあなたのサイトへアクセスするときの URL です。

通常はシステムが自動で判定するため、手入力は不要です。初回更新時やドメイン変更時に確認メッセージが表示されます。

### 同期フォルダ

同期フォルダで、どのファイルを共有対象にするかを決めます。

```text
/1/
/2/
```

このように指定した場合、相手はこの 2 つのディレクトリ内のファイルだけを同期できます。

### 送信インデックスを更新

自分が他のノードへ公開するファイル一覧を更新します。

次のタイミングで使います。

- フェデレーションを初めて有効にしたとき。
- 共有したいファイルを新しくアップロードしたとき。
- 同期フォルダを変更したとき。
- 公開ドメインを変更したとき。

## 他のノードに参加する

「参加中のノード」は、他の ImgBed ノードを購読する場所です。

![参加中のノード](../../image/other/联盟图/我加入的节点.png)

1. 相手から招待リンクを受け取ります。
2. 入力欄に貼り付けます。
3. 「参加申請」をクリックします。
4. 相手の管理画面で承認されるのを待ちます。

承認後、「受信インデックスを更新」すると相手の共有ファイル一覧が同期されます。

![インデックス更新](../../image/other/联盟图/更新索引.png)

## 自分のノードへの申請を処理する

![参加申請](../../image/other/联盟图/加入我的节点.png)

招待リンクを作るには、ローカルノードを有効にし、一度「送信インデックスを更新」して公開ドメインを確認してから、「招待リンクをリセット」をクリックします。

申請が届いたら、次の操作を選べます。

| 操作 | 内容 |
| --- | --- |
| 承認 | 相手があなたの共有ファイル一覧を同期できるようにします |
| 拒否 | 参加を許可しません |
| 削除 | 終了した記録を削除します |
| 状態確認 | 相手側が関係を維持しているか確認します |

承認後、相手側でも受信インデックスの更新が必要です。

![承認](../../image/other/联盟图/邀请节点同意.png)

## メッセージ

承認済みの関係では、ノードカードからメッセージを送れます。

メッセージは連絡用であり、ファイル、タグ、ディレクトリ、アクセス権には影響しません。

![メッセージ](../../image/other/联盟图/留言功能.png)

## 共有ファイルを見る

同期が終わったら管理画面のファイル一覧に戻ります。上部でローカルファイルとフェデレーションファイルを切り替えられます。

フェデレーションファイルは外部ノードの一覧なので、自分のサイト上で移動、削除、タグ変更、バックアップはできません。

![管理画面での表示](../../image/other/联盟图/联盟管理显示效果图.png)

## よくある質問

### 「関係記録がない」と表示される

相手側であなたの記録が削除された可能性があります。再度参加申請を行ってください。

![再申請](../../image/other/联盟图/无关系记录重新申请.png)

### 参加後にファイルが見えない

次を確認します。

1. 相手が申請を承認している。
2. 相手が送信インデックスを更新している。
3. 自分が受信インデックスを更新している。
4. 相手の同期フォルダに共有対象のディレクトリが含まれている。
