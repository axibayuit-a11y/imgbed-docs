# 冗長バックアップと読み取り元切り替え

冗長バックアップは、すでにアップロード済みのファイルを別の保存先にも複製する機能です。

通常の閲覧では主ファイルとバックアップファイルの違いは見えません。違うのは、実際にどの保存先から読み込むかだけです。

## できること

| 機能 | 説明 |
| --- | --- |
| もう 1 つ保存する | 別チャンネルへ複製し、単一保存先の障害リスクを下げます |
| 読み取り元を切り替える | 主チャンネルとバックアップチャンネルを切り替えられます |
| 単一ファイルのバックアップ | ファイル詳細画面から重要なファイルだけ追加でバックアップできます |
| 一括バックアップ | 管理画面で複数ファイルを選択して処理できます |
| 全体バックアップ | その他設定からフォルダ単位でまとめて補完できます |

## 設定場所

```text
システム設定 -> その他の設定 -> 冗長バックアップ
```

![冗長バックアップ](../../image/other/冗余备份截图.png)

ここでは特定フォルダ、または全ファイルにまとめてバックアップを追加できます。バックアップ先は手動選択のほか、自動選択も利用できます。

## ファイル詳細からバックアップする

管理画面でファイル詳細を開くと、そのファイルだけをバックアップできます。

![ファイル詳細でバックアップ](../../image/other/文件详情里文件备份.png)

重要なファイルだけを後から保護したいときに便利です。成功すると、詳細画面に読み取り元の切り替えが表示されます。

## 複数ファイルをまとめてバックアップする

管理画面でファイルを複数選択し、一括バックアップを実行できます。

![一括バックアップ](../../image/other/批量备份截图.png)

入口は違っても、ファイル詳細、複数選択、その他設定の冗長バックアップは同じ仕組みを使います。

## 読み取り元の切り替え

| 読み取り元 | 説明 |
| --- | --- |
| 主チャンネル | 最初にアップロードした保存先から読み込みます |
| バックアップチャンネル | 冗長バックアップで作成した保存先から読み込みます |

![読み取り元切り替え](../../image/other/备份成功切换读取源.png)

切り替え後は、画像表示、動画再生、ダウンロードリンクなどが選択した読み取り元を優先します。

## スキップされるケース

| ケース | 理由 |
| --- | --- |
| すでにバックアップ済み | 重複保存を避けるため |
| 主チャンネルと同じ保存先 | 同じ場所への複製ではバックアップにならないため |
| 利用できる保存先がない | 適切な別チャンネルが見つからないため |

## 削除時の動作

ファイルを削除すると、主ファイルとバックアップファイルの両方が削除されます。
