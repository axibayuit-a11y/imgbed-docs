# ထပ်ပွားအရန်သိမ်းခြင်းနှင့် ဖတ်ယူရင်းမြစ်ပြောင်းခြင်း

ထပ်ပွားအရန်သိမ်းခြင်းသည် အပ်လုဒ်တင်ပြီးသား ဖိုင်တစ်ခုအတွက် အပိုမိတ္တူတစ်ခုကို သိမ်းဆည်းပေးသည်။

မူလဖိုင်နှင့် အရန်ဖိုင် နှစ်ခုလုံးကို ဖတ်ယူရင်းမြစ်အဖြစ် အသုံးပြုနိုင်သည်။ ဧည့်သည်များအတွက် ပုံမှန်အားဖြင့် ကွာခြားချက် မရှိပါ။ ကွာခြားချက်မှာ ဖိုင်ကို မည်သည့်သိုလှောင်မှုချန်နယ်မှ ပေးပို့နေသည်ဆိုခြင်းသာ ဖြစ်သည်။

## ထပ်ပွားအရန်သိမ်းခြင်းက ဘာလုပ်နိုင်သလဲ

| လုပ်ဆောင်ချက် | ဖော်ပြချက် |
| --- | --- |
| အပိုမိတ္တူ သိမ်းဆည်းခြင်း | ချန်နယ်တစ်ခုတည်း ပျက်ကွက်နိုင်သည့် အန္တရာယ်ကို လျှော့ချရန် ဖိုင်များကို အခြားအပ်လုဒ်ချန်နယ်သို့ အရန်သိမ်းသည်။ |
| ဖတ်ယူရင်းမြစ်ပြောင်းခြင်း | အရန်သိမ်းမှု အောင်မြင်ပြီးနောက် ဖိုင်ဖတ်ယူမှုကို မူလချန်နယ်နှင့် အရန်ချန်နယ်အကြား ပြောင်းနိုင်သည်။ |
| ဖိုင်တစ်ခုချင်း အရန်သိမ်းခြင်း | ဖိုင်အသေးစိတ်စာမျက်နှာမှ ဖိုင်တစ်ခုကို အရန်သိမ်းသည်။ |
| အစုလိုက် အရန်သိမ်းခြင်း | စီမံခန့်ခွဲရေးစာမျက်နှာတွင် ဖိုင်အများအပြားကို ရွေးချယ်ပြီး အတူတကွ အရန်သိမ်းသည်။ |
| စနစ်တစ်ခုလုံး ထပ်ပွားအရန်သိမ်းခြင်း | အခြားဆက်တင်များမှ ဖိုင်တွဲအလိုက် ဖိုင်များကို အရန်သိမ်းသည်။ |

## ထပ်ပွားအရန်သိမ်းခြင်း ဝင်ရောက်ရာနေရာ

ဖွင့်ပါ:

```text
System Settings -> Other Settings -> Redundant Backup
```

![ထပ်ပွားအရန်သိမ်းခြင်း](../../image/other/冗余备份截图.png)

ဤဝင်ရောက်ရာနေရာသည် ဖိုင်တွဲတစ်ခု သို့မဟုတ် ဖိုင်အားလုံးအတွက် အရန်မိတ္တူများကို အစုလိုက် ထည့်ရန် သင့်တော်သည်။

အရန်ချန်နယ်ကို ကိုယ်တိုင်ရွေးချယ်နိုင်သည်။ သို့မဟုတ် အလိုအလျောက်ပြောင်းခြင်းကို ရွေးပြီး ImgBed ကို သင့်တော်သော အရန်ချန်နယ် ရှာစေနိုင်သည်။

## ဖိုင်အသေးစိတ်မှ အရန်သိမ်းခြင်း

စီမံခန့်ခွဲရေးပန်နယ်တွင် ဖိုင်အသေးစိတ်စာမျက်နှာကို ဖွင့်ပြီး အရန်သိမ်းရန် နှိပ်ပါ။

![ဖိုင်အသေးစိတ်ထဲမှ အရန်သိမ်းခြင်း](../../image/other/文件详情里文件备份.png)

ဤနည်းလမ်းသည် လိုအပ်သည့်အချိန်တွင် အရေးကြီးသော ဖိုင်တစ်ခုတည်းကို အရန်သိမ်းရန် သင့်တော်သည်။

အရန်သိမ်းမှု အောင်မြင်ပြီးနောက် ဖိုင်အသေးစိတ်စာမျက်နှာတွင် အသုံးပြုနိုင်သော ဖတ်ယူရင်းမြစ်များကို ပြသမည်။

## ရွေးချယ်မှုဖြင့် အစုလိုက် အရန်သိမ်းခြင်း

စီမံခန့်ခွဲရေးပန်နယ်တွင် ဖိုင်အများအပြားကို ရွေးချယ်ပြီး အစုလိုက် အရန်သိမ်းခြင်းကို လုပ်ဆောင်ပါ။

![အစုလိုက် အရန်သိမ်းခြင်း](../../image/other/批量备份截图.png)

ဤနည်းလမ်းသည် ဖိုင်အုပ်စုတစ်ခုကို တစ်ကြိမ်တည်း လုပ်ဆောင်ရန် သင့်တော်သည်။

ရွေးချယ်မှုဖြင့် အရန်သိမ်းခြင်း၊ ဖိုင်အသေးစိတ်မှ အရန်သိမ်းခြင်းနှင့် အခြားဆက်တင်များအောက်ရှိ ထပ်ပွားအရန်သိမ်းခြင်းတို့သည် အရန်သိမ်းစနစ်တစ်ခုတည်းကို အသုံးပြုသည်။ ဝင်ရောက်ရာနေရာများသာ ကွဲပြားသည်။

## အရန်သိမ်းပြီးနောက် ဖတ်ယူရင်းမြစ်ပြောင်းခြင်း

အရန်သိမ်းမှု ပြီးဆုံးပြီးနောက် ဖိုင်အသေးစိတ်စာမျက်နှာမှ ဖတ်ယူရင်းမြစ်ကို ပြောင်းနိုင်သည်။

| --- | --- |
| မူလချန်နယ် | မူလအပ်လုဒ်ချန်နယ်မှ ဖိုင်ကို ဖတ်ယူသည်။ |
| အရန်ချန်နယ် | အရန်ချန်နယ်မှ ဖိုင်ကို ဖတ်ယူသည်။ |

![အရန်သိမ်းမှုအောင်မြင်ပြီးနောက် ဖတ်ယူရင်းမြစ်ပြောင်းခြင်း](../../image/other/备份成功切换读取源.png)

ဧည့်သည်များသည် ဖိုင်ကို မူလချန်နယ်မှ ပေးပို့နေသည်လား၊ အရန်ချန်နယ်မှ ပေးပို့နေသည်လား သိရန် မလိုအပ်ပါ။

သင်ရွေးချယ်ထားသော ဖတ်ယူရင်းမြစ်သည် နောက်ပိုင်း ဖိုင်ဝင်ရောက်မှုများအတွက် ဦးစားပေးရင်းမြစ် ဖြစ်လာမည်။

## အရန်သိမ်းမှုကို ကျော်သွားမည့်အခြေအနေများ

အရန်သိမ်းနေစဉ် အောက်ပါအခြေအနေများကို ကျော်သွားမည်။ ၎င်းတို့သည် အမှားများ မဟုတ်ပါ။

| အခြေအနေ | ကျော်သွားရသည့်အကြောင်းရင်း |
| --- | --- |
| အရန်သိမ်းပြီးသား | အရန်မိတ္တူရှိပြီးသား ဖိုင်ကို ထပ်မံ အရန်မသိမ်းပါ။ |
| မူလချန်နယ်နှင့် အရန်ချန်နယ် တူနေသည် | အရန်သိမ်းခြင်းသည် အဓိပ္ပာယ်ရှိရန် အခြားချန်နယ်တစ်ခုတွင် သိမ်းဆည်းရမည်။ |
| အသုံးပြုနိုင်သော အရန်ချန်နယ် မရှိပါ | သင့်တော်သော အခြားချန်နယ် မရှိပါ။ |

အကျဉ်းချုပ်အားဖြင့် အရန်မိတ္တူများသည် အခြားချန်နယ်သို့ သွားရမည်ဖြစ်ပြီး အရန်သိမ်းပြီးသား ဖိုင်များသည် သိုလှောင်မှုနေရာကို ထပ်မံမသုံးပါ။

| အရန်ချန်နယ် | ထပ်ပွားမိတ္တူကို သိမ်းဆည်းထားသော ချန်နယ်။ |
| မူလဖတ်ယူရင်းမြစ် | လက်ရှိတွင် ဖိုင်ကို မူလချန်နယ်မှ ဖတ်ယူနေသည်။ |

အရန်ဖိုင် အသုံးပြုနိုင်နေသမျှ အရန်ဖတ်ယူရင်းမြစ်သို့ ပြောင်းပြီးနောက် ရုပ်ပုံများ၊ ဗီဒီယိုများနှင့် ဒေါင်းလုဒ်လင့်ခ်များ ဆက်လက်အလုပ်လုပ်မည်။

## ဖိုင်ဖျက်သည့်အခါ ဖြစ်ပျက်မည့်အရာ

ဖိုင်တစ်ခုကို ဖျက်သည့်အခါ ImgBed သည် မူလဖိုင်နှင့် အရန်ဖိုင် နှစ်ခုလုံးကို ဖျက်သည်။
