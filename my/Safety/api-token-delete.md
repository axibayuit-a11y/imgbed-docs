# API Token ဖြင့် ဖိုင်ဖျက်ခြင်း

API Token ဖျက်ခြင်းစခရစ်သည် စခရစ်၊ အလိုအလျောက်လုပ်ငန်းစဉ်များနှင့် ပြင်ပပရိုဂရမ်များမှ ImgBed ထဲရှိ ဖိုင်များကို ဖျက်ရန် အသုံးပြုသည်။ စီမံခန့်ခွဲရေးမျက်နှာပြင်ကို ဖွင့်ရန် မလိုပါ။ ဆိုက်လိပ်စာ၊ Token နှင့် တိကျသော ဖိုင် ID ကို ပေးလိုက်လျှင် ImgBed ထဲရှိ ဖိုင်တစ်ခု သို့မဟုတ် ဖိုင်အများအပြားကို ဖျက်နိုင်သည်။

ဖျက်ခြင်းသည် ရေးသားပြောင်းလဲသော လုပ်ဆောင်ချက်ဖြစ်သည်။ အမိန့်ကို လုပ်ဆောင်ပြီးသည်နှင့် ဖိုင်များကို တကယ်ဖျက်မည်။ ထို့ကြောင့် `imgbed-token-list.mjs` ဖြင့် ဖျက်လိုသော `fileId` ကို အရင်စစ်ပြီးမှ ထို ID များကို ဖျက်ခြင်းစခရစ်သို့ ပေးရန် အကြံပြုသည်။

![API Token ပြင်ဆင်ခြင်း](../../image/Safety/apitoken/编辑删除权限api.png)

## ကြိုတင်ပြင်ဆင်ရန်

စီမံခန့်ခွဲရေးမျက်နှာပြင်တွင် အောက်ပါနေရာသို့ သွားပါ။

```text
System Settings -> Security Settings -> API Token
```

API Token ကို ဖန်တီးခြင်း သို့မဟုတ် ပြင်ဆင်ခြင်းပြုလုပ်ပါက ထို Token တွင် ဖျက်ခွင့်ရှိကြောင်း စစ်ဆေးပါ။ ဤစခရစ်အတွက် `delete` ခွင့်ပြုချက်သာ လိုအပ်သည်။

Token ကို ပတ်ဝန်းကျင်ပြောင်းလဲကိန်းတွင်လည်း ထားနိုင်သည်။

```powershell
$env:IMGBED_API_TOKEN="your API Token"
```

## ဖျက်ခြင်း API ၏ လုပ်ဆောင်ပုံ

ဖျက်ခြင်းစခရစ်သည် ဆာဗာဘက် ဖျက်ခြင်း API ကို ခေါ်သည်။

```text
POST /api/manage/delete/batch
```

တောင်းဆိုမှုတွင် API Token ပါရမည်။

```text
Authorization: Bearer <token>
```

တောင်းဆိုမှုကိုယ်ထည် ဥပမာ:

```json
{
  "fileIds": ["photos/2026/a.txt"],
  "deleteStrictness": "strict"
}
```

`fileIds` ထဲတွင် ဖိုင်တစ်ခုသာ ပါပါက ဖိုင်တစ်ခုဖျက်ခြင်းဖြစ်ပြီး၊ ဖိုင်အများအပြား ပါပါက အုပ်စုလိုက်ဖျက်ခြင်းဖြစ်သည်။ ဆာဗာဘက်သည် တောင်းဆိုမှုတစ်ကြိမ်လျှင် အများဆုံး ဖိုင် ၁၅ ခုကိုသာ ကိုင်တွယ်သည်။ စခရစ်သည် `--batch-size` အတိုင်း တောင်းဆိုမှုများကို အလိုအလျောက် ခွဲပေးသည်။

API သည် NDJSON တိုးတက်မှုစီးကြောင်းကို ပြန်ပေးသည်။ မကြာခဏတွေ့ရသော ဖြစ်ရပ်များမှာ `batch_start`၊ `file_step`၊ `file_done`၊ `batch_complete`၊ `batch_error` တို့ဖြစ်သည်။ စခရစ်သည် ထိုဖြစ်ရပ်များကို ဖတ်ပြီး လူဖတ်ရလွယ်သော ရလဒ် သို့မဟုတ် JSON ရလဒ်အဖြစ် စုစည်းပေးသည်။

ဖျက်ခြင်းအောင်မြင်ပြီးနောက် ဆာဗာဘက်သည် ဖိုင်အညွှန်း၊ ဒါရိုက်ထရီစာရင်းအင်း၊ သိုလှောင်ပမာဏစာရင်းအင်းနှင့် ယာယီသိုလှောင်မှု ရှင်းလင်းခြင်းတို့ကို အလိုအလျောက် ဆက်လက်ကိုင်တွယ်သည်။

## ဖျက်ခြင်းစခရစ် ပါရာမီတာများ

| ပါရာမီတာ | လိုအပ်မှု | ရှင်းလင်းချက် |
| --- | --- | --- |
| `--base-url <url>` | လိုအပ် | ImgBed ဆိုက် URL။ ဥပမာ `https://image.ai6.me` |
| `--token <token>` | လိုအပ် | API Token။ `IMGBED_API_TOKEN` ပတ်ဝန်းကျင်ပြောင်းလဲကိန်းကိုလည်း သုံးနိုင်သည်။ |
| `--file-id <id>` | လိုအပ် | ဖျက်လိုသော ဖိုင် ID။ အကြိမ်များစွာ ပေးနိုင်သည်။ |
| `--strictness <strict\|soft>` | မလိုအပ် | ဖျက်ခြင်း၏ တင်းကျပ်မှု။ မူရင်းတန်ဖိုး `strict`။ |
| `--batch-size <n>` | မလိုအပ် | တောင်းဆိုမှုတစ်ကြိမ်တွင် ဖျက်မည့် ဖိုင်အရေအတွက်။ မူရင်းတန်ဖိုး `15`၊ အများဆုံး `15`။ |
| `--retries <n>` | မလိုအပ် | ယာယီမအောင်မြင်ပါက ပြန်ကြိုးစားမည့် အကြိမ်ရေ။ မူရင်းတန်ဖိုး `3`။ |
| `--timeout-ms <n>` | မလိုအပ် | တောင်းဆိုမှုတစ်ကြိမ်၏ အချိန်ကန့်သတ်ချက်။ မူရင်းတန်ဖိုး `180000`။ |
| `--output <pretty\|json>` | မလိုအပ် | ထုတ်ပြပုံ။ မူရင်းတန်ဖိုး `pretty`။ |
| `--save-response <path>` | မလိုအပ် | နောက်ဆုံးရလဒ်ကို JSON ဖိုင်အဖြစ် သိမ်းသည်။ |
| `-h` / `--help` | မလိုအပ် | စခရစ်အကူအညီကို ပြသည်။ |

ဤစခရစ်သည် `--file-id` ဖြင့် တိတိကျကျ ပေးထားသော ဖိုင်များကိုသာ ဖျက်သည်။ မတိကျသော ကိုက်ညီမှုရှာဖွေခြင်း မလုပ်ပါ။ ဒါရိုက်ထရီတစ်ခုလုံးကို အုပ်စုလိုက် မရှင်းပါ။ ကော်မာခွဲစာရင်း သို့မဟုတ် စက်တွင်းဖိုင်မှ ဖျက်ရန် ID များကိုလည်း မဖတ်ပါ။

## တင်းကျပ်ဖျက်ခြင်းနှင့် ပျော့ပျောင်းဖျက်ခြင်း

| မုဒ် | ရှင်းလင်းချက် |
| --- | --- |
| `strict` | မူရင်းမုဒ်ဖြစ်သည်။ အဝေးသိုလှောင်မှုဘက်တွင် ဖျက်ခြင်းမအောင်မြင်ပါက ImgBed မှတ်တမ်းကို ဆက်လက်ထားရှိသည်။ နောက်မှ ထပ်ကြိုးစားရန် သို့မဟုတ် စစ်ဆေးရန် လွယ်ကူစေသည်။ |
| `soft` | အဝေးသိုလှောင်မှုဘက်တွင် ဖျက်ခြင်းမအောင်မြင်ပါကလည်း ImgBed မှတ်တမ်းကို ဆက်လက်ရှင်းလင်းပြီး ရလဒ်ထဲတွင် သတိပေးချက်ကို ပြန်ပေးသည်။ |

အဝေးဘက်ရှိ ဖိုင်ကိုပါ ဖျက်နိုင်မှ အောင်မြင်သည်ဟု သတ်မှတ်လိုပါက မူရင်း `strict` ကို သုံးပါ။ အဝေးပလက်ဖောင်းတစ်ခုတွင် ဖျက်မရတော့သော်လည်း ImgBed မှတ်တမ်းကိုသာ ရှင်းလိုပါက `soft` ကို သုံးနိုင်သည်။

## အသုံးပြုပုံ ဥပမာများ

ဖိုင်တစ်ခု ဖျက်ရန်:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --file-id "photos/2026/a.txt"
```

ပတ်ဝန်းကျင်ပြောင်းလဲကိန်းထဲရှိ Token ကို သုံးရန်:

```powershell
$env:IMGBED_API_TOKEN="your API Token"

node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt"
```

ဖိုင်အများအပြား ဖျက်ရန်:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --file-id "photos/2026/b.txt" `
  --file-id "photos/2026/c.txt"
```

အဝေးဘက်ဖျက်ခြင်းမအောင်မြင်သော်လည်း ImgBed မှတ်တမ်းကို ရှင်းရန်:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --strictness soft
```

JSON အဖြစ် ထုတ်ပြီး ရလဒ်သိမ်းရန်:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --output json `
  --save-response ".\delete-result.json"
```

တောင်းဆိုမှုတစ်ကြိမ်တွင် ဖျက်မည့် ဖိုင်ကို ၅ ခုသာ ကန့်သတ်ရန်:

```powershell
node imgbed-token-delete.mjs `
  --base-url "https://your-domain" `
  --file-id "photos/2026/a.txt" `
  --file-id "photos/2026/b.txt" `
  --batch-size 5
```

## မဖျက်မီ fileId စစ်ရန်

ဖျက်ခြင်းစခရစ်အတွက် လိုအပ်သည်မှာ ImgBed ဖိုင် ID ဖြစ်သည်။ ဒါရိုက်ထရီထဲရှိ ဖိုင်များကို စာရင်းစခရစ်ဖြင့် အရင်ကြည့်နိုင်သည်။

```powershell
node imgbed-token-list.mjs `
  --base-url "https://your-domain" `
  --token "your API Token" `
  --files `
  --dir "photos/2026" `
  --count 10 `
  --output json
```

ပြန်လာသောရလဒ်ထဲရှိ `name` သည် များသောအားဖြင့် ဖျက်ခြင်းစခရစ်သို့ ပေးနိုင်သော `fileId` ဖြစ်သည်။

## မေးလေ့ရှိသော မေးခွန်းများ

### ဖျက်ခြင်းမအောင်မြင်သော်လည်း ဖိုင်သည် စာရင်းထဲတွင် ကျန်နေသေးသလဲ

မူရင်း `strict` ကို သုံးပါက အဝေးသိုလှောင်မှုဘက် ဖျက်ခြင်းမအောင်မြင်သောအခါ ImgBed မှတ်တမ်းကို ဆက်လက်ထားရှိသည်။ စက်တွင်းအညွှန်းသာ ဖျက်ပြီး အဝေးဖိုင်က ကျန်နေခြင်းကို ရှောင်ရန်ဖြစ်သည်။ ImgBed မှတ်တမ်းကိုသာ ရှင်းနိုင်ကြောင်း သေချာပါက ထို `fileId` ကို `soft` ဖြင့် ထပ်ကြိုးစားပါ။

### ရလဒ်ထဲတွင် သတိပေးချက် ပါလာရခြင်းအကြောင်းရင်းက ဘာလဲ

သတိပေးချက်သည် များသောအားဖြင့် အဝေးဘက်ဖျက်ခြင်း၊ ယာယီသိုလှောင်မှု ရှင်းလင်းခြင်း သို့မဟုတ် စာရင်းအင်းအဆုံးသတ်လုပ်ငန်းစဉ်တွင် မပြင်းထန်သော ပြဿနာ ရှိကြောင်း ဆိုလိုသည်။ စခရစ်သည် သတိပေးချက်များကို စုစည်းပြသပေးသောကြောင့် ထပ်ကြိုးစားရန် လိုမလို ဆုံးဖြတ်နိုင်သည်။

### ဒါရိုက်ထရီအလိုက် တစ်ကြိမ်တည်း ဖျက်နိုင်သလား

ဤစခရစ်တွင် ဒါရိုက်ထရီတစ်ခုလုံးကို ရှင်းလင်းသော လုပ်ဆောင်ချက် မပါပါ။ စာရင်းစခရစ်ဖြင့် တိကျသော `fileId` များကို အရင်ရွေးပြီး၊ ဖျက်လိုသော ဖိုင်များကို တစ်ခုချင်း ဖျက်ခြင်းစခရစ်သို့ ပေးရမည်။
