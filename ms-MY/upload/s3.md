# Tambah Saluran S3

## Paling Sesuai

Gunakan saluran S3 apabila:

- Anda mahu menyambungkan mana-mana perkhidmatan storan objek yang serasi dengan S3, seperti Cloudflare R2, Backblaze B2, MinIO, AWS S3 atau penyedia serasi lain.
- Anda memilih model konfigurasi S3 standard: `Endpoint`, Access Key dan Secret Key.
- Anda tidak mahu menggunakan saluran binding Cloudflare R2 asli, atau penyedia storan anda bukan Cloudflare R2.

## Perkara Yang Diperlukan Sebelum Bermula

| Keperluan | Tujuan |
| --- | --- |
| S3 Endpoint | Endpoint API S3 bagi perkhidmatan storan objek. |
| Nama bucket | Bucket tempat fail akan disimpan. |
| Region | Rantau storan. Untuk Cloudflare R2, nilai ini biasanya `auto`. |
| Access Key ID | ID access key S3. |
| Secret Access Key | Secret access key S3. |
| CDN domain | Domain tersuai pilihan untuk akses fail. |

Contoh untuk Cloudflare R2 melalui API S3:

```text
Endpoint: https://your-account-id.r2.cloudflarestorage.com
Bucket: your-r2-bucket-name
Region: auto
Access Key ID: copied from the Cloudflare R2 API token
Secret Access Key: copied from the Cloudflare R2 API token
```

## Tempat Menambahkannya

1. Buka Tetapan Sistem.
2. Pergi ke Tetapan Muat Naik.
3. Klik Tambah Saluran di penjuru kanan atas.
4. Pilih `S3`.

## Rujukan Medan

| Medan | Fungsi | Wajib |
| --- | --- | --- |
| Nama saluran | Nama mesra untuk saluran S3 ini, seperti `s3test` atau `R2-S3`. | Ya |
| Dayakan saluran | Mengawal sama ada saluran ini mengambil bahagian dalam pemilihan muat naik. | Disyorkan |
| Endpoint | Endpoint penuh perkhidmatan S3, termasuk `https://`. | Ya |
| Domain CDN tersuai | Pilihan. Apabila ditetapkan, pautan fail yang dijana akan mengutamakan domain ini. | Tidak |
| Nama bucket | Nama bucket, seperti `s3test` atau `img-r2`. | Ya |
| Rantau bucket | Region. Untuk Cloudflare R2, nilai ini biasanya `auto`. | Ya |
| Access Key ID | ID access key S3. | Ya |
| Secret Access Key | Secret access key S3. | Ya |
| Akses path-style | Suis keserasian. Dimatikan secara lalai. Sesetengah MinIO, OpenList atau perkhidmatan S3 self-hosted mungkin memerlukannya. | Tidak |
| Had kuota | Mengawal sama ada saluran S3 ini mengambil bahagian dalam pemilihan muat naik berdasarkan kapasiti. | Tidak |
| Had kapasiti | Diperlukan selepas had kuota didayakan, contohnya `10 GB`. | Wajib apabila had kuota didayakan |
| Ambang | Menghentikan penulisan apabila penggunaan mencapai peratusan ini, contohnya `90%`. | Wajib apabila had kuota didayakan |
| Catatan | Nota untuk penyelenggaraan anda sendiri. | Tidak |

## Buat Kunci API S3 Cloudflare R2

1. Buka Cloudflare Dashboard.
2. Pergi ke `R2 Object Storage`.
3. Cari `Manage R2 API Tokens` atau entri pengurusan token API.

![Entri token API R2](../../image/upload/s3/r2%20api%20token入口.png)

4. Buat token API R2 yang boleh mengakses bucket sasaran.

![Buat token API akaun](../../image/upload/s3/创建账户api令牌.png)

5. Salin `Access Key ID` dan `Secret Access Key` yang dijana.

![Salin nilai token](../../image/upload/s3/复制令牌值.png)

6. Kembali ke halaman bucket R2 dan sahkan nama bucket.
7. Rekodkan endpoint API S3 untuk akaun. Biasanya bentuknya seperti ini:

```text
https://your-account-id.r2.cloudflarestorage.com
```

## Langkah Persediaan

1. Buka Tetapan Muat Naik.
2. Klik Tambah Saluran.
3. Pilih `S3`.
4. Masukkan nama saluran yang mudah dikenal pasti, contohnya `s3test`.
5. Masukkan endpoint API S3 dalam `Endpoint`.
6. Jika anda menggunakan CDN tersuai, masukkannya dalam `Custom CDN domain`; jika tidak, biarkan kosong.
7. Masukkan nama bucket.
8. Masukkan region. Untuk contoh Cloudflare R2, gunakan `auto`.
9. Masukkan Access Key ID dan Secret Access Key.
10. Biarkan akses path-style dimatikan kecuali penyedia anda secara jelas memerlukannya.
11. Jika anda mahu kawalan kapasiti, dayakan had kuota dan masukkan had kapasiti serta ambang.
12. Klik Simpan.

![Tambah konfigurasi S3](../../image/upload/s3/添加s3配置.png)

## Cara Mengesahkan

| Semakan | Cara Mengesahkan |
| --- | --- |
| Kad saluran muncul | Selepas menyimpan, halaman Tetapan Muat Naik sepatutnya memaparkan kad saluran S3. |
| Saluran didayakan | Suis di penjuru kanan atas kad sepatutnya kekal hidup. |
| Medan utama disimpan | Paparan butiran sepatutnya menunjukkan Endpoint, bucket, region, akses path-style dan medan berkaitan. |
| Muat naik berfungsi | Muat naik imej ujian dan sahkan objek muncul dalam bucket sasaran. |
| Pautan dibuka | Pautan imej yang dikembalikan selepas muat naik sepatutnya dibuka seperti biasa. |
| Paparan kapasiti berfungsi | Jika had kuota didayakan, kad sepatutnya menunjukkan kapasiti digunakan dan had yang dikonfigurasikan. |

Statistik kapasiti S3 dikira daripada rekod fail setempat ImgBed, bukan dengan menanya bucket penyedia secara masa nyata. Selepas konfigurasi S3 disimpan, sistem mengira semula lejar kuota daripada rekod D1 semasa.

## FAQ

### Patutkah akses path-style didayakan?

Biasanya tidak.

Dayakannya hanya apabila penyedia serasi S3 anda tidak menyokong URL virtual-hosted-style, atau apabila dokumentasi penyedia secara jelas memerlukan akses path-style. Sesetengah endpoint MinIO, S3 self-hosted dan endpoint serasi OpenList mungkin memerlukannya.

### Apakah kegunaan CDN domain?

Jika anda meletakkan CDN atau domain akses tersuai di hadapan bucket, masukkannya di sini.

Selepas dikonfigurasikan, pautan fail yang dijana akan mengutamakan domain ini. Jika anda tidak menggunakan CDN, biarkan kosong.

### Mengapa muat naik gagal?

Semak perkara berikut dahulu:

1. Endpoint menyertakan URL penuh dengan `https://`.
2. Nama bucket adalah betul.
3. Region sepadan dengan keperluan penyedia.
4. Access Key ID dan Secret Access Key telah disalin sepenuhnya.
5. Kunci mempunyai kebenaran tulis untuk bucket sasaran.
6. Penyedia tidak memerlukan akses path-style, atau akses path-style telah didayakan jika diperlukan.

```text
Prepare S3 Endpoint, Bucket, Region, Access Key, and Secret Key
-> Open Upload Settings
-> Add Channel

-> Leave path-style access off by default
-> Enable quota limit if needed
-> Save
-> Upload a test image and check the result
```
